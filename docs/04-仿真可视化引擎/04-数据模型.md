# M4 仿真可视化引擎 — 数据模型

> PostgreSQL。仿真包定义为平台级;会话、操作序列、检查点和分享索引为租户级(tenant_id + RLS)。
> 建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 | 范围 |
| --- | --- | --- |
| `sim_package` | 仿真包定义 + 版本 | 全局 |
| `sim_package_review` | 仿真包接入审核 | 全局 |
| `sim_session` | 仿真会话(一次运行) | 租户 |
| `sim_action_log` | 操作序列(种子+事件,支持回放) | 租户 |
| `sim_checkpoint` | 叙事设问/目标达成检查点上报 | 租户 |
| `sim_share` | 分享剧本索引(全局 code 索引 + 租户访问控制) | 租户 |

> 消灭冗余/合并说明:
> - **仿真包内容(state结构/reducer/view/interactions/narrative)体积大且为代码/JSON**,正文存对象存储(MinIO),表中只存元信息 + 存储引用 + hash,不入库大字段。
> - **版本不单独建表**:`sim_package` 以 `(code, version)` 联合唯一表达多版本,同一 code 多行即多版本(避免包主表与版本表拆分后的冗余 join)。
> - **操作序列不存状态快照**:`sim_action_log` 只存 `seed + 有序事件`,回放时重算,天然去重(确定性)。
> - **检查点不存答案正本**:`sim_checkpoint` 只保存本次会话的学生作答/达成结果快照,正确答案仍归 M5 或仿真包叙事元数据所有,M4 不承担判分。
> - **分享码只做索引**:`sim_share` 保存分享码、会话引用与过期/撤销状态,分享内容由 session + action_log 重建,不复制一份剧本。

## 2. 平台级表

### 2.1 sim_package — 仿真包(含版本)

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| code | VARCHAR(96) | 仿真标识(含命名空间前缀,如 `teacher_12__xxx`) |
| version | VARCHAR(32) | semver |
| name | VARCHAR(128) | 展示名 |
| category | VARCHAR(32) | 领域分类(检索/配色) |
| compute | SMALLINT | 1前端 / 2后端 |
| scale_limit | JSONB | 性能边界(节点数/max_tick) |
| bundle_key | VARCHAR(255) | 包正文在对象存储路径(逻辑/视图/交互/叙事) |
| bundle_hash | VARCHAR(64) | 包内容哈希(完整性/复现) |
| backend_adapter | VARCHAR(96) NULL | `compute=backend` 时的 M4 后端计算适配器标识 |
| backend_config | JSONB | `compute=backend` 时的资源/协议配置;前端仿真为空对象 |
| interaction_schema | JSONB | 从 `sim-package.json.interactions` 提取的事件与参数白名单 |
| code_trace | JSONB | 从 `sim-package.json.codeTrace` 提取的代码追踪审核摘要,不含源码正文 |
| author_type | SMALLINT | 1平台内置 / 2教师 / 3第三方 |
| author_id | BIGINT NULL | 作者(教师扩展时),取 M1 全平台唯一雪花账号 ID |
| status | SMALLINT | 1草稿 / 2审核中 / 3已上架 / 4已下架 / 5退回 |
| created_at / updated_at | TIMESTAMPTZ | |

唯一约束:`(code, version)`。
> 教师提交只能落入 `author_type=2` 且 `code` 形如 `teacher_<account_id>__...`;平台内置与第三方命名空间由后端/平台端受控生成,不接受教师请求伪造。`compute=frontend` 时 `backend_adapter` 与 `backend_config` 必须为空对象;`compute=backend` 只有在组合根真实注册适配器时才允许进入审核。`interaction_schema` 是操作上报服务端校验的权威白名单;`code_trace` 只保存语言、行数、映射数和变量数等审核摘要,源码正文仍只存在 bundle 对象内。

### 2.2 sim_package_review — 接入审核

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| package_id | BIGINT | 被审核的仿真包 |
| submitter_id | BIGINT | 提交人,取 M1 全平台唯一雪花账号 ID |
| preview_report | JSONB | 静态扫描、确定性校验与受控预览结果 |
| reviewer_id | BIGINT NULL | 审核人(平台管理员) |
| result | SMALLINT | 1待审 / 2通过 / 3退回 |
| comment | VARCHAR(500) NULL | 审核意见 |
| created_at / updated_at | TIMESTAMPTZ | |

## 3. 租户级表

### 3.1 sim_session — 仿真会话

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| package_id | BIGINT | 使用的仿真包(锁定版本) |
| source_ref | VARCHAR(128) | 来源标识(实验/课时,M4 不解析语义) |
| owner_account_id | BIGINT | 使用者 |
| seed | BIGINT | 随机种子(确定性复现) |
| init_params | JSONB | 初始参数 |
| compute | SMALLINT | 实际运行位置 1前端/2后端 |
| status | SMALLINT | 1创建中 / 2进行中 / 3空闲 / 4已完成 / 5已归档 / 6失败 |
| created_at / updated_at | TIMESTAMPTZ | |

### 3.2 sim_action_log — 操作序列

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| session_id | BIGINT | 所属会话 |
| seq | INT | 序号(有序) |
| at_tick | INT | 发生在第几 tick |
| event_type | VARCHAR(64) | 事件类型 |
| payload | JSONB | 参数 + target |
| created_at | TIMESTAMPTZ | |

> 会话回放 = `seed + 按 seq 重放 sim_action_log`。检查点结果不在此存,经 M3 处理。公开分享入口输出 `payload` 前必须按 `pkg/privacy` 的用户可见结果敏感字段规则过滤答案、flag、token、密钥和 `_` 开头内部字段;租户内授权回放可读取完整操作序列。

### 3.3 sim_checkpoint — 检查点上报

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| session_id | BIGINT | 所属仿真会话 |
| checkpoint_id | VARCHAR(96) | 仿真叙事/目标检查点标识 |
| answer | JSONB | 学生作答或目标状态快照 |
| achieved | BOOLEAN | 是否由前端/后端运行时判定达成 |
| created_at | TIMESTAMPTZ | |

唯一约束:`(tenant_id, session_id, checkpoint_id)`,重复上报按最新结果覆盖,保证 M3 读取同一检查点时有唯一快照。

### 3.4 sim_share — 分享剧本索引

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | 被分享会话所属租户,用于解析分享码后回到租户 RLS 读取会话 |
| session_id | BIGINT | 被分享的会话 |
| code | VARCHAR(48) | 分享码,全局唯一,不可从 session_id 推导 |
| created_by | BIGINT | 分享人 |
| status | SMALLINT | 1有效 / 2撤销 / 3过期 |
| expire_at | TIMESTAMPTZ NULL | 过期时间,为空表示不自动过期 |
| created_at / updated_at | TIMESTAMPTZ | |

> `sim_share.code` 全局唯一,用于公开分享码解析;记录本身仍属于被分享会话的租户并启用 RLS。公开分享入口先按 code 解析出 `tenant_id`,再设置租户上下文读取 `sim_share`、`sim_session` 与 `sim_action_log`。该表不保存剧本正文、操作序列或答案,只保存不可推导的 code 与 session_id/tenant_id 指针。

## 4. 多租户隔离

- 租户级表(`sim_session`/`sim_action_log`/`sim_checkpoint`/`sim_share`)带 `tenant_id` + RLS(同 M1)。
- `sim_share` 为公开分享码解析保留全局唯一 code,但记录读取仍必须回到所属租户上下文,不存敏感正文。
- `sim_package`/`sim_package_review` 平台级;教师提交的扩展包经审核后全平台可见(命名空间前缀防冲突)。`author_id`、`submitter_id`、`reviewer_id` 均使用 M1 雪花账号/平台管理员 ID,在全平台唯一;M4 不在平台级包表重复保存作者租户,租户边界通过上传对象 key、提交审计和服务端身份派生的 `teacher_<account_id>__` 命名空间约束。
- 私有化:`tenant_id` 恒定,逻辑一致;仿真包可离线导入。

## 5. 关键索引

- `sim_package(code, version)` UNIQUE、`sim_package(status, category)`
- `sim_session(tenant_id, owner_account_id)`、`sim_session(tenant_id, source_ref)`
- `sim_action_log(tenant_id, session_id, seq)` UNIQUE(回放顺序读取与幂等上报)
- `sim_checkpoint(tenant_id, session_id, checkpoint_id)` UNIQUE
- `sim_share(code)` UNIQUE、`sim_share(tenant_id, session_id)`
