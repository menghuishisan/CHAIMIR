# M4 仿真可视化引擎 — 安全设计

> M4 运行第三方/教师提交的仿真包(代码),且默认在前端执行。本文覆盖:插件可信边界、前端执行隔离、审核、数据隔离。
> 最后更新:2026-05-29

---

## 1. 核心风险

仿真包是**外部提交的可执行代码**,主要风险:
- 恶意代码:窃取 token、发恶意请求、挖矿、XSS。
- 不稳定代码:死循环、内存泄漏、卡死浏览器。
- 不纯 reducer:破坏确定性,导致回放/判分不一致。

---

## 2. 前端执行隔离

仿真包代码**不得在主页面上下文直接执行**:

- **Web Worker 沙箱**:仿真 reducer/initState 在独立 Web Worker 中运行,无法访问主页面 DOM、cookie、localStorage、平台 token。
- **能力白名单**:Worker 内只暴露受控的 SDK 渲染消息通道;平台 Worker loader 只允许加载已审核通过的 bundle 入口,仿真包自身禁止 `fetch`/`XMLHttpRequest`/`eval`/动态 import 等危险能力(通过静态扫描 + 受限运行时 + CSP)。
- **渲染消息化**:仿真产出 ViewSpec(纯数据)经消息传回主线程,由平台可信渲染器绘制;仿真代码**不直接操作画布/DOM**。
- **CSP 加固**:页面 Content-Security-Policy 禁止内联脚本与外部脚本注入,阻断 XSS。

---

## 3. 资源与稳定性保护

- **执行超时**:单次 Worker 命令设超时,超时终止该 Worker 并提示仿真异常。阈值从
  `deploy/config/chaimir.env` 的 `SIM_WORKER_COMMAND_TIMEOUT_MS` 注入前端装配层,
  再作为 `workerCommandTimeoutMs` 传入 `sim-sdk`,不得在 SDK 内写死生产阈值。
- **规模上限**:仿真包声明 `scale_limit`(节点数/max_tick),运行时强制;超限降级或拒绝。
- **内存监控**:Worker 内存异常增长则熔断,防卡死浏览器。
- **确定性校验**:审核时用同 seed 跑两遍比对结果,reducer 非纯函数(结果不一致)则拒绝上架。

---

## 4. 仿真包审核

上架前必经:

- **沙箱化预览**:在隔离环境(同前端 Worker 隔离 + 受限网络)运行仿真包,人工 + 自动检查。
- **静态扫描**:扫描包代码危险调用模式(网络、eval、全局对象访问),命中告警/拒绝。
- **确定性测试**:同输入多次运行结果一致性校验。
- **命名空间**:教师/第三方包强制 `teacher_<id>__` / `org_<id>__` 前缀,防覆盖内置包。教师入口的 `author_type` 与 `author_id` 由服务端身份派生,不得接受客户端传入值伪造平台内置或第三方命名空间。
- **版本不可变**:已上架版本内容不可改,修改须发新版本(保证已用实验稳定)。
- **审核门禁归后端控制**:上传阶段由后端写入 `bundle_hash`、`metadata_validation`、`static_scan`;受控预览只能通过内部服务签名补充 `determinism_check`、`worker_preview`,平台管理员不得手工伪造预览结果。审核通过时必须重新校验报告中的 hash 与 `sim_package.bundle_hash` 一致,且四项均为 `passed`,不能信任调用方临时提交的 validation report。
- **预览数据最小可见**:`GET /packages/{id}/preview` 仅允许包作者读取自己的待审报告;平台管理员通过审核列表读取。任意教师/学校管理员不得通过猜测 id 读取其他作者 pending review 数据。

---

## 5. 后端计算仿真的隔离(compute=backend)

少数后端运行的仿真:

- 由 M4 自有后端计算适配器承接,适配器只接受审核通过的 `backend_adapter` 与协议配置。
- 当前部署未注册真实适配器时,`compute=backend` 统一返回 `42005` 并禁止上架,不得用空实现、回显实现或伪适配器冒充生产路径。
- 后端仿真执行必须运行在受控仿真镜像会话或等价隔离环境中,具备资源限制、网络 deny-all、non-root 与超时控制。
- M4 不直接调用 M2 沙箱接口,避免同层模块互相依赖;真实链沙箱数据由 M7 等上层编排后作为输入事件传入。
- 同一确定性与超时约束。
- 与平台核心服务、其他租户数据隔离。

---

## 6. 数据与多租户

- 仿真会话/操作序列带 `tenant_id` + RLS,跨租户不可见。
- 操作上报接口校验会话归属(owner/source_ref 对应租户与用户),防越权写他人会话。内部单会话回收、批量回收和检查点上报必须校验已验签服务请求中的 `source_ref` 与目标会话一致,禁止合法内部调用者篡改其他来源会话。
- 分享剧本:分享码不含敏感信息,仅含 `package+version+seed+params+actions`;跨租户复现仅复制公开仿真逻辑,不泄露数据。
- 检查点上报只保存学生作答/达成结果快照,不保存正确答案;正确答案和判分规则仍由 M5/M3 管理。
- 包审核通过/退回、上下架、会话创建/归档等敏感生命周期动作必须写入 M1 统一 `audit_log`;模块内 `sim_action_log` 只保存仿真交互序列,不得替代审计。

---

## 7. 安全测试要点

- 仿真包尝试访问 DOM/cookie/token/网络必须被 Worker 隔离 + CSP 阻断。
- 恶意/死循环仿真被超时与资源熔断,不影响页面与其他仿真。
- 非纯 reducer(破坏确定性)在审核期被拒。
- 越权上报/读取他人会话被拒。
- 内置包 code 不可被教师扩展包覆盖。
