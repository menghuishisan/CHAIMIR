# M4 仿真可视化引擎 — 需求规格

> 模块定位:教学可视化引擎。让区块链概念可视化、可交互、可探索,服务"理解"而非"展示"。
> 核心理念:从"渲染引擎"转向"教学引擎"。平台只提供框架,具体仿真为自描述插件。
> 最后更新:2026-05-29

---

## 1. 模块目标

- **可交互**:学生不只看动画,可对仿真注入操作(调参、发交易、注入攻击、制造分区等),即时看到后果。
- **可扩展**:任意仿真为自描述插件包,第三方/教师按规范开发上传,审核即接入,平台代码不改。
- **教学导向**:可视化嵌入引导式叙事(分步讲解、关键步骤释义、设问),让学生学懂"为什么"。
- **即时低成本**:仿真默认在前端运行,零延迟、秒开、可离线、不占服务器算力。
- **可复现**:种子化确定性,支持单步回退、回放、分享操作序列、被 M3 判分。
- **够用且可达**:补齐响应式与基础无障碍,砍掉过度复杂的机制。

## 2. 设计哲学(对旧版的根本性优化)

旧版 SimEngine 是"渲染引擎"(关注怎么把状态画出来),M4 转向"教学引擎"(关注学生怎么学懂)。三个转变:

| 维度 | 旧版 | M4 |
| --- | --- | --- |
| 表达单位 | 47 个几何原语堆叠 | 区块链概念模型 + 高频复合组件 |
| 内容形态 | 动画播放 | 引导式叙事(分步/释义/设问) |
| 机制取向 | 4 模式判定 + 跨画布联动弧线 + owner 共享状态 | 单人仿真 + 轻量并排对比,够用可达 |
| 品质 | 放弃 i18n/a11y/移动端 | 补齐响应式 + 基础无障碍 |

## 3. 职责边界

| M4 负责 | M4 不负责(归属) |
| --- | --- |
| 仿真运行时(加载/驱动状态机) | 真实链运行环境(→ M2) |
| 可视化 SDK 与通用交互渲染 | 判分(→ M3,M4 提供检查点数据) |
| 仿真插件的接入/审核/版本 | 题库与正确答案存储(→ M5) |
| 操作序列记录与回放 | 挂哪节课、算不算分、谁做了(→ M6/M7) |
| 教学叙事承载 | 课程/班级/成绩(→ M6) |

**核心交互**:调用方传 `仿真包 + 初始参数 + 种子 + 来源标识` → M4 启动仿真会话 → 学生交互 → 产出 `操作序列 + 状态 + 检查点结果`。

## 4. 仿真包(Sim Package)—— 自描述四件事

平台不内置任何具体仿真或交互;每个仿真是自描述插件,声明:

| 部分 | 内容 |
| --- | --- |
| **state 状态** | 该仿真有哪些数据(节点/链/内存池/轮次…),完全自定义 |
| **interactions 交互** | 声明式列表;每条自述触发方式、参数、作用目标、可用条件、视觉标签(见 §5) |
| **view 视图** | 用平台可视化 SDK 描述怎么把状态画出来 |
| **narrative 教学叙事** | 可选;分步引导 + 关键步骤释义 + 设问检查点 |

## 5. 通用交互机制(彻底不固定)

交互是声明式列表,**攻击只是交互的一种,不被特殊对待**。每条交互自描述:

| 字段 | 说明 | 例 |
| --- | --- | --- |
| `kind` 触发方式 | 按钮 / 滑块调参 / 按住持续 / 选中元素 / 拖拽 / 表单 | 调矿工算力=滑块;发交易=表单 |
| `params` 参数 | 任意类型字段 + 约束(范围/枚举/默认),仿真自定义 | 算力数值、目标地址 |
| `target` 作用目标 | 全局 / 作用于选中的某节点或边 | 让某节点变拜占庭 |
| `available_when` 可用条件 | 在什么状态/阶段可用 | 仅出块阶段可发起 |
| `label_tag` 视觉标签 | 常规/扰动/攻击,仅用于配色与二次确认,机制统一 | 攻击类红色+二次确认 |

→ 调参、发交易、投票、质押、加节点、制造分区、注入拜占庭、自由攻击……全是同一套通用机制的不同声明,无任何写死。

## 6. 功能需求

### A. 仿真运行时
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 加载仿真包 | 解析自描述包,初始化 state |
| A2 | 驱动状态机 | 按 tick 演化;消费交互事件 → 产出新状态 |
| A3 | 计算位置 | 默认前端运行;重计算仿真可声明后端运行,协议一致 |
| A4 | 种子化确定性 | 固定 seed 演化唯一,支持复现 |

### B. 可视化渲染(SDK)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 分层 SDK | 底层画布能力 + 高频复合组件(区块链/默克尔树/共识轮次面板) |
| B2 | 通用交互渲染器 | 按交互声明自动渲染控件(滑块/输入/下拉/选目标/按钮) |
| B3 | 响应式布局 | 适配不同屏幕(含笔记本) |
| B4 | 基础无障碍 | 键盘可达、语义化、对比度 |

### C. 交互与探索
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 通用交互 | 见 §5,任意交互声明即可用 |
| C2 | 单步推进/回退 | 基于确定性重算到上一 tick |
| C3 | 变速/暂停 | 动画节奏控制 |
| C4 | 选中元素操作 | 点选节点/边后施加针对性交互 |

### D. 教学叙事
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 分步引导 | 按步骤推进 + 高亮 + 解说 |
| D2 | 关键步骤释义 | 解释"为什么这步重要" |
| D3 | 设问检查点 | 学生预测再揭晓;结果可供 M3 判分 |
| D4 | 可选层 | 纯沙盒探索仿真可不写叙事 |

### E. 回放与分享
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 操作序列记录 | 记录种子 + 操作序列(非状态快照) |
| E2 | 回放 | 重放操作序列复现演化 |
| E3 | 分享剧本 | 导出操作剧本供他人复现 |

### F. 插件生命周期
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 仿真包注册 | 提交自描述包 |
| F2 | 审核上架 | 平台审核 + 沙箱化预览,通过才上架 |
| F3 | 版本管理 | semver,实验锁定版本 |

### G. 并排对比(轻量)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| G1 | 多仿真并排 | 几个独立仿真同屏并排,各自独立演化 |
| G2 | 参数对比 | 同一仿真不同参数双实例并排(替代旧版复杂状态联动) |

## 7. 非功能需求

- **即时**:前端运行,操作零延迟,仿真秒开。
- **可复现**:种子化确定性,回放/分享/判分一致。
- **可扩展**:仿真包插件化,接入不改平台代码;与 M2 链、M3 判题器同一扩展哲学。
- **性能边界**:前端规模上限(节点数/tick 上限),超限降级或提示。
- **可达**:响应式 + 基础无障碍。

## 8. 明确不做(边界)

- 不做旧版跨画布因果弧线、owner 共享状态、4 模式判定等复杂联动。
- 不提供真实链环境(归 M2);不算成绩(归 M6);不存正确答案(归 M5)。
- 不内置任何具体仿真/交互(全部由插件声明)。

## 9. 技术取向

仿真运行时与 SDK 以 TypeScript/前端为主(React);确定性状态机纯函数化;重计算仿真可选后端执行同一协议;绘图用 Canvas/SVG/WebGL 按需。最终技术细节由《技术选型》统一收口。
