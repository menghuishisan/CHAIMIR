# 前端 UI/UX 全面重构审计（2026-07-22）

## 1. 目的与边界

本文基于以下事实源形成前端重构方案：

- `docs/总-前端设计规范.md`：令牌、组件、日常/沉浸双模态、响应式、动效与无障碍的单一真相源。
- `docs/前端页面设计/*.md`：认证、学生、教师、学校管理、平台管理共 104 个页面位置的布局与交互草图。
- `docs/前端后端功能对齐清单.md`：公开能力、角色权限、入口层级、DTO 与异步任务契约。
- `frontend/apps/web`、`frontend/packages/ui`、`frontend/packages/sim-sdk`：当前真实实现。
- `.tmp/e2e-2026-07-14/screenshots` 与 `.tmp/e2e-regression-2026-07-15/screenshots`：2026-07-14 至 2026-07-15 的历史浏览器证据。

本轮只确定设计问题、重构边界、优先级和验收标准，不直接改造业务页面。历史截图用于定位问题，不作为新视觉基准；重构必须以已实现功能、页面语义和后端真实能力重新设计，不照抄现有外观，也不恢复已删除的旧原型。

共享层与仿真工作台的目标设计、旧设计隔离规则、组件批次和七类模式验收见 `docs/前端共享UI与仿真工作台目标设计-2026-07-22.md`。四端页面重构必须先消费该共享基线,不得先在业务页复制一套新外观再反向抽象。

## 2. 成功标准

重构完成后必须同时满足：

1. 104 个页面位置均有明确的信息架构、状态、入口层级、响应式和权限行为，不存在空白页、假进度、假图表或内部字段泄漏。
2. 日常页面、沉浸工作台、认证页分别使用统一结构原语，但不同角色和业务不被强制套成同一布局。
3. 断点、间距、颜色、圆角、阴影、动效和状态全部由设计系统约束；业务 CSS 不再各自发明同类规则。
4. 375、768、1024、1440 四个验收宽度无横向溢出、文字遮挡、固定栏覆盖和不可达操作。
5. 键盘、读屏语义、焦点可见、44px 触控目标、WCAG AA 对比度和 `prefers-reduced-motion` 均通过浏览器验收。
6. 所有数据、进度、图表、下载、WebSocket 和工作台状态来自公开后端契约；服务端未实现的能力不显示为可操作功能。

## 3. 当前实现盘点

审计快照中，单一前端入口已有 96 个懒加载页面导出、121 个路由元素、101 个业务页面 TSX 文件和 79 个 CSS 文件。设计文档定义 104 个页面位置，数量差异来自拦截态、共享页复用和一个页面承载多个路由，不应只以路由数量判断覆盖率。

当前的正向基础包括：

- `frontend/apps/web` 已是唯一浏览器入口，路由按角色和业务模块拆分。
- `@chaimir/ui` 已提供 Table、Modal、Drawer、FormField、状态组件、WorkbenchShell 等共享原语。
- 业务 CSS 未发现绕过令牌的裸十六进制颜色。
- Table 已具备移动端重排机制，ResourceState 已具备加载、错误、空状态和重试基础。
- 认证页的左右分栏、深浅层级和品牌表达是当前最接近规范的页面族。
- 仿真工作台已经连接真实 SimulationWorkbench 和后端会话，重构重点是视觉层级与交互性能，不是另写仿真引擎。

## 4. 全局问题与目标方案

优先级定义：P0 阻断真实可用性或暴露内部语义；P1 影响主要流程、移动端或全站一致性；P2 影响质感、效率和长期维护。

| 优先级 | Before | After | Why |
| --- | --- | --- | --- |
| P0 | 实验报告批改直接展示 `student_id`、`content_ref`、`instance_id`，预览区为空白框 | 展示学生身份摘要、已渲染 Markdown/附件/截图、实例时间线和评分表；内部引用仅用于请求参数 | 教师需要判断报告内容，不应理解存储引用或数据库字段 |
| P0 | 任务中心按状态套用 15/35/55/100 的固定宽度伪装进度 | 只有后端提供总量和已完成量时显示确定进度；否则显示排队、处理中、成功、失败等离散状态 | 假进度误导用户，也违反后端能力边界 |
| P0 | 历史截图存在教师实验编排、监控页近乎空白 | 每个路由必须具备加载、错误、空、无权限、正常五类状态，并用真实数据完成主流程 | 页面可访问不等于功能可用；需重新浏览器验证当前源码 |

| P2 | 33 处 `backdrop-filter`，包括工作台和普通状态页 | 仅弹层或确有背景关系的短生命周期表面保留低强度模糊；其余使用实色语义表面 | 降低 GPU 成本，避免视觉浑浊 |

## 5. 信息架构与共享体验

### 5.1 入口层级

- 侧栏只放角色高频主入口；通知、下载任务、个人中心固定在顶栏。
- 详情、编辑、审核、向导、配置等上下文页面从列表、行操作、Tab 或详情区进入，不为凑齐功能而增加侧栏项。
- 登录后不显示角色选择；角色来自服务端会话。平台管理入口仅 SaaS 形态注册。
- 实验、仿真、竞赛答题和对抗回放进入沉浸布局，日常侧栏不与工作台同时存在。

### 5.2 页面结构

日常页面只复用结构原语，不复用一套固定业务模板：

1. PageHeader：面包屑、标题、简短语义说明、主动作和必要状态。
2. ContextBar：筛选、Tab、批量操作、日期/范围和已选摘要。
3. PageBody：表格、主从视图、详情、表单或可视化，按任务选择布局。
4. PageSection：无框分区；仅重复对象、模态、工作工具使用 Card/Panel。
5. ResourceState：在内容所在区域内显示，错误提供用户向 message、重试和可用的 trace_id，不暴露内部错误。

### 5.3 响应式规则

- 375：单列；顶栏短品牌；筛选折叠为 Drawer；表格转优先级卡片；底部主动作不得遮挡内容。
- 768：侧栏抽屉或窄栏；双列仅用于短卡片；详情 rail 默认下移。
- 1024：稳定桌面壳；允许主从布局，但保证主内容最小宽度。
- 1440：限制正文行长和表单宽度；表格/拓扑可扩展，不把文本无限拉宽。
- 固定顶栏、底栏和沉浸工具条必须计入 safe-area 与滚动内边距。

### 5.4 动效规则

- 高频导航、键盘操作、表格选择和状态筛选不做位移动画。
- Button pressed 100-160ms；Tooltip/Popover 125-180ms；Select 150-220ms；Modal/Drawer 200-260ms。
- 进入/离开使用 ease-out，屏上重排使用 ease-in-out，持续运动使用 linear；禁止 ease-in 起步。
- 只动画 transform 和 opacity；不得动画 width、height、padding、margin、left、right。
- reduced-motion 下取消位移、缩放、辉光脉冲和流程粒子，仅保留即时状态更新与当前值。

| 运行时与工具 | 链运行时、运行时详情、沙箱工具、配额管理 | 版本、自检、预拉取和状态证据清楚；配额从学校/运行时上下文进入，不新增孤立侧栏入口 |
| 引擎治理 | 判题器、仿真治理、漏洞题源 | 展示节点/队列、审核证据、来源记录和允许动作；危险操作需要确认与结果反馈 |
| 告警 | 告警中心、告警规则 | 事件与规则在同一上下文；按严重度、对象、时间和处理状态筛选，规则变更可审计 |
| 平台运维 | 系统配置、监控面板、备份记录、平台审计 | 敏感配置不回显；嵌入监控具备不可用态；备份仅展示后端支持操作；审计导出走任务 |
| 顶栏共享 | 平台公告、通知中心、任务中心、个人中心 | 共享视图统一；平台特权通过语义和权限体现，不用满屏高饱和色制造特权感 |

## 7. 后端契约约束

- 前端只调用公开 API client，不根据界面需要猜测内部接口或字段。
- 多步骤流程的服务端草稿是权威状态；无草稿契约时使用单页表单，不创建只存在浏览器中的长向导。
- 下载和导出统一进入异步任务/下载中心；不能把一次请求成功当成文件已生成。
- WebSocket 必须先获取 ticket，再按授权 topic 建连；失败显示可恢复状态，不直接暴露连接地址或原始错误。
- 学生端只能取得题面和允许公开的判题信息，不能取得答案、隐藏测试或教师配置。
- 教师仿真权限为提交、预览及教学使用；平台治理负责审核、发布与全局运行治理。
- IDE 和仿真只渲染实例能力声明中存在的文件、终端、工具和链操作；缺失时不请求接口、不建连接、不显示禁用占位。

## 8. 分阶段实施方案

### Phase 1：共享基础（全量页面前置门禁）

1. 修订前端规范中的断点实现方式、动效时长和指针能力规则。
2. 重构 PageScaffold 为无卡片默认结构；统一 PageHeader、ContextBar、ResourceState 和详情列表。
3. 修复顶栏、移动侧栏、触控目标、focus-visible、hover gate、reduced-motion 和布局属性动画。
4. 建立静态检查：禁止 `transition: all`、禁止未保护业务 hover、限制 backdrop-filter、检查裸色和非规范断点。

### Phase 2：认证与共享页（全部入口）

1. 完成认证字段错误、密码显隐、自动填充、拦截态和窄屏顶栏。
2. 重构通知、任务、个人中心；删除假进度，解析设备信息，统一用户向状态文案。
3. 用真实浏览器覆盖公共流程与四角色顶栏入口。

### Phase 3：四端全部日常页面（P0/P1/P2）

1. 学生：课程、作业、实验、仿真、竞赛、战绩、成绩、预警及全部共享入口。
2. 教师：课程、成员、讨论、作业、批改、实验编排/报告、赛事监控、漏洞题、内容、仿真、成绩、组织及全部共享入口。
3. 学校管理：账号、导入、历史、组织、看板、审核/申诉、成绩、告警、公告、认证、设置、审计及全部共享入口。
4. 平台管理：学校、入驻审核、看板、运行时/工具、引擎治理、告警、设置、监控、备份、审计及全部共享入口。
5. P2 只表示实施顺序靠后，不表示可跳过；任何已挂载路由都必须完成状态、布局、响应式、无障碍和动效收敛。

### Phase 4：沉浸工作台（P1/P2）

1. 在不重写业务引擎的前提下调整 WorkbenchShell、IdeWorkbench 和 SimulationWorkbench 的视觉层级。
2. 统一 Dock、抽屉、检查器、焦点逃逸、连接状态和窄屏叠放。
3. PBFT、Gas 回放等场景逐个验证过程、因果、当前状态和真实回放数据，降低辉光和空容器。

### Phase 5：全量验收与新高保真原型

1. 逐路由完成角色、权限、加载、错误、空、正常、长文本和大数据量矩阵。
2. 在 375/768/1024/1440 生成新截图，进行布局、像素非空和交互检查。
3. 验收通过后再建立新的高保真原型；原型从当前功能和本审计派生，不反向成为虚构后端能力的来源。

## 9.1 全量完成门禁

本轮实现不得用“共享层已改”或“代表页已改”替代逐文件审查。当前代码基线必须覆盖：

- `frontend/apps/web/src/routes/lazy-pages.ts` 的 96 个懒加载页面入口；同一实现被公开分享路由复用时按入口分别核对权限与壳层。
- `frontend/apps/web/src/features` 的全部 104 个 TSX 文件，包括页面内部操作组件、工作区组件和安全设置等深层组合。
- `apps/web`、`packages/ui`、`packages/ide`、`packages/sim-sdk` 范围内全部 79 个 CSS 文件。
- 公共布局、顶栏、侧栏、资源状态、路由兜底、IDE、WorkbenchShell、SimulationWorkbench 与七类渲染器。

完成判定同时满足：

1. 每个懒加载入口已映射到页面职责和后端契约，不存在未审查入口。
2. 每个业务 TSX 已完成用户文案、交互语义、错误恢复、内部字段暴露和可访问性扫描。
3. 每个 CSS 已完成令牌、圆角、阴影、渐变/玻璃、hover、动效、断点和溢出扫描。
4. P0/P1/P2 命中项全部修复；后端契约确实缺失的项目必须记录明确证据，不得用前端假数据填补。
5. `pnpm lint`、`pnpm type-check`、`pnpm build`、全量源码门禁和 `git diff --check` 通过。

6. 所有终端用户页面不得出现自由 JSON 编辑器、原始 JSON 转储、内部对象字段名或仅以编号代替可取得的名称；JSONB 字段必须按 `总-前端设计规范.md` §6.7 转为领域表单或可读视图。

代表页只用于验证共享组件组合是否正确，不能作为其余页面已完成的证明。
