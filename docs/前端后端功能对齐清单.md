# 前端后端功能对齐清单

> 本文用于后续前端 UI/UX、布局和动效重做时核对功能范围。设计可以重做，功能、权限、接口、状态流和错误文案不能偏离后端与总文档。
>
> 依据：`AGENTS.md`、`docs/00-平台总体蓝图.md`、`docs/总-API接口总览.md`、`docs/总-前端设计规范.md`、`docs/01-*` 到 `docs/11-*`、`frontend/packages/api-client/src/modules/*`、`frontend/packages/api-client/src/types/*`。

## 1. 全局对齐原则

### 1.1 单入口与角色落点

| 项 | 必须对齐的规则 |
| --- | --- |
| 浏览器入口 | 公共登录页和四角色功能由 `apps/web` 同一个 Web 应用承载,生产镜像只构建和托管这一份 SPA 静态产物。 |
| 角色路径 | 学生 `/student/`；教师 `/teacher/`；学校管理员 `/school-admin/`；平台管理员 `/platform-admin/`。 |
| 角色来源 | 登录成功后以前端保存的后端账号信息恢复路径；真正权限以后端会话和接口鉴权为准。前端不能让用户手动选择角色。 |
| 登录后落点 | 学生进入课程页；教师进入课程管理页；学校管理员进入账号管理页；平台管理员进入学校管理页。 |
| 私有化部署 | 私有化无平台管理员；平台级功能在私有化下按本校范围由学校管理员承接，前端不得暴露跨租户平台入口。 |

已对齐的关键项：
- 学校管理员端 `homePath` 已对齐为账号管理，符合总纲“学校管理员登录直达用户管理页”。
- 教师端不承载平台级“仿真审核”通过/驳回入口；教师保留“仿真场景/仿真包提交与预览”，审核通过/驳回归平台管理端。

### 1.2 API 与错误契约

| 项 | 前端必须遵守 |
| --- | --- |
| API 根路径 | `/api/v1`。由 `API_BASE_PATH` 和前端配置统一拼接，不在页面里硬编码散落路径。 |
| 响应信封 | 后端统一 `{ code, message, data, trace_id }`；前端只展示用户向 `message` 和必要的 `trace_id`。 |
| SDK 契约核对 | `frontend/packages/api-client` 变更后必须对照 `backend/api/paths/*.yaml`、后端公开 `dto.go` 和对应 `enum.go` 逐项核对路径、方法、`/internal` 暴露、`/api/v1` 硬编码和契约常量。 |
| 分页 | 请求 `page/size`；响应 `data: { list, total, page, size }`。列表页不得假定数组响应，除非 API 客户端方法类型就是数组。 |
| 鉴权 | `Authorization: Bearer <access_token>`；刷新走 `/auth/refresh`；未登录由 `AuthGate` 拦截。 |
| 技术错误 | 不向用户展示堆栈、字段名、SQL、error_code、pod、端口、Vite、null、undefined、timeout 等开发语义。 |
| 下载 | 导入导出、成绩单、审计导出等文件下载走 transfer 或对应模块的下载授权，不直接暴露对象存储直链。 |
| 实时 | 业务实时数据走 M10 `WS /api/ws` topic；沙箱终端、沙箱进度、判题进度、仿真 stream 走各模块自有 WS。浏览器连接前统一通过 `/api/v1/auth/ws-ticket` 为目标 WS path 换取短时 `ticket`，普通 WS URL 不携带 access token。 |
| 多步骤流程 | 入驻申请、账号导入、实验编排、作业草稿、竞赛报名/出题/参战等中间态以服务端为权威，前端本地状态只能做即时兜底。 |

### 1.3 页面入口类型

不是所有功能都应该进入侧边栏。侧边栏只承载高频、稳定、角色主任务；其余页面按任务流进入。

| 入口类型 | 放置规则 | 典型页面 |
| --- | --- | --- |
| 侧栏主入口 | 角色每天高频使用、可独立理解的一级功能。 | 学生课程、实验实训、竞赛参赛、成绩中心；教师课程管理、实验编排、题库内容、赛事组织；学校账号管理、组织架构、成绩审核；平台学校管理、入驻申请、链运行时。 |
| 深层详情页 | 依赖上级资源编号，不能脱离列表或上下文独立使用。 | 课程详情、课时学习、作业作答、作业结果、课程编辑、章节课时、成员、实验详情、竞赛详情、学校详情、运行时详情。 |
| 流程编辑页 | 是创建、配置、审核、导入等流程的一步，应该从主资源页的按钮进入。 | 作业编辑、竞赛配置、竞赛出题、组卷编辑、账号编辑、账号导入、实验编排向导、入驻审核详情。 |
| 沉浸式工作台 | 需要全屏专注和实时状态，不进入日常侧栏列表。 | 实验工作台、仿真工作台、竞赛答题、对抗回放。 |
| 顶栏/共享入口 | 跨角色通用，和当前业务无关或弱相关。 | 通知、系统公告、任务与下载、个人中心。 |
| 行内操作/抽屉 | 只针对单条记录的轻量操作，不单独做页面。 | 标记已读、启停账号、处理告警、重判、下载授权、归档、置顶。 |

### 1.5 当前实现收敛基线（2026-07-11）

- `frontend/apps/web` 的用户可达页面已覆盖 `frontend/packages/api-client/src/modules/*` 中全部公开异步能力；内部服务接口不进入浏览器 SDK。新增能力必须落到现有主入口、详情页、流程页、行操作或共享入口之一，不得仅为“接口覆盖”建立无业务入口的按钮。
- 实验与竞赛代码环境统一由应用层 `SandboxIdeWorkspace` 适配后端 M2，再复用 `@chaimir/ide` 的 `IdeWorkbench`；文件、Monaco、终端、进度 WS、命令工具、链操作和 Web 工具入口不允许在业务页面重复实现布局。M2 `SandboxResponse.capabilities` 是文件、终端、命令工具和链操作是否出现的权威来源,前端不得无条件建立连接或展示运行时未声明的动作。
- 仿真统一复用 `@chaimir/sim-sdk` 的 `SimulationWorkbench`；公开分享使用无角色守卫的 `/sim/shared/:shareCode` 深链，不进入学生侧栏。前端计算只按独立连续序列上报用户交互，不上报自动 `tick`；回放必须恢复元素 `target`。后端计算只在包声明 `compute=backend` 时连接模块 stream，交互经 WS 发给已注册适配器，服务端状态回到共享 Worker 渲染，不在页面复制 reducer、舞台、时间线或检查器。
- 全局业务事件统一使用 `eventWebSocketUrl()` 指向 `WS /api/ws`，沙箱、判题和仿真使用各模块专用 WS；所有连接统一经 `webSocketTicketProvider` 以目标 path 换取短时票据。
- 平台租户读取统一归 identity 的 `/platform/tenants`，不保留 admin 模块同义列表封装；实时事件 URL 同样不挂在 notify 业务模块下。
- 账号/组织导入、实验编排、课程草稿等多步骤流程继续以服务端预览编号、草稿或实例为权威；页面刷新后的恢复信息使用后端记录或 URL 中的资源编号，不以页面内临时状态冒充持久化。

## 2. 公共登录前页面

| 页面 | 前端路由 | 后端对齐 | 前端自有能力 |
| --- | --- | --- | --- |
| 学校用户登录 | `/auth/login` | 手机号密码 `POST /auth/login/phone`；学号/工号 `POST /auth/login/no`；短信登录 `POST /auth/login/sms`；短信发送 `POST /auth/sms/send` | 手机号是主流程，密码和短信验证码在主流程内切换；学号/工号与学校统一认证作为次级入口。后端返回 `need_select_tenant` 时只展示学校选择，不展示角色选择。 |
| 租户选择 | `/auth/tenant-select` | 由登录响应 `need_select_tenant` 与 `tenants` 驱动，二次提交带 `tenant_id` | 只选择学校，不选择学生/教师/管理员角色；没有待选登录上下文时返回登录页。 |
| 找回密码 | `/auth/forgot` | `POST /auth/sms/send`，`POST /auth/password/reset` | 验证手机号、学校数字编号、验证码、新密码；成功后引导回登录。 |
| 账号激活 | `/auth/activate` | `POST /auth/activate` | 输入激活码和新密码；后端只完成账号激活和密码设置，不签发登录会话。成功后应提示返回登录，不得假定可直接写入 token 或按角色跳转。 |
| 学校入驻申请 | `/auth/apply` | `POST /platform/applications` | 只面向学校，不是个人注册；表单应保留提交失败后的用户输入。 |
| 统一认证 | `/auth/sso` | CAS：`GET /auth/sso/{tenant_code}/login`、`GET /auth/sso/{tenant_code}/callback`；LDAP：`POST /auth/sso/{tenant_code}/ldap` | 学校短码、统一认证账号密码；CAS 跳转由后端返回 URL 驱动。 |
| 平台管理员登录 | `/auth/platform-login` | `POST /auth/login/platform` | 仅 SaaS 部署的平台管理员入口；不得与学校用户登录混成“选择角色”。 |
| 首次登录强制改密 | `/auth/change-pwd` | 登录响应 `must_change_pwd`；当前会话调用 `POST /me/password` | 登录成功后先保存受限会话并跳转改密页；改密完成后进入后端角色对应的首个功能页，改密前不能访问业务页面。 |

## 5. 前端自有能力清单

这些能力不应该由后端替前端实现，但必须和后端契约配合。

| 能力 | 前端职责 | 不能做 |
| --- | --- | --- |
| 应用壳 | 深色框架、侧栏、顶栏、移动抽屉、通知铃、头像菜单、跳过链接、焦点管理。 | 不能新增独立工作台落地页或角色选择页。 |
| 路由 | 使用 Browser History 路由,同源 path 区分四角色及其页面；Nginx 对深链接统一回退根 `index.html`。详情页和沉浸页可隐藏于侧栏。 | 不能拆分四套静态应用,也不能把所有页面都强行放进侧栏。 |
| 资源页 | 统一列表、筛选、搜索、分页、行操作、空/错/载/权限不足状态。 | 不能用空白页或技术报错替代业务状态。 |
| 表单 | 前端即时校验、提交 loading、防重复提交、首个错误聚焦、用户向错误文案。 | 不能信任前端校验替代后端校验。 |
| 多步骤流程 | 展示步骤、自动保存指示、恢复入口、冲突提示。 | 不能只存 localStorage；服务端草稿是权威。 |
| 沉浸工作台 | 实验、仿真、答题、对抗回放使用深色全屏工作台；移动端可堆叠或只读降级。 | 不能在小屏空白、横向撑破或假造进度。 |
| 可视化 | 图例、数值、表格替代、键盘可达、空/错/载态；PBFT、区块、回放必须绑定真实数据。 | 不能用装饰性雷达、假飞线、无来源动画代替真实过程。 |
| 实时 | 连接状态、重连提示、订阅 topic 展示、降级轮询或刷新入口。 | 不能把内部 WS token 暴露给工具容器或跨租户 topic。 |
| 错误处理 | `message + trace_id` 用户向展示；按 code 决定重试、跳转、权限提示。 | 不能展示 SQL、字段名、堆栈、error_code、pod、端口等技术细节。 |
| 动效 | 只表达因果关系：进入、提交、状态变化、加载、节点消息、时间轴推进；支持 reduced-motion。 | 除 loading 外不得无限装饰性动画。 |
| 响应式 | 375/768/1024/1440 四档；表格小屏转卡片或隐藏次要列；独立横滚区不能带动整页。 | 不能固定像素撑破、遮挡、整页横向滚动。 |
| 设计系统 | 青电 × 琥珀、深色框架 + 白底内容、Lucide 图标、令牌化颜色和间距。 | 禁止 emoji、裸 hex、AI 紫粉渐变、无意义装饰页。 |

## 6. 实际代码复核记录

> 本节基于 2026-07-05 对当前后端与前端代码的复核，用于防止后续 UI 重做时只按想象补页面。后续修复前端时，以下偏差项必须逐一关闭。

### 6.1 后端实际能力边界

| 模块 | 代码证据 | 已确认的功能边界 |
| --- | --- | --- |
| M1 身份认证 | `backend/internal/modules/identity/api_auth.go`、`service_auth.go`、`dto.go` | 学校登录分为手机号密码、学工号密码、短信验证码三类；手机号密码一号多校时返回 `need_select_tenant + tenants`，只让用户选学校，不选角色；学工号登录必须带 `tenant_code`；短信登录必须解析到租户；`/auth/activate` 只激活账号和设置密码，不返回有效登录态。 |
| M1 账号组织 | `backend/internal/modules/identity/api_account.go`、`api_org.go`、`api_tenant.go`、`api_platform.go` | 学校管理员维护账号、组织、租户配置和 SSO；平台管理员维护入驻申请和租户；账号导入分 preview/commit/template/batches，前端导入流程必须围绕服务端批次，不只存本地。 |

| M6 教学 | `backend/internal/modules/teaching/api.go` | 学生课程、课时、作业草稿/提交、课程评价；教师课程、章节课时、成员、作业、批改、讨论公告、课程成绩权重/计算/导出。多步骤作业和课程编辑不能只靠前端临时状态。 |
| M7 实验 | `backend/internal/modules/experiment/api.go` | 教师维护实验、校验、发布、报告评分和小组；学生创建实例、检查点判分、提交报告；实例暂停/恢复/阶段流转等是共享授权能力。 |
| M8 竞赛 | `backend/internal/modules/contest/api.go` | 教师维护赛事、赛题、状态、防作弊、漏洞源和漏洞题转化；学生报名、组队、创建答题环境、提交、对抗参战、查看回放和个人战绩。 |
| M9 管理 | `backend/internal/modules/admin/api.go` | 平台和学校看板/统计分开；配置、审计、告警事件/规则、监控面板、备份记录走管理模块。告警规则应作为告警中心内的规则维护，不作为独立侧栏主入口。 |
| M10 通知 | `backend/internal/modules/notify/api.go` | 通知、未读、偏好、公告和统一业务 WS `/api/ws` 是共享能力；沙箱/判题/仿真各自的实时流不混进通知 WS。 |
| M11 成绩 | `backend/internal/modules/grade/api.go` | 等级规则、学期、成绩审核、GPA、申诉、预警规则/扫描、成绩单生成和下载是成绩中心能力；教师报送、学校审核、学生申诉三方入口要分清。 |
| transfer | `backend/internal/platform/transfer/api.go` | 导入导出任务、任务详情和下载授权是横切能力；前端不能绕过授权直接暴露对象存储地址。 |

### 6.2 前端应用层当前状态

| 范围 | 当前状态 | 重建约束 |
| --- | --- | --- |
| 应用层 | 采用唯一的 `apps/web` 单体 SPA + FSD 架构。 | `routes/lazy-pages.ts` 集中维护路由懒加载边界；页面实现放在强映射后端 11 模块的 `features/<module>/pages/`。不得恢复旧页面工厂或独立 `src/pages` 代理层。 |
| 共享能力层 | `frontend/packages/api-client`、`ui`、`sim-sdk`、`ide` 保留。 | 新页面必须复用这些包;不能在 `apps/` 内重复实现 API SDK、设计系统、仿真引擎、Monaco/xterm 工作台。 |
| 后端对齐 | 以后端公开 API、DTO、枚举和本清单 §6.1 为准。 | 有后端能力的功能必须接真实接口;没有后端能力的页面不创建,不得用模拟数据填充。 |

### 6.3 重建时必须保留的后端语义

| 范围 | 后端语义 | 前端重建要求 |

| ID 字段类型 | 后端 `json:",string"` 字段按字符串;无 `,string` 的请求体字段按 number/number[]。 | 前端类型必须以 `packages/api-client/src/types/*` 为准,路径参数由调用处显式转字符串。 |
| M2 管理 DTO | runtime/image/tool 使用公开响应 DTO,不暴露内部模型字段名。 | 页面必须读取公开字段,不得依赖 Go 内部结构字段名。 |

### 6.5 前端结构化编辑所需的后端契约缺口

> 以下公开 DTO 只返回自由对象，没有字段描述、类型、单位、可选值、必填和敏感性契约。前端不得因此提供自由 JSON/键值编辑器；本轮只保留已明确建模的字段和可读展示，相应动态编辑入口删除。

| 缺口 | 代码证据 | 影响页面/功能 | 前端处理 |
| --- | --- | --- | --- |
| 链部署/交易 payload 无 schema | `backend/internal/modules/sandbox/dto.go` `ChainRequest.Payload` | 实验 IDE 的链部署和交易表单 | 不显示无法根据契约构造的操作；查询和命令工具继续使用已建模输入。 |
| 仿真初始参数/判分附加输入无 schema | `backend/internal/modules/experiment/model.go` `SimComponent.Params`/`CheckpointConfig.ExtraInput` | 教师实验编排 | 保留已存值，本轮不提供自由编辑入口。 |
| 漏洞链步 payload 由链运行时适配器定义 | `backend/internal/modules/contest/service_vuln.go` `VulnChainStep.Payload` | 教师漏洞题转化 | 漏洞源配置已固定为端点、方法、超时、请求体和映射字段；链步 payload 只存在于后端隔离的 M2 链适配器契约，前端不提供自由 JSON 编辑。 |

该缺口阻塞教师漏洞题转化导入流程；前端已删除导入向导、自由 payload 编辑器和对应路由，保留已存漏洞题的预验证与固化操作。

## 7. 后续前端改造验收清单

- 公共认证路由覆盖 `/auth/login`、`/auth/forgot`、`/auth/sso`、`/auth/apply`、`/auth/activate`、`/auth/tenant-select`、`/auth/change-pwd`；SaaS 部署额外开放 `/auth/platform-login`，且没有“选择角色”。
- 四端路径存在且登录后落点正确：学生课程、教师课程、学校管理员账号、平台管理员学校。
- 侧栏只放主入口；详情、编辑、导入、审核、工作台、行级操作按上下文进入。
- 每个页面的主操作能对应到本文列出的后端接口或明确为前端自有操作。
- 教师端仿真功能只做提交、预览、维护；平台端才做仿真审核通过/驳回。
- 学生端题面、作业、竞赛答题不读取答案、flag、判题配置或 M5 full 内容。
- 所有导入导出和文件下载走任务或授权，不暴露对象存储直链。
- 实验、仿真、答题、对抗回放都进入沉浸式工作台，并绑定真实实例、会话、提交或回放数据。
- 业务实时数据订阅 M10 `WS /api/ws` topic；沙箱终端、判题、仿真 stream 使用模块自有 WS，并通过短时 `ticket` 建连。
- 每个列表页具备加载、空、错误、权限不足、处理中、成功、失败状态。
- 所有用户可见错误都面向用户，并在需要报障时只展示 `trace_id`。
- 小屏无整页横向滚动、遮挡、错位；表格可转卡片或隐藏次要列。
- 可视化具备图例、数值、空/错/载态、键盘或表格替代。
- 动效支持 `prefers-reduced-motion`，除 loading 外没有无限装饰动画。
- 新增功能先确认 `docs/总-API接口总览.md` 和对应模块接口文档；没有后端契约的功能不得虚构。
