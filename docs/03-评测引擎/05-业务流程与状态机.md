# M3 评测引擎 — 业务流程与状态机

> Mermaid 描述判题流程、任务状态机、重判与人工评分流程。
> 最后更新:2026-06-08

---

## 1. 异步判题流程

```mermaid
flowchart TD
    A[调用方提交] --> B{提交限频校验}
    B -->|超频| Z[拒绝+提示]
    B -->|通过| C{tenant_id+source_ref 是否已存在}
    C -->|已存在| R[返回既有 judge_task]
    C -->|不存在| D[生成 judge_task, 存输入快照]
    D --> E[记录提交指纹]
    D --> F[入优先级队列]
    F --> G[judge worker 消费]
    G --> H[向 M2 申请干净 judge 沙箱]
    H --> I[M3 校验重打包后注入学生代码 + 判题套件]
    I --> J[执行判题器 Judge]
    J --> K[解析为可解释结果]
    K --> L[销毁 judge 沙箱]
    L --> M[回写 judge_result]
    M --> N[同事务写 judge_event_outbox]
    N --> O[发布终态事件并标记 published]
```

调用方必须为同一次业务提交传入稳定 `source_ref`。提交接口以 `(tenant_id, source_ref)` 作为幂等键:重复提交只返回既有任务,不重新写指纹、不重新入队、不重复审计。重判只能走 rejudge 流程,以保留原输入快照和审计链。

## 2. 判题任务状态机

```mermaid
stateDiagram-v2
    [*] --> queued: 提交入队
    queued --> judging: worker 领取
    queued --> cancelled: 调用方取消
    judging --> done: 判定完成(出结果)
    judging --> timeout: 超时终止
    judging --> error: 系统性失败
    timeout --> queued: 重试(未达上限)
    error --> queued: 重试(未达上限)
    timeout --> failed: 重试耗尽/策略不允许
    error --> failed: 重试耗尽/策略不允许
    done --> judging: 重判(rejudge)
    done --> [*]
    failed --> [*]
    cancelled --> [*]
```

- `done` 含通过与不通过,均为正常终态(代码错误不是 error)。
- `error`/`timeout` 指系统性问题(沙箱申请失败、判题器异常),按策略重试。
- **`failed` 为失败终态**:重试达上限或策略不允许重试时进入。进入 failed 时与状态更新同事务写入 `judge_event_outbox`,随后发布 `judge.failed` 事件(携带 task_id/source_ref/失败原因);调用方(teaching/experiment/contest)订阅后进入各自异常分支(如提交标记"判题失败可重试"、实例检查点标记异常),不再无限等待。用事件而非反向接口调用,避免 judge 反向依赖业务层(见工程目录设计 §3.1.1)。事件发布失败不得回滚已完成/已失败的判题终态,只能保留 outbox 待重试;单条 outbox 发布失败会记录 `next_attempt_at` 退避,worker 继续处理后续事件,避免 poison/stuck 记录永久阻塞。
- `judge_task.status` 枚举相应增加 `failed`。

## 3. 重判流程(申诉/题目修复)

```mermaid
flowchart LR
    A[触发重判] --> B[读取原 judge_task 输入快照]
    B --> C[按快照固定镜像/套件版本重跑]
    C --> D[追加新 judge_result version+1, is_rejudge=true]
    D --> E[保留历史结果, 标注重判]
```

- 单条重判:申诉场景。
- 批量重判:题目/判题器修复后,按 `source_ref` 全量回溯。

## 4. 人工评分流程(J6)

```mermaid
flowchart TD
    A[提交人工评分类题目] --> B[生成 judge_task, status=judging]
    B --> C[进入待人工评分列表]
    C --> D[教师查看提交内容]
    D --> E[录入分数 + 评语]
    E --> F[生成 judge_result, status=done]
```

人工评分不进 judge 沙箱、不自动执行,由教师产出结果。
人工评分接口必须校验任务 `source_owner_id`/提交者归属,普通教师不能按租户范围横向评分其他教师或其他来源任务。

## 5. 确定性保障流程

```mermaid
flowchart LR
    A[判题开始] --> B[锁定镜像版本]
    B --> C[加载固定创世状态]
    C --> D[禁用随机/真实时间/外网]
    D --> E[执行判定]
    E --> F[结果 + 输入快照入库]
    F --> G[同快照重跑结果必一致]
```
