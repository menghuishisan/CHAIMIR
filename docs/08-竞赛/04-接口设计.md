# M8 竞赛 — 接口设计

> RESTful + WebSocket(排行榜/对局状态)。鉴权沿用 M1 JWT。复用 M2/M3/M5。
> 最后更新:2026-05-29

---

## 1. 通用约定

- Base:`/api/v1/contest`
- 统一响应体同 M1。错误码段:`8xxxx`(81xxx 赛事/82xxx 报名/83xxx 解题/84xxx 对抗/85xxx 漏洞源)。

## 2. 赛事管理 `[教师/管理员]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/contests?status=` | 竞赛列表 |
| POST | `/contests` | 创建竞赛(赛制/赛程/规则) |
| PATCH | `/contests/{id}` | 编辑 |
| POST | `/contests/{id}/problems` | 编排题目(引用 M5/组卷) |
| POST | `/contests/{id}/publish` | 发布(进入报名) |
| POST | `/contests/{id}/start` / `end` | 开始/结束 |
| POST | `/contests/{id}/archive` | 归档:级联回收环境(调 M2 recycle)+ 生成成绩快照。可由结束后系统自动触发或管理员手动 |

## 4. 解题赛 `[学生]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/contests/{id}/problems` | 题目列表(题面取 M5,过滤答案) |
| POST | `/contests/{id}/problems/{pid}/env` | 起实操题环境(调 M2) |
| POST | `/contests/{id}/problems/{pid}/submit` | 提交(flag/代码/理论答案)→ 调 M3 判 |
| GET | `/submissions/{id}` | 查看判定结果 |

> 提交流程:M8 收提交 → 调 M3(带 M5 题目引用 + 沙箱状态)→ 回写 → 更新排行。
> M8 题目列表必须经 M5 题面接口返回过滤后的内容;M5 题面契约不可用时返回 M8 专用错误码,不能返回无题面或未过滤数据。实操题环境创建必须经 M2 契约,解题提交必须经 M3 契约,缺必需契约时显式失败。

## 5. 对抗赛 `[学生]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/contests/{id}/battle/entry` | 提交参战物(防御合约/攻击脚本/策略合约) |
| GET | `/contests/{id}/battle/entries` | 我的参战历史(各版本) |
| GET | `/contests/{id}/battle/matches?page=&size=` | 我的对局分页列表,响应 data 为 `{list,total,page,size}` |
| GET | `/matches/{id}/replay` | 对局回放数据(交易序列) |
| GET | `/contests/{id}/ladder` | 天梯排名 |

> 提交参战物 → 进对战池 → 撮合调度器异步起对局(M2)→ 判定 → 按标准 ELO 公式更新天梯 → 通知。ELO 初始分与 K 因子统一来自 `CONTEST_BATTLE_ELO_INITIAL_SCORE`、`CONTEST_BATTLE_ELO_K_FACTOR`,不得在代码或 SQL 中写死。参战物提交必须同时提供 `artifact_ref` 和内容 SHA256 `code_hash`;M8 持久化该 hash 用于复现、审计和查重。对局判题先通过 M2 `RestoreSandboxArchive(expected_hash=code_hash)` 校验并恢复双方参战物到对局沙箱 `battle/entry_a`、`battle/entry_b`,再用 M3 reuse 模式读取现场沙箱和 M5 锁定判题配置;M8 不把参战物伪装成 M3 代码提交字段。

## 6. 排行榜(实时)

- 经 M10 统一通道:M8 判题/对局后调 M10 `POST /notify/push(tenant_id, topic=tenant:{tenant_id}:contest:{id}:leaderboard)`;客户端订阅 M10 `WS /api/ws` 的该 topic。
- 封榜后停止推送实时变化(显示冻结榜)。

## 6.5 个人竞赛战绩档案 `[学生/教师]`

> B11 修复:竞赛成绩不计 GPA,但需有落点供学生查看。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/my/contest-records` `[学生]` | 我的参赛历史(竞赛/名次/得分/天梯ELO/徽章) |
| GET | `/contests/{id}/result-snapshot` | 竞赛结束生成的成绩快照(最终榜单存档) |
| GET | `/students/{id}/contest-achievements` `[内部]` | 供 M11 独立聚合"竞赛成就"(与 GPA 分离,只读) |

> 竞赛结束(已结束→已归档)时生成 `result-snapshot` 成绩快照存档,避免战绩只活在 ladder_rank/battle_match 流水里。学生竞赛战绩独立展示,不混入学业 GPA。

## 8.5 内部聚合 `[内部]`(B2 修复)

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/internal/stats?tenant_id=` | 供 M9 看板聚合:竞赛数、参赛人数、活跃竞赛等只读统计 |
| GET | `/students/{id}/contest-achievements` `[内部]` | 供 M11 聚合竞赛成就(与 GPA 分离) |

> 漏洞题最终落 M5;运行时/答题零外部依赖。
> 异步判题事件订阅失败、事件 payload 无效、source_ref 未匹配分别使用独立错误码,便于定位是订阅链路、事件格式还是提交绑定问题。
> 内部聚合只保留 `/students/{id}/contest-achievements` 这一条稳定路径,不得新增旧式别名或旁路路由。

## 9. 典型时序

### 解题赛提交
```
学生 → 起环境(M2) → 提交 → M8 调 M3 判(取 M5 配置, judge 沙箱)
M3 → 回写结果 → M8 更新 ladder_rank → WS 推排行榜
```

### 对抗赛对局
```
学生 → 提交参战物 → 进对战池
撮合调度器 → 选对手 → 起对局沙箱(M2) → 跑对局 → M3 判定
→ 记 battle_match(含 replay_ref) → 更新 ELO → WS 推天梯
学生 → 看对局回放(交易序列重放)
```
