# M5 题库与模板中心 — 数据模型

> PostgreSQL。内容外壳与内容体为租户级(tenant_id + RLS);共享库内容跨租户只读可克隆。
> 建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 | 范围 |
| --- | --- | --- |
| `content_item` | 内容外壳(统一元信息 + 版本) | 租户 |
| `content_body` | 类型化内容体(实验/竞赛/理论,JSONB) | 租户 |
| `content_category` | 分类树 | 租户 |
| `paper` | 组卷(题目引用集合) | 租户 |
| `paper_item` | 卷与题目关联 + 分值 | 租户 |

> 消灭冗余/合并说明:
> - **版本不单独建表**:`content_item` 以 `(code, version)` 联合唯一表达多版本,同 code 多行即多版本。
> - **内容体与外壳一对一拆表**:外壳查列表频繁、字段小;内容体是大 JSONB(含答案/判题配置),拆开避免列表查询拉大字段,也便于权限层只在取 full 时读内容体。
> - **标签/知识点不单建关联表**:用 `content_item.tags TEXT[]` / `knowledge_points TEXT[]`(PG 数组 + GIN 索引),避免 N×M 关联表冗余;分类是树形强结构故独立成表。

## 3. 多租户隔离

- 全部表带 `tenant_id` + RLS(同 M1)。
- `content_item.visibility=3(共享库)`:跨租户只读可克隆;数据库 RLS 对共享内容只放开 SELECT,写入/更新/删除仍只能命中当前租户;克隆时复制到目标租户生成新 code,与源解耦。
- 私有化:`tenant_id` 恒定;共享库内容可随离线包分发。

## 4. 关键索引

- `content_item(tenant_id, code, version)` UNIQUE
- `content_item(tenant_id, type, status)`、`content_item(visibility)`(共享库检索)
- `content_item USING GIN(tags)`、`GIN(knowledge_points)`
- `paper_item(tenant_id, paper_id, seq)`
