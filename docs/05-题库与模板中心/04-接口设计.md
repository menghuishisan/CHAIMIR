# M5 题库与模板中心 — 接口设计

> RESTful。鉴权沿用 M1 JWT。取题面与取答案接口分离;判题取全量为内部接口。
> 最后更新:2026-05-29

---

## 1. 通用约定

- Base:`/api/v1/content`
- 统一响应体同 M1。错误码段:`5xxxx`(51xxx 内容/52xxx 版本/53xxx 共享/54xxx 组卷)。
- `[内部]` 供 M2/M3/M4/M6/M7/M8 调用。

---

## 2. 内容管理 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/items?type=&category=&difficulty=&tag=&kp=&keyword=&visibility=&author=` | 内容检索(分页) |
| POST | `/items` | 创建内容(类型 + 元信息 + 内容体,草稿) |
| GET | `/items/{code}/{version}` | 取内容**题面**(过滤敏感字段) |
| GET | `/items/{code}/{version}/full` `[内部/作者]` | 取全量(含答案/判题配置) |
| PATCH | `/items/{id}` | 编辑草稿 |
| POST | `/items/{id}/publish` | 发布(定版) |
| POST | `/items/{id}/deprecate` | 弃用 |
| DELETE | `/items/{id}` | 软删(无引用时) |
| POST | `/items/system-import` `[内部]` | 系统/外部源自动建题入口(B10):M8 真实漏洞转化固化后调此入库,只允许 `author_type=系统/外部源`,带固化内容、预验证结果 |

> **系统建题入口(B10 修复)**:除教师手动创建外,M5 支持"系统/外部源自动建题"——M8 真实漏洞外部源经分级+预验证后,调 `system-import` 把固化的自包含题目写入 M5(`content_item.author_type` 标记来源)。该内部入口不得接受请求体伪装成教师来源,未传 `author_type` 时按系统来源处理,显式来源只能是系统或外部源。跨模块 contracts 调用必须保证请求 `tenant_id` 与服务端鉴权上下文租户一致,不得用请求体覆盖租户边界。入库的漏洞题已通过预验证,默认进草稿待人工审核或按来源级别自动发布。

## 3. 版本 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/items/{code}/versions` | 版本列表 |
| POST | `/items/{code}/new-version` | 基于现有版本创建新版本草稿 |

> 业务引用锁定 `(code, version)`;弃用版本不可被新引用,旧引用仍有效。

---

## 4. 复用 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/items/{code}/{version}/clone` | 克隆为独立草稿(新 code,不带引用/统计) |
| POST | `/items/{id}/share` | 设为共享库(visibility=shared) |
| POST | `/items/{id}/unshare` | 取消共享(仅停止被新克隆;已克隆副本独立不受影响) |
| GET | `/shared?type=&keyword=` | 浏览跨校共享库 |

> **共享撤回语义(B13 修复)**:克隆是**深拷贝**——他校克隆瞬间即生成独立新 code 的副本,与源完全解耦。因此:① 取消共享(unshare)仅使该内容**不再出现在共享库、不可被新克隆**,已克隆的副本不受任何影响;② 源内容发新版本,已克隆副本也不自动更新(克隆即独立)。与"版本化:旧引用锁版本不受影响"不冲突——版本锁针对的是**同租户内引用**,克隆针对的是**跨租户复制**,二者是不同机制。

## 5. 分类与组卷 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET/POST/PATCH/DELETE | `/categories[/{id}]` | 分类树维护 |
| GET | `/papers` | 卷列表 |
| POST | `/papers` | 创建卷(手动选题 或 按条件随机抽题) |
| GET | `/papers/{id}` | 卷详情(题目引用 + 分值,题面视角) |
| POST | `/papers/{id}/regenerate` | 按条件重新抽题 |

随机抽题请求示例:
```json
{ "name": "第3章测验", "gen_mode": 2,
  "gen_criteria": { "knowledge_points": ["共识","哈希"], "difficulty": [2,3], "count": 10 } }
```

## 6. 内部取用 `[内部]`

供引擎/业务按引用取内容:

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/items/{code}/{version}/full` | M3 判题取判题配置 + 答案;M7 取实验模板配置;M2/M4 取运行时/仿真引用 |
| POST | `/items/batch` | 批量取(组卷展开) |
| POST | `/items/{code}/{version}/usage` | 引用计数 +1(被引用时上报) |

> `full` 接口强制内部服务鉴权 + 角色校验,绝不对学生开放。

---

## 7. 典型时序(实验引用模板)

```
M7 创建实验 → GET /items/{code}/{version}/full(取实验模板配置)
M7 → 把 runtime_code/tools 交 M2 起沙箱
M7 → 把 sim_package_ref 交 M4 跑仿真
学生提交 → M3 → GET /items/{code}/{version}/full(取判题配置+答案)→ 判题
被引用 → POST /usage 计数
```
