# M5 题库与模板中心 — 需求规格

> 模块定位:内容枢纽。统一存放和复用"可被业务引用的内容资产",被 M2/M3/M4 引擎与 M6/M7/M8 业务引用。
> 核心理念:统一外壳 + 类型化内容体;内容创作一次,多处引用,版本复用。
> 最后更新:2026-05-29

---

## 1. 模块目标

- **统一管理**三类内容:实验模板、竞赛题、理论题。
- **一处创作、多处引用**:内容被课时/作业/竞赛引用,引用锁定版本。
- **复用**:克隆 + 跨校共享库。
- **版本化**:改内容发新版,不影响已引用的旧业务。
- **可检索可组卷**:分类/标签/难度/知识点,支持按条件抽题。

## 2. 三类内容

| 类型 | 内容体要点 | 被谁引用 |
| --- | --- | --- |
| **实验模板** | M2 运行时+工具集、初始代码、M4 仿真包引用、M3 判题配置、教学说明、个性化部署脚本 | M7 实验 |
| **竞赛题** | 题面、判题配置(测试用例/链上断言/flag)、初始合约、对抗赛配置 | M8 竞赛 |
| **理论题** | 题面、题型(选择/判断/填空/简答)、选项、正确答案、解析 | M6 作业、M8 理论赛 |

**共同点**:都共享元信息(标题/分类/难度/标签/作者/版本/可见范围/复用统计),都可克隆、共享、版本化。

## 3. 核心架构:统一外壳 + 类型化内容体

- **统一外壳**:所有内容共享一套元信息表,检索/克隆/版本/权限只实现一套。
- **类型化内容体**:三类各自专有结构以 `类型 + JSONB` 承载,互不污染。
- 呼应"消灭冗余、合理合表":共性提取一处,差异各自承载。

## 4. 关键边界:答案存 M5

- M5 存题目**全部内容**,含正确答案 / 判题规则 / 测试用例 / flag。
- M3 判题时按"题目引用"来 M5 取判题配置与答案;**M3 自身不存任何题目数据**。
- 理由:答案与题面一体,克隆/版本化须一起走;M3 是"怎么判"的算法,M5 是"判什么"的数据。
- 安全:答案字段权限隔离(学生不可取),M3 在隔离 judge 沙箱取用不下发。

## 5. 职责边界

| M5 负责 | M5 不负责(归属) |
| --- | --- |
| 内容存储、检索、组卷 | 判题执行(→ M3,来取配置) |
| 克隆、共享库、版本管理 | 运行环境(→ M2) |
| 答案/判题配置存储 | 仿真运行(→ M4,引用仿真包) |
| 内容权限(答案隔离) | 成绩(→ M6) |
| 元信息与复用统计 | 课程/班级编排(→ M6) |

## 6. 功能需求

### A. 内容管理
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 创建内容 | 选类型(实验模板/竞赛题/理论题),填元信息 + 类型化内容体 |
| A2 | 编辑/草稿 | 草稿态自由编辑 |
| A3 | 发布 | 草稿 → 发布,可被引用 |
| A4 | 删除 | 软删;有引用时禁删或仅停用 |

### B. 版本
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 版本化 | semver;发布即定版 |
| B2 | 引用锁版本 | 业务引用时锁定版本号 |
| B3 | 发新版 | 改内容发新版本,旧引用不受影响 |

### C. 复用
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 克隆 | 复制为独立草稿,不带学生数据/提交 |
| C2 | 共享库 | 设为共享,跨校可见可克隆 |
| C3 | 取消共享 | 随时撤回 |

### D. 检索与组卷
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 分类树 | 内容分类层级 |
| D2 | 标签/难度/知识点 | 多维筛选 |
| D3 | 组卷 | 按条件(知识点/难度/数量)随机或手动抽题成卷 |

### E. 权限
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 答案隔离 | 取题面接口与取答案接口分离,学生取不到答案 |
| E2 | 归属 | 教师管自己内容 |
| E3 | 共享可见 | 共享库内容全平台可见可克隆 |

## 7. 非功能需求

- **一致性**:答案随题目克隆/版本化一起走,杜绝跨模块不一致。
- **可扩展**:内容体 JSONB 容纳未来新类型/新字段,不改外壳。
- **多租户**:内容带 tenant_id 隔离;共享库为跨租户只读可克隆。
- **版本不可变**:已发布版本内容不可改,改则发新版。

## 8. 明确不做(边界)

- 不判题(M3 来取配置)、不提供环境(M2)、不跑仿真(M4)、不管成绩(M6)。
- 只存内容、供检索引用、管版本复用。

## 9. 技术取向

Go 后端 + PostgreSQL(JSONB 承载类型化内容体);大文本/附件存对象存储(MinIO)。最终技术细节由《技术选型》统一收口。
