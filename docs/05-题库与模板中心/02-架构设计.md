## 1. 总体架构

```
┌──────────────────────────────────────────────────────────┐

```

## 2. 统一外壳 + 类型化内容体(核心)

### 2.1 内容外壳(共性元信息,一套)

```
ContentItem {
  id, tenant_id, type,           // type: experiment_template | contest_problem | theory_question
  code, version,                 // 版本化标识
  title, category_id, difficulty,
  tags[], knowledge_points[],
  author_id, visibility,         // private | tenant | shared(共享库)
  status,                        // draft | published | deprecated
  body_ref,                      // 指向类型化内容体
  usage_count,                   // 复用统计
  created_at, updated_at
}
```

### 2.2 类型化内容体(差异,各承载)

`body` 以 JSONB 按类型存专有结构:

| type | body 结构要点 |
| --- | --- |
| 实验模板 | `runtime_code`(M2运行时)+`tools[]`+`init_code_ref`+`sim_package_ref`(M4)+`judge_config`(M3)+`description`+`init_script` |
| 竞赛题 | `statement`+`judge_config`(测试用例/断言/flag)+`init_contracts`+`ad_config`(对抗赛) |
| 理论题 | `statement`+`q_type`(选择/判断/填空/简答)+`options[]`+`answer`+`explanation` |

> 检索/克隆/版本/权限只对外壳实现一套;内容体只在创作与取用时按类型解析。这是"消灭冗余、合理合表"的落地。

### 2.3 敏感字段标注

内容体中 `answer` / `judge_config.testcases` / `flag` 等标记为**敏感字段**,经权限层过滤(见 §6)。

## 3. 版本化

```
内容 = (code, version)
发布即定版(版本内容不可变)
业务引用时锁定具体 version
改内容 → 发新 version → 旧引用仍指向旧版本
```

- 与 M2 运行时、M4 仿真包同一版本哲学,全平台一致。
- 已被引用的版本不可删,只能弃用(deprecated),保证旧实验/旧竞赛稳定可复现。

---

## 4. 克隆与共享

### 4.1 克隆

```
克隆源(自己的/共享库的)→ 复制外壳 + 内容体 → 生成新草稿(新 code)
不带:引用关系、学生数据、提交记录、复用统计(归零)
```

### 4.2 共享库

- `visibility=shared`:进入跨校共享库,全平台可见可克隆。
- 共享为**只读 + 可克隆**,他人克隆后成自己的独立内容,与原内容无关联。
- 可随时取消共享(已克隆的副本不受影响)。

---

## 5. 检索与组卷

- **分类树**:多级内容分类。
- **多维筛选**:标签 / 难度 / 知识点 / 类型 / 作者 / 可见范围。
- **组卷**:按条件(知识点+难度+数量)随机抽题,或手动选题,生成"卷"(题目引用集合 + 各题分值)。组卷结果被 M6 作业 / M8 竞赛引用。

---

## 6. 权限层(答案隔离)

```
取内容接口分两类:
  GET .../content      → 题面 + 非敏感内容(学生可用,过滤掉 answer/testcases/flag)
  GET .../content/full → 全量含敏感字段(仅教师/M3 内部判题鉴权可用)
```

- 学生侧任何路径都拿不到敏感字段。
- M3 判题经内部鉴权取 full,在隔离 judge 沙箱使用,不下发学生。

---

## 7. 与各模块供给关系

| 模块 | 取用 |
| --- | --- |
| M7 实验 | 取实验模板:运行时/工具/初始代码/仿真引用/判题配置 |
| M8 竞赛 | 取竞赛题 + 组卷 |
| M6 作业/理论赛 | 取理论题 + 组卷 |
| M2 | 实验模板里的 runtime_code/tools 交 M2 起沙箱 |
| M3 | 按题目引用取 judge_config + 答案判题 |
| M4 | 实验模板里的 sim_package_ref 交 M4 跑仿真 |

> M5 只提供内容数据,不执行;执行由各引擎负责。

---

## 8. 多租户

- 内容外壳带 `tenant_id` + RLS;`visibility=shared` 时跨租户只读可克隆。
- 私有化:`tenant_id` 恒定;共享库内容可随离线包分发。
