# M1 身份与租户 — 业务流程与状态机

> 用 Mermaid 描述关键流程与状态流转。
> 最后更新:2026-05-29

---

## 1. 学校入驻流程(SaaS)

```mermaid
sequenceDiagram
    participant V as 访客(学校联系人)
    participant P as 平台管理员
    participant S as 系统
    V->>S: 提交入驻申请(学校信息+联系人)
    S-->>V: 申请已受理(状态=申请中)
    P->>S: 查看申请列表
    alt 通过
        P->>S: 审核通过
        S->>S: 创建租户 + 分配短码 + 建学校管理员账号
        S-->>P: 生成激活码
        S-->>V: 通知联系人(激活码/链接)
    else 驳回
        P->>S: 驳回(填原因)
        S-->>V: 通知驳回原因
    end
```

私有化部署:无此流程,初始化脚本或 `chaimir/migrate migrate-and-seed` 直接创建租户与顶层学校管理员。
脚本只做编排,实际建租户、建账号、授予教师/学校管理员角色和写审计均调用 M1 bootstrap 服务,
不在 shell 或原始 SQL 中复制账号开通规则。

首个学校管理员说明:此时学校真实院系/专业/班级尚未录入,系统只创建账号与教师、学校管理员角色,
不自动创建任何默认院系,也不写入虚假的组织档案。该管理员登录后先维护真实组织架构,再补齐自己的
工号、院系、职称等 `account_profile` 信息;后续创建/导入师生必须挂靠真实组织。

## 2. 账号导入流程(两步:预览 + 提交)

```mermaid
flowchart TD
    A[管理员上传 Excel/CSV] --> B[系统逐行校验]
    B --> C{全部通过?}
    C -->|是| D[预览:显示将导入数据]
    C -->|否| E[预览:标红失败行+原因]
    D --> P[服务端暂存 preview_id]
    E --> P
    P --> F[管理员确认提交 preview_id]
    F --> G[读取服务端预览行,仅写入校验通过的行]
    G --> H[生成 import_batch 记录]
    H --> I[新账号状态=待激活]
```

校验项:必填项、手机号格式、学号/工号租户内去重、班级/院系存在性。预览结果写入 `import_preview`,
提交只传 `preview_id`,不得由前端回传 rows 作为权威数据。

## 3. 账号开通与首次登录

```mermaid
flowchart TD
    A[账号创建,状态=待激活] --> B{开通方式}
    B -->|初始密码| C[管理员发放初始密码]
    B -->|激活码| D[发放激活码,用户自设密码]
    B -->|SSO| E[名单就绪,等待SSO首登]
    C --> F[首次登录]
    F --> G{must_change_pwd?}
    G -->|是| H[强制修改密码]
    H --> I[状态=正常]
    D --> I
    E --> J[SSO回调匹配名单] --> I
```

SSO 首登细则:
- CAS 走浏览器跳转:服务端生成登录地址,回调时校验 `ticket` 与 `service`,取得学校身份标识后只匹配已导入账号。
- LDAP 走账号密码直连:服务端使用学校 LDAP 服务绑定并查询用户条目,取得 `match_attribute` 后只匹配已导入账号。
- SSO 不负责自动建账号或补组织档案;未导入名单返回"账号未在名单中",由学校管理员先完成导入/维护。

## 4. 登录与双 Token 无感刷新(单端)

```mermaid
sequenceDiagram
    participant C as 客户端
    participant S as 服务端
    C->>S: 登录(凭证)
    S->>S: 校验 + 吊销该账号其余会话(单端)
    S-->>C: Access(15min) + Refresh(7d)
    Note over C,S: 正常请求携带 Access
    C->>S: 请求(Access 已过期)
    S-->>C: 401 / code=11001
    C->>S: POST /auth/refresh(Refresh)
    alt Refresh 有效
        S->>S: 轮转:签发新 Access+Refresh,旧 Refresh 失效
        S-->>C: 新 Token 对
    else Refresh 失效/被踢
        S-->>C: code=11001 要求重新登录
    end
```

## 5. 账号状态机

```mermaid
stateDiagram-v2
    [*] --> 待激活: 导入/新增
    待激活 --> 正常: 激活/首登改密
    正常 --> 停用: 管理员停用
    停用 --> 正常: 管理员启用
    正常 --> 归档: 学年归档(毕业)
    归档 --> 正常: 管理员恢复
    正常 --> 注销: 管理员注销
    停用 --> 注销: 管理员注销
    归档 --> 注销: 清理
    注销 --> [*]
```

- 归档:数据保留,不可登录,可恢复(毕业生)。
- 注销:终态,不可登录;数据按合规保留,不物理删除(软删 `deleted_at`)。

## 6. 租户状态机

```mermaid
stateDiagram-v2
    [*] --> 申请中: 提交申请(SaaS)
    申请中 --> 正常: 审核通过(建租户)
    申请中 --> [*]: 驳回
    正常 --> 停用: 平台管理员停用
    停用 --> 正常: 恢复
    正常 --> 到期: 超过 expire_at
    到期 --> 正常: 续期
```

- 停用/到期:全校无法登录;私有化部署无此状态流转(恒正常)。

## 7. 班级归档与升级

```mermaid
flowchart LR
    A[选择学年/班级] --> B{操作}
    B -->|归档| C[班级status=已归档<br/>学生account归档]
    B -->|升级| D[班级enrollment_year/名称调整<br/>学生随班级上移]
```

归档与升级为批量操作,全程记审计日志。
