# M1 身份与租户 — 需求规格

> 模块定位:平台地基。提供多租户、组织架构、账号、认证、授权、安全合规能力,是所有其他模块的前置依赖。
> 最后更新:2026-05-29

---

## 1. 模块目标

- 支撑 **SaaS 多租户** 与 **私有化部署** 两种形态,同一套代码。
- 以学校为租户,通过 `tenant_id` 实现数据隔离。
- 禁止自助注册:学校申请审核制,师生由学校管理员导入。
- 提供安全、合规的身份认证与基于角色的访问控制。

## 2. 角色定义

| 角色 | 范围 | 说明 |
| --- | --- | --- |
| 平台管理员 | 全局(仅 SaaS) | 审核学校入驻,管理租户。不属于任何租户;私有化无此角色 |
| 学校管理员 | 租户内最高 | 是被授予管理权限的**教师**,非独立身份。导入/管理师生、组织架构、租户配置 |
| 教师 | 租户内 | 开课、布置实验、组织竞赛、批改 |
| 学生 | 租户内 | 学习、做实验、参赛 |

**身份模型铁律:**
- 一个自然人 = 一个账号(手机号唯一)。
- 租户内基础身份为**学生**或**教师**,二者互斥,入职即定。
- 学校管理员 = 教师 + 管理权限标记,可授予/撤销,一个账号两套权限并存,无需切换。
- 平台管理员为独立体系,不属于任何租户,与校内身份正交。

## 3. 功能需求

### A. 租户管理(学校)

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 学校入驻申请(SaaS) | 申请表:学校名/类型/联系人/联系方式 → 平台管理员审核 → 通过即创建租户并开通学校管理员账号(激活码方式) |
| A2 | 学校短码 | 全局唯一(如 `pku`),用于登录入口区分与品牌化 |
| A3 | 租户配置 | logo、名称、功能开关、认证方式(本地/SSO)、是否启用激活码 |
| A4 | 租户状态 | 申请中 / 正常 / 停用 / 到期;停用后全校无法登录 |
| A5 | 私有化初始化 | 初始化脚本直建顶层学校管理员,无 A1 申请审核环节 |

### B. 组织架构

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 三级树形 | 院系 → 专业 → 班级 |
| B2 | 人员挂靠 | 教师挂院系,学生挂班级 |
| B3 | 学年关联 | 班级关联入学年份,支持按学年归档/升级 |
| B4 | 增删改查 | 组织结构维护 + 批量导入 |

### C. 账号与导入

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 批量导入 | Excel/CSV 导入教师/学生,提供模板下载 |
| C2 | 导入校验 | 格式校验、手机号格式、学号去重、必填项校验、**导入前预览 + 部分失败逐行报错** |
| C3 | 开通方式 | ① 初始密码 + 首登强制改密;② 激活码自设密码;③ SSO 对接(名单先导入,SSO 仅核验) |
| C4 | 单账号维护 | 手动新增、编辑、停用 |
| C5 | 账号状态机 | 待激活 / 正常 / 停用 / 归档 / 注销 |
| C6 | 批量操作 | 批量停用、按学年归档、班级升级 |

### D. 认证与会话

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 主登录 | 手机号 + 密码 |
| D2 | 备用登录 | 学校短码 + 学号 + 密码 |
| D3 | 验证码 | 短信验证码登录 / 找回密码 |
| D4 | 双 Token | JWT Access(短期)+ Refresh(长期),无感刷新;Refresh 轮转 + 可吊销 |
| D5 | **单端登录** | 强制单端,新登录踢掉旧会话(旧 Refresh 立即吊销) |
| D6 | SSO 认证 | 落地可插拔认证适配器框架 + **CAS、LDAP 两种实现**(国内高校最主流);名单先导入,SSO 仅核验身份,匹配名单才放行 |
| D7 | 登出/下线 | 主动登出、管理员强制下线 |

### E. 授权(权限)

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | RBAC | 平台管理员 / 学校管理员 / 教师 / 学生,四级固定角色 |
| E2 | 数据范围 | 权限 = 角色 × scope:教师只管自己班级/课程,学生只看自己 |
| E3 | 接口+数据双鉴权 | 接口级权限校验 + 框架层强制 `tenant_id` 过滤 |

> 自定义角色:已明确不做。

### F. 安全与合规

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 密码安全 | argon2id 加盐哈希、强度校验、首登强制改密 |
| F2 | 防爆破 | 登录失败锁定、验证码防刷 |
| F3 | PII 加密 | 手机号等敏感信息加密存储 |
| F4 | 审计日志 | 谁 / 何时 / 对谁 / 做了什么 |
| F5 | 归档非删除 | 学生毕业归档,数据保留 |
| F6 | 越权防护 | 横向(跨租户/跨班级)+ 纵向(提权)防护 |

### G. 个人中心

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| G1 | 账户维护 | 改密、改手机号(验证码验证)、查看个人信息 |
| G2 | 学籍只读 | 学号/院系/班级由管理员维护,本人不可改 |

## 4. 非功能需求

- **多租户隔离**:任何业务查询强制带 `tenant_id`,框架层拦截,杜绝越权。
- **私有化同构**:单租户部署,`tenant_id` 恒定,逻辑零差异,去掉平台管理员层。
- **面向区域**:国内高校,手机号按国内格式校验,短信走国内服务商,中文界面。
- **可扩展**:认证方式(SSO/LDAP)做成可插拔适配器,呼应平台插件化原则。

## 5. 明确不做(边界)

- 自定义角色编辑器(已明确四级固定 RBAC)

## 6. 技术栈

按平台约定:**各模块按特性单独选型,不强求单一语言**。M1 倾向后端 Go、前端 React+TS、数据库 PostgreSQL + Redis,最终在所有模块聊透后由《技术选型》文档统一收口。
