# M1 身份与租户 — 安全设计

> 教育平台合规要求高。本文覆盖密码、令牌、PII、防爆破、越权、审计、租户隔离。
> 最后更新:2026-05-29

---

## 1. 密码安全

- 哈希算法:**argon2id**(salt 唯一,内存/迭代参数按部署环境调优);不使用 MD5/SHA1。
- 强度策略:最少 8 位,含字母+数字,禁弱口令字典;管理员可配置租户级策略。
- 首次登录:`must_change_pwd=true` 时强制改密后方可使用。
- 管理员重置:生成临时密码并置 `must_change_pwd`,不可见用户原密码(单向哈希)。
- 传输:全站 HTTPS,密码仅明文存在于传输层,落库即哈希。

---

## 2. 令牌与会话(双 Token)

| 项 | Access Token | Refresh Token |
| --- | --- | --- |
| 形式 | JWT(签名) | 不透明随机串 |
| 有效期 | 15 分钟 | 7 天 |
| 存储 | 客户端内存/安全存储 | 哈希存 `auth_session`/`platform_auth_session`(不存明文) |
| 载荷 | account_id/tenant_id/roles/base_identity | 无(仅作凭证) |

**关键策略:**
- **轮转(rotation)**:每次 `/auth/refresh` 签发新 Refresh 并吊销旧记录;旧 Refresh 再次使用 → 判定为重放,吊销该账号全部会话并告警。
- **单端登录**:登录时吊销该账号其余 `status=有效` 的会话。
- **吊销**:登出、改密、被踢、管理员强制下线均置会话失效。
- **签名**:JWT 用非对称(RS256)或对称(HS256)密钥,密钥经环境注入,不入库不入仓。
- tenant_id 写入 JWT 并签名,客户端不可篡改以越权访问他校数据。

---

## 3. PII 与敏感数据

- 手机号:`phone_enc`(对称加密,AES-GCM,密钥经 KMS/环境注入)+ `phone_hash`(HMAC,用于唯一约束与登录查询)。
- 日志脱敏:手机号在日志/响应中掩码(`138****1234`)。
- SSO/LDAP 配置中的 bind 密码等加密存储。
- CAS 登录与回调的 `service` 参数只能使用部署白名单 `IDENTITY_SSO_ALLOWED_SERVICE_ORIGINS` 中的 origin;后端不得接受任意第三方回调域,避免 CAS 票据被引导到非平台域名后换取本系统 Token。
- 验证码:仅存哈希;短信内容不落明文日志。

---

## 4. 防爆破与限频

| 场景 | 策略 |
| --- | --- |
| 密码登录失败 | 连续失败计数,达阈值锁定账号一段时间(`locked_until`) |
| 短信验证码 | 同号 60s 一条、单日上限;Redis 计数 |
| 验证码校验 | 限尝试次数,过期失效,一次性使用 |
| 接口限流 | 网关层按 IP / 账号限流,防撞库 |

---

## 5. 越权防护

- **租户隔离(横向)**:PostgreSQL RLS 强制 `tenant_id = current_setting('app.tenant_id')`;应用每请求按 JWT 设置会话变量。
- **范围隔离(横向)**:教师/学生数据查询叠加 scope;学生强制命中本人 account_id。
- **提权防护(纵向)**:角色来自服务端会话,不接受客户端传参;敏感操作(授予管理员、重置密码)二次校验角色并记审计。
- **私有化**:禁用 `/platform/*`,无平台管理员越权面。

---

## 6. 审计日志

- 记录:操作人、角色、动作码、对象、变更详情(diff)、IP、时间。
- 覆盖:登录/登出、账号增删改、权限授予、导入、租户审核、配置变更。
- 审计日志只追加不可改;查询受权限控制(本校管理员看本校,平台管理员看平台级)。

---

## 7. 数据生命周期与合规

- 归档而非删除:学生毕业归档,数据保留可恢复。
- 注销:软删(`deleted_at`),按合规保留期后再清理。
- 备份与最小权限:数据库账号最小权限;平台级连接与租户连接分离。

---

## 8. 安全测试要点(供实现阶段验证)

- 跨租户访问尝试必须被 RLS 拦截(渗透用例)。
- Refresh Token 重放、并发刷新竞争。
- 越权调用管理接口(普通教师/学生调 `/accounts/*`)。
- 短信/登录接口压测限频有效性。
- JWT 篡改 tenant_id/roles 必须签名校验失败。
