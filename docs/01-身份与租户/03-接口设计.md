# M1 身份与租户 — 接口设计

> 风格:RESTful,JSON。鉴权:JWT 双 Token。所有租户内接口由网关注入 `tenant_id`,业务层不可越权。
> 最后更新:2026-05-29

---

## 1. 通用约定

**Base URL**:`/api/v1`

**请求头:**
- `Authorization: Bearer <access_token>`(需鉴权的接口)
- `X-Tenant-Code: <学校短码>`(登录前用于定位租户,如 CAS 跳转、备用登录)

**统一响应体:**

```json
{ "code": 0, "message": "ok", "data": {} }
```

- `code`:0 成功,非 0 为业务错误码。
- 分页:请求 `?page=1&size=20`,响应 `data: { list, total, page, size }`。

**错误码段(M1):**

| 段 | 含义 |
| --- | --- |
| 0 | 成功 |
| 1xxxx | 通用(参数/鉴权/权限) |
| 11001 | 未登录 / Token 失效 |
| 11002 | 权限不足 |
| 11003 | 越权访问(跨租户/跨范围) |
| 12xxx | 账号与认证 |
| 13xxx | 租户、组织与身份模块装配 |
| 14xxx | 导入 |

补充: `13021` 表示身份模块 HTTP 路由装配缺少必要依赖,属于服务端装配错误,详细原因只进入日志。

**鉴权说明:**
- Access Token:JWT,有效期 15 分钟,载荷含 `account_id / tenant_id / plat`;角色从服务端会话/契约查询,不信任前端传参。
- Refresh Token:不透明随机串,租户账号哈希存 `auth_session`,平台管理员哈希存 `platform_auth_session`,有效期 7 天,轮转使用。
- 单端登录:新登录吊销该账号其余会话。

## 2. 认证接口 `/auth`

### 2.1 平台管理员登录(SaaS)
`POST /auth/login/platform`

请求:`{ "username": "admin", "password": "..." }`

说明:仅 SaaS 平台层启用;成功签发 `plat=true` 的平台级 Token,不进入任何学校租户。私有化部署关闭该入口。

### 2.2 手机号密码登录
`POST /auth/login/phone`

请求:`{ "phone": "138...", "password": "..." }`

响应(单租户匹配):
```json
{ "code": 0, "data": {
  "access_token": "...", "refresh_token": "...",
  "must_change_pwd": false,
  "account": { "id": 1, "name": "张三", "base_identity": 2, "roles": [3,2] }
}}
```

响应(一号多校,需选租户):
```json
{ "code": 0, "data": { "need_select_tenant": true,
  "tenants": [{ "tenant_id": 10, "name": "北大", "code": "pku" }] }}
```
→ 客户端带 `tenant_id` 重新请求 `POST /auth/login/phone`。

### 2.3 学号登录(备用)
`POST /auth/login/no`
请求:`{ "tenant_code": "pku", "no": "2021001", "password": "..." }`
说明:用学校短码 + 学号定位账号,响应同 2.1。

### 2.4 短信验证码登录
`POST /auth/login/sms`
请求:`{ "phone": "138...", "code": "123456" }`,响应同 2.1。

### 2.5 发送短信验证码
`POST /auth/sms/send`
请求:`{ "phone": "138...", "scene": 1, "tenant_id": "10" }`(1登录/2找回/3换绑;一号多校登录/找回短信必须带 `tenant_id`,单租户可省略;换绑手机号必须携带当前登录态,租户以服务端会话为准)
说明:Redis 限频(同号 60s、每日上限),返回 `{ "code": 0 }`。

### 2.6 刷新 Token(无感)
`POST /auth/refresh`
请求:`{ "refresh_token": "..." }`
响应:`{ "access_token": "...", "refresh_token": "..." }`(新旧轮转,旧 Refresh 立即失效)
失败:`code=11001`,要求重新登录。

### 2.7 登出
`POST /auth/logout`(需鉴权)→ 吊销当前会话。

### 2.8 找回密码
`POST /auth/password/reset`
请求:`{ "phone": "138...", "code": "...", "new_password": "...", "tenant_id": "10" }`(一号多校时必须带 `tenant_id`)

### 2.9 激活码开通
`POST /auth/activate`
请求:`{ "activation_code": "...", "password": "..." }`
说明:学校管理员启用激活码开通时,新增/导入账号返回一次性激活码;用户通过激活码自设密码并激活账号。

### 2.10 SSO 登录

CAS 与 LDAP 采用不同协议形态,不得用同一个回调接口伪装:

- `GET /auth/sso/{tenant_code}/login` → 根据该租户 CAS 配置生成 CAS 登录地址并返回 `redirect_url`,前端负责跳转。
- `GET /auth/sso/{tenant_code}/callback?ticket=...&service=...` → CAS 回调,校验 Service Ticket 后按 `sso_config.match_field` 匹配已导入名单;匹配成功签发 Token,失败返回"账号未在名单中"。
- `POST /auth/sso/{tenant_code}/ldap` → LDAP 账号密码登录,请求 `{ "username": "...", "password": "..." }`;服务端用学校 LDAP 配置完成绑定与目录查询,再按 `sso_config.match_field` 匹配名单并签发 Token。

SSO 配置要求:
- CAS 配置使用 `server_url`,可选 `username_attribute`;`service` 必须来自当前回调地址或前端提交的完整回调 URL,不得硬编码。
- CAS `service` 的 origin 必须命中部署配置 `IDENTITY_SSO_ALLOWED_SERVICE_ORIGINS`;未配置或未命中时拒绝生成登录地址与验票,避免租户配置或前端参数把票据回调导向非平台域名。
- LDAP 配置使用 `url / bind_dn / bind_password / base_dn / user_filter / match_attribute`;`url` 必须为 `ldaps://`,不允许跳过 TLS 证书校验。敏感字段保存时加密、读取给前端时脱敏。
- `match_field=1` 表示学号/工号匹配,`match_field=2` 表示手机号匹配;未命中已导入名单不得自动创建账号。

## 3. 平台管理员接口 `/platform`(仅 SaaS)

### 3.1 学校入驻申请(公开)
`POST /platform/applications`
请求:`{ "school_name": "...", "school_type": 3, "contact_name": "...", "contact_phone": "...", "contact_email": "..." }`
说明:无需登录,提交后状态为"申请中"。这是申请,不是注册。

### 3.2 申请列表 / 审核
- `GET /platform/applications?status=1`(平台管理员)
- `POST /platform/applications/{id}/approve` → 创建租户 + 分配短码 + 开通学校管理员账号(激活码)。
- `POST /platform/applications/{id}/reject`,请求 `{ "reason": "..." }`。

### 3.3 租户管理
- `GET /platform/tenants?page=&size=`、`GET /platform/tenants/{id}`
- `PATCH /platform/tenants/{id}`(改状态/到期时间)
- `GET /platform/tenants` 返回统一分页数据 `{ "list": [...], "total": 0, "page": 1, "size": 20 }`,不得退回数组或模块私有分页结构。

## 6. 账号与导入接口 `/accounts`(学校管理员)

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/accounts?role=&class_id=&status=&keyword=` | 账号列表(分页) |
| POST | `/accounts` | 单个新增(教师/学生) |
| PATCH | `/accounts/{id}` | 编辑(不含学号/工号等不可变字段) |
| POST | `/accounts/{id}/disable` | 停用 |
| POST | `/accounts/{id}/enable` | 启用 |
| POST | `/accounts/{id}/archive` | 归档(毕业/离校,可恢复) |
| POST | `/accounts/{id}/restore` | 恢复归档账号 → 正常 |
| POST | `/accounts/{id}/cancel` | 注销(终态软删,数据合规保留) |
| POST | `/accounts/{id}/force-logout` | 强制下线(吊销该账号全部会话,即时踢出) |
| POST | `/accounts/{id}/reset-password` | 管理员重置密码 |
| POST | `/accounts/{id}/grant-admin` | 授予教师"学校管理员"权限 |
| POST | `/accounts/{id}/revoke-admin` | 撤销管理员权限 |
| POST | `/accounts/batch/disable` | 批量停用 |
| POST | `/accounts/batch/archive` | 按学年批量归档 |
| POST | `/accounts/batch/restore` | 批量恢复归档 |

### 6.1 导入流程(两步)
- `GET /accounts` 返回统一分页数据 `{ "list": [...], "total": 0, "page": 1, "size": 20 }`,不得使用模块私有分页响应。
- `POST /accounts/import/preview`:上传 Excel/CSV(`multipart/form-data`: `type=student|teacher`, `file=<csv/xlsx>`),服务端解析并暂存预览结果,返回逐行校验结果(成功/失败原因),不写账号。
  响应:`{ "preview_id": "9001", "total": 100, "valid": 95, "invalid": 5, "rows": [{ "line": 3, "error": "手机号格式错误" }] }`
- `POST /accounts/import/commit`:确认导入,请求 `{ "preview_id": "9001" }`,服务端读取暂存预览行,仅写入校验通过的行,返回 `import_batch` 结果;启用激活码开通且导入行未提供初始密码时,响应一次性返回激活码明文。
- `GET /accounts/import/template?type=student|teacher&format=xlsx|csv`:下载模板;`format` 省略时默认返回 Excel 模板。
- `GET /accounts/import/batches`:导入批次历史。
- `POST /accounts/batch/archive`:按学年归档学生账号,请求 `{ "enrollment_year": 2022 }`;仅归档该学年当前处于正常状态的学生账号,教师账号及停用/注销账号不受影响。
