> 汇总 11 模块与基础横切层全部 97 张表 + 建库规范。详细字段见各模块"数据模型"文档,本文是总览与规范 SSOT。
> 最后更新:2026-05-29

## 一、建库规范

### 1. 命名
- 表名:小写下划线,单数(`account`、`course_grade`);避免复数,避免缩写歧义。
- 字段:小写下划线(蛇形);布尔 `is_` 前缀(`is_locked`);时间 `_at` 后缀(`created_at`)。
- 索引:`idx_表_字段`;唯一 `uk_表_字段`。

### 2. 主键与ID
- 主键统一 `id BIGINT`,**雪花算法**生成(非自增,避免跨节点冲突;JSON 传输转字符串防精度丢失)。

### 3. 通用字段
- `created_at` / `updated_at`:`TIMESTAMPTZ`,UTC 存储。
- 软删:`deleted_at TIMESTAMPTZ NULL`,查询默认 `deleted_at IS NULL`;唯一索引用部分索引 `WHERE deleted_at IS NULL`。
- 不物理删除业务数据;归档用状态字段。

### 4. 多租户隔离(核心)
- 租户内业务表必含 `tenant_id BIGINT NOT NULL`,建索引。
- **启用 PostgreSQL 行级安全(RLS)**:策略 `USING (tenant_id = current_setting('app.tenant_id')::BIGINT)`;应用每请求 `SET app.tenant_id`。
- 平台级表(无 tenant_id)仅平台管理员连接角色可访问。
- 私有化:`tenant_id` 恒定值,RLS 仍启用(恒真),逻辑零差异。

### 5. 类型约定
- 枚举:`SMALLINT` + 代码常量(不用 DB ENUM,便于演进);文档注明取值。
- 金额/绩点:`NUMERIC`(定点,不用 float)。
- 结构化/可变:`JSONB`(配置、内容体、详情);大文本/二进制存 MinIO,表存引用 key。
- 敏感字段(手机号):加密列 `BYTEA` + 查询用 HMAC 哈希列。

### 6. 索引约束
- 外键关联字段必建索引;单表索引数量克制(按查询模式建,避免冗余)。
- 数组字段(tags/knowledge_points)用 GIN 索引。

### 7. 范围标注
- **全局表**:不含 tenant_id,平台级共享(运行时/工具/判题器/仿真包/模板等平台配置 + platform_admin)。
- **租户表**:含 tenant_id + RLS。
- **共享库例外**:M5 内容仅 `visibility=shared && status=published` 项跨租户只读可克隆;M6 课程的 `visibility=shared` 项按课程共享规则跨租户只读可克隆。

## 三、跨模块数据归属铁律

- **审计**:全平台仅 M1 `audit_log` 一张表,各模块写入,M9 查询;审计记录必须保留 `trace_id`,便于把前端报障编号、服务日志与关键操作串联。`actor_role`:1平台管理员、2学校管理员、3教师、4学生、5内部系统任务;内部系统任务使用 `actor_id=0` 并在 `detail.source_ref` 记录已验证来源。
- **答案/判题配置**:存 M5 content_body,M3 取用不存。
- **学生代码**:MinIO 以 M2 持久化为主,M3/M6/M8 存引用+hash,不另存副本。
- **成绩**:单课程在 M6 course_grade,跨课程聚合在 M11(只读 M6,不写);审核流程锁定态在 M11 grade_review,同步写保护投影在 M6 course_grade.is_locked。
- **题目内容不拷贝**:M6/M7/M8 引用 M5 `(item_code, item_version)`,锁版本;内容复用统计只通过 M5 `ContentReadService.ReplaceUsageRefs` 写入 `content_usage_ref`,业务模块不得自增 `usage_count` 或自建引用统计。
