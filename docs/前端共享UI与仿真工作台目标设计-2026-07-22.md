# 前端共享 UI 与仿真工作台目标设计（2026-07-22）

## 1. 设计结论

共享层必须先于四端业务页重构,但“共享”只共享稳定的视觉语言、交互机制、状态契约和结构原语,不共享具体业务页面。

实施顺序固定为:

1. 设计令牌与全局交互规则。
2. `@chaimir/ui` 基础组件。
3. `PageScaffold`、`WorkbenchShell` 等结构原语。
4. `@chaimir/ui/biz` 稳定业务语义组件。
5. `@chaimir/sim-sdk` 的 SimulationWorkbench 与七类封闭渲染器。
6. 四端业务页按用户任务组合上述能力。

四端不得等待全部业务页面设计完成才修共享层,否则同一缺陷会在每个页面重复实现;共享层也不得提前抽象未稳定的业务组合,否则会把错误布局固化成组件 API。

## 2. 权威依据

- 产品与后端边界:`docs/前端后端功能对齐清单.md`、公开 API/DTO、`@chaimir/api-client`。
- 全站设计约束:`docs/总-前端设计规范.md`。
- 页面任务与信息架构:`docs/前端页面设计/*.md`。
- 仿真教学契约:`docs/04-仿真可视化引擎/*.md` 和 TeachingFrame 协议。
- 当前问题证据:`docs/前端UIUX全面重构审计-2026-07-22.md` 与历史 E2E 截图。

当前代码和截图只用于核对已有功能、状态与缺陷,不用于继承颜色比例、卡片层级、阴影、辉光、动效、空间关系或页面模板。

## 3. 防止旧设计回流

每个共享组件重构前都执行以下流程:

1. **冻结语义**:列出现有 props、状态、无障碍行为和真实调用方,区分必须保留的功能与可删除的外观。
2. **脱离截图建模**:从用户任务、数据密度、错误恢复和后端状态画低保真结构,不先看现有 CSS。
3. **建立目标契约**:先定义默认状态、交互状态、窄屏、键盘、触屏和 reduced-motion,再写视觉实现。
4. **对照旧证据查回归**:只在目标方案形成后回看旧截图,确认没有复现已知问题。
5. **真实浏览器验收**:以新实现生成的新截图为准,旧截图不得成为视觉 diff 的批准基线。

以下表现一律视为旧设计回流并阻止合并:

- 页面或区块默认套卡片,卡片内再套卡片。
- 大面积青色辉光、毛玻璃、渐变或装饰网格压过业务信息。
- 所有页面固定套“图标标题 + 统计卡 + 右侧动作卡 + 表格”。
- hover 位移、侧栏右移、列表逐项入场或页面统一淡入成为默认行为。
- 内部 ID、引用、事件键、User-Agent 或后端字段名成为主信息。
- 用前端固定百分比、动画或图表伪造后端状态。
- 为了复用把业务页面、仿真算法或权限逻辑塞进 `@chaimir/ui`。

## 4. 共享层职责

| 层 | 负责 | 不负责 |
| --- | --- | --- |
| tokens | 原始色、语义色、字体、间距、圆角、边框、阴影、层级、时长、缓动 | 页面布局和业务状态映射 |
| `@chaimir/ui/components` | Button、Form、Table、Overlay、Feedback、Navigation 等无业务语义组件 | 课程、实验、成绩、租户等 DTO 与 API |
| PageScaffold | 页面头、上下文栏、主体、无框区段、可选 rail 和响应式槽位 | 固定统计卡、固定右侧动作区和角色专属布局 |
| WorkbenchShell | 全屏壳、顶栏、主舞台、辅助面板、控制区、焦点与响应式 | 文件树、仿真叙事、题面、判题和实例状态机 |
| `@chaimir/ui/biz` | 两处以上稳定复用的业务视觉语义,如 SandboxStatus、CheckpointCard | 单页面临时组合和跨模块业务流程 |
| `@chaimir/sim-sdk` | Worker、TeachingFrame、仿真叙事、交互、控制、七类模式渲染 | 课程入口、权限、分享路由、审核流程和成绩 |
| `apps/web/features` | 角色任务、API 状态、权限、页面信息架构和共享层组合 | 复制共享组件或仿真引擎 |

## 5. 视觉方向

### 5.1 日常界面

- 深色应用壳 + 浅色内容面,但内容区以无框页面结构为主。
- 青电用于主操作、焦点和当前选择;琥珀用于真实提醒、能量或对抗,不用于普通装饰。
- 主表面使用稳定实色和细边框;阴影只表达真实浮层/悬浮层级。
- Card 圆角不超过 8px;Button、Input、Table 等紧凑工具控件保持稳定高度和边界。
- 页面标题匹配工作界面密度,不使用营销式超大字号。
- 字距统一为 0;状态、数字和代码可使用等宽字体,正文保持高可读性。

### 5.2 沉浸工作台

- 深色舞台以内容和因果关系为第一视觉层级,框架本身退后。
- 不用满屏玻璃和辉光制造“区块链感”;当前焦点最多使用一层细描边/低强度光晕。
- 舞台、检查器和控制区通过尺寸、边界与位置建立层级,不靠嵌套卡片。
- 文本解释保持短且就近;长证据、代码追踪和检查点进入可折叠检查器。

## 6. `@chaimir/ui` 目标组件体系

### 6.1 第一批:令牌与高频控件

- Motion tokens:press 120–160ms、popover 125–180ms、select 150–220ms、modal/drawer 200–260ms。
- Easing tokens:强 ease-out、屏上重排 ease-in-out、持续进度 linear;不提供 ease-in UI 默认值。
- Button/IconButton:稳定尺寸、loading 不引发布局位移、`:active` 微缩、危险动作明确、图标按钮必须有名称。
- Input/Select/Textarea/FormField:可见标签、说明、字段级错误、readonly/disabled/invalid 完整;密码输入统一显隐能力。
- Checkbox/Radio/Switch/SegmentedControl:使用真实语义,触控目标至少 44px,状态不只靠颜色。
- Tooltip/Menu/Select/Popover:定位外层独占 transform,视觉内层从触发源出现;首次 Tooltip 延迟,相邻 Tooltip 即时。

### 6.2 第二批:反馈、数据和浮层

- ResourceState/Empty/Skeleton/Spinner:匹配最终结构,错误可重试并展示可用 trace_id,无权限与空数据分离。
- Toast:可中断 transition,页面隐藏时暂停计时,同类消息堆叠稳定,不得抢焦点。
- Modal/Drawer:焦点陷阱、Esc、恢复触发点焦点、明确标题;移动端按任务选择 bottom sheet 或全高 Drawer。
- Table:桌面高密度扫描;移动端按列优先级转键值卡片或独立横滚;列标题和数据都保持业务语义。
- Progress:只有真实总量时显示百分比;不确定任务使用离散状态或 indeterminate,不得映射假百分比。

### 6.3 第三批:结构原语

`PageScaffold` 目标结构:

```text
PageScaffold
├─ PageHeader: breadcrumb / title / description / status / primary action
├─ ContextBar?: tabs / filters / selection / bulk actions
└─ PageBody
   ├─ main: task-specific content
   └─ rail?: contextual evidence or actions
```

默认行为:

- 外层、PageHeader、PageBody、PageSection 都不带卡片外观和 hover。
- rail 仅在内容存在时占位;1024 以下由调用方声明下移、Drawer 或分层导航。
- 不内置图标标题、指标网格、动作卡、页面入场动画或角色强调色。

`WorkbenchShell` 目标结构:

```text
WorkbenchShell
├─ WorkbenchBar: exit / title / connection or run state / page actions
├─ WorkbenchBody
│  ├─ primary stage
│  └─ optional inspector
└─ WorkbenchControls?: timeline / primary controls / settings
```

默认行为:

- 桌面主舞台优先,inspector 建议 320–384px,可收起但首次教学进入由业务决定是否展开。
- 控制区尺寸稳定,内容变化不得推高舞台或造成布局跳动。
- 1024 以下 inspector 变 Drawer/Sheet;移动端舞台优先,次要设置进入菜单,不把所有桌面控件压成多行工具条。
- 抽屉只动画 transform/opacity,200–260ms ease-out;键盘高频开关可即时。

## 7. `@chaimir/sim-sdk` 目标设计

### 7.1 核心任务

SimulationWorkbench 的第一任务不是“播放动画”,而是让学生在每一阶段回答四个问题:

1. 当前发生了什么。
2. 为什么进入这个阶段。
3. 哪些节点、消息、区块或变量发生变化。
4. 我可以执行什么操作,执行后造成什么结果。

### 7.2 桌面信息架构

- 顶栏:返回、仿真名称、当前阶段、运行状态和页面级动作;去掉重复的引擎宣传标签。
- 舞台头:只保留一句阶段摘要和观察重点,不得重复相同摘要。
- 主舞台:仅渲染 `TeachingFrame.layout.primary`;当前焦点清楚,历史元素降级但仍可追踪。
- inspector:默认展示当前阶段“做了什么/为什么”、可用交互;过程记录、设问、指标、代码追踪、检查点和学习目标按重要性折叠。
- 控制区:回退、播放/暂停、单步为主控制;重置、速度和步骤时长为次级设置;真实时间线占据剩余空间。

### 7.3 七类渲染器

| 模式 | 必须优先表达 | 禁止回流 |
| --- | --- | --- |
| graph | 节点角色、消息方向、发送/到达/丢弃、当前法定人数 | 全连接高亮叠成光团、只靠颜色区分状态 |
| chain | 主链、分叉、区块高度、tip、孤块原因 | 每个区块都悬浮发光、hash 抢占主标签 |
| tree | 父子关系、验证路径、当前计算节点 | 毛玻璃节点和大面积径向背景 |
| matrix | 行列语义、当前单元、是/否/故障及文字替代 | 只靠底色、无键盘选择、整表闪烁 |
| pipeline | 阶段顺序、当前输入输出、完成/失败 | 所有步骤卡片悬浮位移、假进度条 |
| lane | 参与方、时间方向、消息生命周期和因果 | 标签互相遮挡、消息只做装饰飞行 |
| chart | 量纲、系列、当前点、趋势与数据表替代 | 装饰阴影、无数据表、动态更新不可暂停 |

### 7.4 仿真动效

- tick/阶段推进只动画真正变化的元素,其他元素保持稳定。
- 消息移动必须表达发送到接收的空间关系;进入/到达使用 ease-out,持续传播近似 linear。
- 节点选择、hover 和 focus 不位移节点,只改变描边、表面或小范围 scale。
- 不动画工作台 `padding/right/width/inline-size`;inspector 使用独立 transform 层。
- pattern 初次渲染默认不做统一 translateY 入场,避免每次切阶段都像重新加载组件。
- reduced-motion 下关闭路径移动、缩放和辉光脉冲,保留阶段文字、方向箭头、消息状态和单步控制。

### 7.5 历史截图对应修正

| Before | After | Why |
| --- | --- | --- |
| PBFT 顶部同一摘要重复两遍 | 舞台头只保留“当前阶段 + 观察重点”,详细原因放 inspector | 减少重复并把空间还给主过程 |
| inspector 初始收起,学生看不到阶段解释和可用操作 | 首次教学会话默认显示当前阶段说明;回放/小屏可按上下文收起 | 教学叙事不能隐藏在不显眼的边缘按钮后 |
| Prepare 消息全网高亮并叠加粗辉光 | 显示发送中、已到达、已计票的不同生命周期,当前消息轻描边,历史消息降低不透明度 | 让学生读懂消息流而不是看到青色光团 |
| 底部把时间线、四个按钮、步长、速度全部并排 | 回退/播放/单步为主组;重置和节奏设置进入次级组/菜单 | 建立主次并保证 1024 和移动端可用 |
| 节点、区块、树、步骤普遍毛玻璃 + hover 位移 | 使用实色表面和稳定位置;hover 仅精细指针下改变描边/表面 | 降低视觉噪声和触屏粘滞 hover |
| inspector 抽屉动画 `padding-right/right` 且 400ms 弹跳 | 独立 transform 层 200–260ms ease-out,舞台尺寸稳定 | 避免布局重算并提高响应感 |

## 8. 响应式与输入方式

- 375:单列舞台;顶栏只保留返回、短标题和状态;播放控制保持单行核心动作;inspector 为全高 Sheet。
- 768:舞台优先;说明和交互使用 Drawer;支持横屏扩大舞台。
- 1024:桌面壳和可收起 inspector;控制区不得超过两层。
- 1440:主舞台扩展,说明文本和 inspector 保持可读宽度,不把长文本拉满。
- hover 视觉必须放在 `(hover: hover) and (pointer: fine)`;触屏依赖 pressed/selected/focus。
- 拖拽使用 pointer capture、多触点保护和边界阻尼;无拖拽语义的组件不得为了“高级感”增加手势。

## 9. 实现批次与门禁

### Batch A:规范和令牌

- 清理旧动效、辉光、卡片默认和固定资源页模板。
- 统一 motion、surface、border、focus、overlay、control-size 和 breakpoint 契约。
- 建立禁止 `transition: all`、布局属性动画、未保护 hover 和过量 backdrop-filter 的静态检查。

### Batch B:UI 基础组件

- Button/Form/Overlay/Feedback/Table 逐组件重构。
- 为每个组件建立 default/hover/focus/active/disabled/loading/error/reduced-motion 视觉矩阵。
- 使用独立组件演示页验证长文案、图标、键盘、触屏和高对比状态,不从业务页反推默认外观。

### Batch C:结构原语与共享状态

- 重构 PageScaffold、WorkbenchShell、ResourceState、TopNavbar/AppSidebar 调用边界。
- 选择认证、教师批改、学校组织、平台运行时各一个代表页先验证组合能力,但代表页通过后仍须迁移全部页面,且不在共享层写这些业务布局。

### Batch D:SimulationWorkbench

- 先重构壳、inspector 和 controls,保持 Worker/TeachingFrame/回放协议不变。
- 再逐一重构 graph/chain/tree/matrix/pipeline/lane/chart,每类使用至少两个真实内置仿真验证。
- PBFT 必须做阶段逐帧验收;Gas/链/树/矩阵覆盖不同视觉模式,不能只验一个场景。

### Batch E:四端迁移

- 按 `docs/前端UIUX全面重构审计-2026-07-22.md` 的 P0/P1/P2 顺序迁移全部已挂载页面；优先级只决定顺序,不缩减范围。
- 删除被新共享层替代的业务 CSS,不保留新旧两套外观或兼容 class。
- 逐项覆盖 96 个懒加载入口、104 个业务 TSX 和全前端 79 个 CSS 文件；共享样式生效不等于页面语义与交互已经审查。

## 10. 验收

- 静态:`pnpm type-check`、`pnpm lint`、`pnpm build`、CSS 规则扫描、`git diff --check`。
- 组件:键盘、触屏、读屏名称/状态、焦点恢复、disabled/loading、长文本和 reduced-motion。
- 视觉:375/768/1024/1440 新截图;无横向溢出、遮挡、卡片嵌套、过度辉光和空白占位。
- 仿真:七模式非空、阶段可读、元素可选、单步/回退/播放可控、inspector 可达、画布像素非空。
- 业务:四端代表页使用真实 API 和状态,不得用 mock、固定百分比或静态图填充。
- 回归:旧截图仅用于确认已知问题没有复现;新实现不以旧截图像素相似为成功标准。
