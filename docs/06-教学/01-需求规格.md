# M6 教学 — 需求规格

> 模块定位:第一个业务模块。承载课程教学全流程:课程内容、班级选课、作业测验、学习进度、讨论评价、单课程成绩。
> 核心理念:业务薄、引擎厚。题目来自 M5、判题调 M3、实验/仿真交 M2/M4、跨课程成绩归 M11。
> 最后更新:2026-05-29

---

## 1. 模块目标

- 教师组织课程(章节/课时/多形态内容)、建班选课、布置作业、批改、管成绩。
- 学生学习、提交作业、看进度、参与讨论。
- 课时可关联实验(M5 模板)与仿真(M4),由引擎拉起。
- 作业题目来自 M5 题库,判题调 M3,M6 只组织与收集。
- 单课程成绩在 M6;跨课程聚合归 M11。

## 2. 职责边界

| M6 负责 | M6 不负责(归属) |
| --- | --- |
| 课程/章节/课时、内容组织 | 题目内容存储(→ M5,引用/组卷) |
| 班级、选课、课程成员 | 判题执行(→ M3) |
| 作业/测验组织、收提交、批改 | 实验环境/仿真运行(→ M2/M4) |
| 学习进度、讨论、公告、评价 | 跨课程成绩/GPA/学业预警/成绩单(→ M11) |
| 单课程成绩(权重/汇总/调分/导出) | 用户与组织(→ M1) |
| 课程共享/克隆、课程表 | 通知推送底座(→ M10) |

## 3. 功能需求

### A. 课程管理(教师)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 创建/编辑课程 | 名称/描述/类型(理论/实验/混合/项目)/难度/封面/学期/学分 |
| A2 | 课程表 | 每周上课时间段 |
| A3 | 发布/结束/归档 | 课程状态流转 |
| A4 | 克隆/共享 | 复制课程或共享到课程库(复用 M5 克隆理念) |
| A5 | 邀请码 | 生成/刷新供学生加入 |

### B. 内容管理(教师)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 章节/课时 | 增删改排序 |
| B2 | 多形态内容 | 视频、图文(Markdown)、附件 |
| B3 | 关联实验 | 课时挂 M7 实验定义引用;实验定义内部再引用 M5 模板/M2/M4(评分实验走此路径) |
| B4 | 关联仿真 | 课时直挂 M4 仿真包引用,**仅作展示讲解,不计分、不记完整实验进度** |

> **课时仿真双路径澄清(B9 修复)**:课时直挂 M4 仿真(B4)= 教学演示,只看不评分;需要评分/检查点的仿真一律走"课时→M7 实验→M4"(B3),由 M7 编排检查点与得分。避免同一仿真出现展示与评分两套进度重复记录。

### C. 班级与选课
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 学生加入 | 邀请码加入 / 教师按班级批量添加 |
| C2 | 成员管理 | 查看/移除课程学生 |

### D. 作业与测验
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 创建作业 | 标题/截止/提交次数/迟交策略;从 M5 引用题目或组卷 |
| D2 | 题型 | 客观题(M3 自动判)/主观题(教师批)/编程题(M3 判)/实验报告 |
| D3 | 学生作答 | 提交;草稿自动保存(localStorage + 服务端,沿用旧版) |
| D4 | 批改 | 主观题/实验报告教师批改给分评语;客观/编程题 M3 判 |
| D5 | 迟交策略 | 不允许/扣分/不扣分 |
| D6 | 编程题查重 | 教师可启用:M3 提供相似度/哈希查重能力,教师据结果自行认定学术不端并处理(M6 不自动判定) |

### E. 学习进度
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 课时完成 | 视频进度/图文到底/实验完成标记完成 |
| E2 | 学习时长 | 记录时长 |
| E3 | 进度统计 | 教师看班级完成率,学生看自己进度 |

### F. 讨论/公告/评价
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 讨论区 | 发帖/回复/点赞,教师置顶/删除 |
| F2 | 公告 | 教师发布,可置顶 |
| F3 | 评价 | 课程结束后学生评分评论 |

### G. 单课程成绩
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| G1 | 权重配置 | 各作业/实验在总成绩占比,和为 100% |
| G2 | 成绩汇总 | 按权重自动算本课程总成绩 |
| G3 | 手动调整 | 教师覆盖,保留自动值,标记已调整;单课程成绩写保护投影由 M6 自管,申诉改分仍由 M6 执行,M11 只管理申诉/审核状态并在 M6 改分事件后重算 GPA |
| G4 | 导出 | 成绩单 Excel |

> G 仅本课程;跨课程聚合/GPA 归 M11,M11 只读本模块成绩。

## 4. 非功能需求

- **业务薄**:不重复造引擎能力,题目/判题/环境/仿真全部引用对应模块。
- **多租户**:课程及衍生数据带 tenant_id 隔离。
- **可复现引用**:作业引用 M5 题目锁版本。

## 5. 明确不做(边界)

- 不存题目内容(M5)、不判题(M3)、不跑实验/仿真(M2/M4)。
- 不做跨课程成绩/GPA/学业预警/成绩单/审核流程(M11);M6 只提供单课程成绩与写保护投影读写能力,并在成绩变更后发布事件供 M11 重算。
- 不做通知推送底座(M10,M6 触发事件交 M10 投递)。

## 6. 技术取向

Go + PostgreSQL;视频/附件存对象存储(MinIO);富文本 Markdown。最终技术细节由《技术选型》统一收口。
