# M6 教学 — 业务流程与状态机

> Mermaid 描述课程生命周期、作业提交批改、学习进度、单课程成绩计算。
> 最后更新:2026-05-29

---

## 1. 课程状态机

```mermaid
stateDiagram-v2
    [*] --> 草稿: 教师创建
    草稿 --> 已发布: 发布(至少1章节1课时)
    草稿 --> 已删除: 软删
    已发布 --> 进行中: 到达开始时间(自动)
    进行中 --> 已结束: 到达结束时间/教师手动
    已结束 --> 已归档: 教师归档
    已归档 --> [*]
```

- 已发布/进行中可编辑内容,但不能删已有提交的作业。
- 学生名单管理仅在已发布/进行中。
- 自动状态推进由 M6 后台任务完成,轮询间隔由 `TEACHING_COURSE_STATUS_POLL_INTERVAL_SECONDS` 配置,启动时必须为正数。

## 3. 作业提交与判题(跨模块协作)

```mermaid
sequenceDiagram
    participant S as 学生
    participant M6 as 教学
    participant M5 as 题库
    participant M3 as 评测
    S->>M6: 打开作业
    M6->>M5: 取题面(过滤答案)
    M6-->>S: 展示题目
    S->>M6: 提交答案
    M6->>M6: 本地事务写 submission + submission_judge_outbox
    alt 客观题/编程题
        M6->>M6: outbox worker claim pending -> running
        M6->>M3: 派发判题(带 M5 题目引用与 source_ref)
        M3->>M5: 取判题配置+答案
        M3-->>M6: 返回 judge_task_ref
        M6->>M6: 写回 judge_task_ref 并置 outbox done
        M3-->>M6: judge.completed / judge.failed 事件
        M6->>M6: 回写 auto_score/final_score
    else 主观题/实验报告
        M6->>M6: 置"待批改"
        Note over M6: 教师批改 → manual_score
    end
    M6->>M6: 计算 final_score(含迟交扣分)
```
> 自动判题派发使用 M6 本地 outbox。请求事务只写 M6 自有表,不直接调用 M3;outbox worker 将 pending 任务声明为 running 后派发。派发成功写回 `submission.judge_task_ref` 并将 outbox 置为 done;派发失败恢复 pending、递增重试次数并记录 `last_error` 供运维排查。M6 对一次提交内每道自动题生成一条 outbox,使用 `source_ref = teaching:<year>:submission:<id>:item:<assignment_item_id>`,M3 对 `tenant_id + source_ref` 幂等,保证 outbox 重试不会重复创建判题任务。M6 收到单题 `judge.completed` 后先回写该 outbox 的 `score/completed_at`,只有该提交全部自动题完成后才聚合写回 `submission.auto_score/final_score` 并发布 `teaching.grade.updated`。outbox 每轮批量和后台轮询间隔分别由 `TEACHING_JUDGE_OUTBOX_BATCH_SIZE`、`TEACHING_JUDGE_OUTBOX_POLL_INTERVAL_MS` 控制,启动时必须为正数。

## 4. 作业提交状态机

```mermaid
stateDiagram-v2
    [*] --> 未提交
    未提交 --> 已提交: 学生提交
    已提交 --> 已批改: 客观/编程题 M3 自动判
    已提交 --> 待批改: 主观题/报告
    待批改 --> 已批改: 教师批改
    已批改 --> 已提交: 截止前重新提交(受次数限制)
    已批改 --> [*]
```

> 截止后提交标记迟交;按迟交策略处理。

## 5. 作答草稿

```mermaid
flowchart TD
    A[进入作答页] --> B[读本地 localStorage 草稿]
    B --> C[拉服务端草稿]
    C --> D{服务端草稿是否存在}
    D -->|是| E[以服务端草稿恢复]
    D -->|否| F[展示本地未同步草稿提示]
    F --> G[学生确认后保存到服务端]
    E --> H[作答中定期保存]
    G --> H
    H --> I[POST /draft]
    I --> J[更新 submission_draft]
    I --> K[本地记录最新同步版本]
    H --> L[正式提交]
    L --> M[服务端草稿失效]
```

> 服务端 `submission_draft` 是作答草稿权威状态。本地 localStorage 只保存未同步编辑态和最近同步标记,不得在服务端保存失败或读取失败时直接替代服务端草稿进入提交链路;提交必须以服务端鉴权、作业状态和成员关系校验为准。

## 6. 单课程成绩计算

```mermaid
flowchart TD
    A[教师配置权重 和=100%] --> B[触发计算]
    B --> C[汇总各作业/实验成绩]
    C --> D[按权重加权 → auto_total]
    D --> E{教师手动调整?}
    E -->|是| F[override_total + 标记已调整]
    E -->|否| G[用 auto_total]
    F --> H[course_grade]
    G --> H
    H --> I[M11 跨课程聚合只读取用]
```

> 成绩导出必须使用真实 Excel 工作簿并分批读取 `course_grade`;导出分页批量由 `TEACHING_GRADE_EXPORT_BATCH_SIZE` 控制,启动时必须为正数,避免在模块代码中硬编码运行阈值。
