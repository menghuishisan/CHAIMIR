# M6 教学 — 数据模型

> PostgreSQL,全部租户级(tenant_id + RLS)。建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 |
| --- | --- |
| `course` | 课程(含课程表/学分/状态) |
| `chapter` | 章节 |
| `lesson` | 课时(多形态内容 + 实验/仿真引用) |
| `course_member` | 课程成员(选课关系) |
| `assignment` | 作业/测验 |
| `assignment_item` | 作业题目引用(指向 M5) |
| `submission` | 学生提交 |
| `submission_judge_outbox` | 自动判题提交 outbox,用于 M6 本地事务提交后可靠派发 M3 |
| `submission_draft` | 作答草稿 |
| `lesson_progress` | 学习进度 |
| `discussion_post` | 讨论帖/回复 |
| `announcement` | 课程公告 |
| `course_review` | 课程评价 |
| `course_grade` | 单课程成绩 |
| `grade_weight` | 成绩权重配置 |
| `teaching_grade_event_outbox` | 成绩变更事件 outbox,用于可靠发布 `teaching.grade.updated` |

> 消灭冗余/合并说明:
> - **课程表/学分并入 `course`**(一对一小字段,不另立表)。
> - **章节、课时分两表**:课时内容形态多、字段多,且按章节组织,层级清晰故分表。
> - **作业题目引用 `assignment_item` 只存 M5 题目引用 + 分值**,不拷贝题面/答案(答案在 M5,消灭冗余)。
> - **讨论帖与回复同表 `discussion_post`** 用 `parent_id` 自关联(帖/回复结构同构)。
> - **提交内容大字段(代码/报告)存对象存储**,`submission` 存引用与判题结果引用(判题在 M3)。

### 3.2 assignment_item — 作业题目引用

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| assignment_id | BIGINT | |

### 3.4 submission_judge_outbox — 自动判题派发 outbox

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| submission_id | BIGINT | 对应 M6 提交 |
| assignment_id | BIGINT | |
| source_owner_id | BIGINT | 来源归属账号快照,教学 outbox 固定为课程负责教师 |
| source_course_id | BIGINT | 来源课程快照,固定为作业所属课程 |
| source_scope | VARCHAR(32) | 来源类型快照,教学 outbox 固定为 `teaching` |
| student_id | BIGINT | |
| item_code | VARCHAR(96) | M5 题目 code |
| item_version | VARCHAR(32) | 锁定版本 |
| judger_code | VARCHAR(64) | M3 判题器 code |
| code_storage_key | VARCHAR(255) | 提交代码对象存储 key |
| code_hash | VARCHAR(128) | 提交代码哈希 |
| extra_input | JSONB | 判题额外输入 |
| source_ref | VARCHAR(96) | `teaching:<year>:submission:<id>:item:<assignment_item_id>` |
| status | SMALLINT | 1待派发/2派发中/3已完成 |
| retry_count | INT | 派发尝试次数 |
| last_error | TEXT NULL | 最近一次派发失败原因,仅供运维排查 |
| score | INT NULL | 单题判题得分,由 M3 完成事件回写 |
| completed_at | TIMESTAMPTZ NULL | 单题判题完成时间 |
| created_at / updated_at | TIMESTAMPTZ | |

> 自动判题不在请求事务内直接调用 M3。M6 先在本地事务写入 `submission` 与 `submission_judge_outbox`,事务提交后由 outbox worker 派发 M3。M3 按 `tenant_id + source_ref` 幂等,避免 outbox 重试重复创建判题任务。
> 一次提交可包含多道自动判题题目,因此 outbox 是**每提交题目一条**,唯一约束为 `(tenant_id, submission_id, assignment_item_id)` 与 `(tenant_id, source_ref)`。`submission.auto_score/final_score` 只在该提交的全部自动题终态完成后聚合回写,避免单题完成时提前结算整份作业。

## 5. 单课程成绩

### 5.1 grade_weight — 权重配置

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| course_id | BIGINT | |

| weight | NUMERIC(5,2) | 权重% |

> 校验:同课程权重和 = 100%。

### 5.2 course_grade — 单课程成绩

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| course_id | BIGINT | |
| student_id | BIGINT | |
| auto_total | NUMERIC(5,2) | 按权重自动算总分 |
| override_total | NUMERIC(5,2) NULL | 教师手动覆盖值 |
| is_overridden | BOOLEAN | 是否已调整 |
| is_locked | BOOLEAN | 单课程成绩当前是否处于写保护态,供 M6 同步拦截改分 |
| updated_at | TIMESTAMPTZ | |

唯一约束:`(course_id, student_id)`。

> 供 M11 跨课程聚合只读;M11 不写本表。
> **写保护投影**:`course_grade.is_locked` 是 M6 自有写保护投影,用于在改单课程成绩时同步拦截旁路修改。其状态由 M11 审核/申诉流程经事件驱动同步到 M6,但最终写入仍由 M6 自己完成,避免第2层反向依赖第3层。M6 改分成功后发布成绩变更事件,M11 订阅后重算 GPA、复核预警并重新驱动锁定投影恢复。

### 5.3 teaching_grade_event_outbox — 成绩变更事件 outbox

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |

| event_updated_at | TIMESTAMPTZ | 成绩事件时间 |
| status | SMALLINT | 1待发布/2发布中/3已发布/4发布失败 |
| retry_count | INT | 发布尝试次数 |
| last_error | TEXT NULL | 最近一次发布失败原因,仅供运维排查 |
| created_at / updated_at | TIMESTAMPTZ | |

> `course_grade` 写入或改分与 `teaching_grade_event_outbox` 必须在同一 M6 事务内完成。后台任务领取 outbox 后发布 `teaching.grade.updated`,失败只回写 outbox 状态并重试,不得在成绩事务提交后裸调用事件总线作为唯一投递路径。

## 6. 多租户与索引

- 全部表 `tenant_id` + RLS(同 M1)。`course.visibility=2` 共享课程库跨租户只读可克隆(克隆复制内容,不含成员/提交/成绩)。
- 关键索引:
  - `course(tenant_id, teacher_id, status)`、`course(invite_code)`
  - `course_member(course_id, student_id)` UNIQUE
  - `submission(tenant_id, assignment_id, student_id)`、`submission(judge_task_ref)`
  - `submission_judge_outbox(tenant_id, submission_id, assignment_item_id)` UNIQUE
  - `submission_judge_outbox(tenant_id, source_ref)` UNIQUE
  - `submission_judge_outbox(tenant_id, status, created_at)`
  - `teaching_grade_event_outbox(tenant_id, status, created_at)`
  - `teaching_grade_event_outbox(tenant_id, course_id, student_id)`
  - `lesson_progress(lesson_id, student_id)` UNIQUE
  - `course_grade(course_id, student_id)` UNIQUE
  - `discussion_post(tenant_id, course_id, parent_id)`
