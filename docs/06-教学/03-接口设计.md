# M6 教学 — 接口设计

> RESTful。鉴权沿用 M1 JWT。题目取自 M5、判题调 M3,M6 接口只管教学业务。
> 最后更新:2026-05-29

---

## 1. 通用约定

- Base:`/api/v1/teaching`
- 统一响应体同 M1。错误码段:`6xxxx`(61xxx 课程/62xxx 作业/63xxx 进度/64xxx 成绩)。
- 角色:教师建课/批改/管成绩;学生学习/提交。

---

## 2. 课程管理 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/courses?role=teacher&status=` | 我的课程 |
| POST | `/courses` | 创建课程(草稿) |
| PATCH | `/courses/{id}` | 编辑 |
| POST | `/courses/{id}/publish` | 发布 |
| POST | `/courses/{id}/end` | 结束 |
| POST | `/courses/{id}/archive` | 归档 |
| POST | `/courses/{id}/clone` | 克隆 |
| POST | `/courses/{id}/share` | 共享到课程库 |
| POST | `/courses/{id}/invite-code/refresh` | 刷新邀请码 |

## 6. 作业与测验

### 6.1 教师
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/courses/{id}/assignments` | 创建作业(从 M5 引用题目/组卷) |
| PATCH | `/assignments/{id}` | 编辑 |
| POST | `/assignments/{id}/publish` | 发布 |
| GET | `/assignments/{id}/submissions` | 查看提交情况 |
| POST | `/submissions/{id}/grade` | 批改(主观题/报告:给分+评语) |

### 6.2 学生
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/assignments/{id}` | 作业详情(题面取自 M5,过滤答案) |
| POST | `/assignments/{id}/draft` | 保存草稿(自动每60s + 手动) |
| POST | `/assignments/{id}/submit` | 提交;客观/编程题转 M3 判题 |
| GET | `/submissions/{id}` | 查看反馈(成绩/评语/判题详情) |

> 提交流程:M6 收提交 → 客观/编程题调 M3(`POST /judge/tasks`,带 M5 题目引用)→ M3 回写 → M6 记 submission。

## 8. 单课程成绩 `[教师]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET/PUT | `/courses/{id}/grade-weights` | 权重配置(和=100%校验) |
| POST | `/courses/{id}/grades/compute` | 按权重计算全班成绩 |
| GET | `/courses/{id}/grades` | 成绩汇总 |
| PATCH | `/courses/{id}/grades/{sid}` | 手动调整(仅授课教师;申诉受理后的单课程改分仍由 M6 执行) |
| GET | `/courses/{id}/grades/export` | 导出 Excel |
| GET | `/courses/{id}/grades` `[内部]` | M11 聚合只读取用(带 tenant_id) |
| `teaching.grade.updated` | 事件 | M6 成绩变更后发布,供 M11 重算 GPA |

> **成绩锁定一致性(A6 修复)**:成绩审核锁状态 `is_locked` 由 M11 管理。为遵守分层依赖规则和聚合层只读边界,M6 不直接调用 M11,M11 对 M6 仅使用只读成绩契约。申诉改分链路为:学生提交申诉 → M11 受理并解锁/记录审核状态 → 授课教师在 M6 执行单课程改分 → M6 发布 `teaching.grade.updated` 事件 → M11 订阅后重算 GPA、复核预警并重新进入审核锁定流程。M6 不保存锁状态、不反向依赖 M11。
