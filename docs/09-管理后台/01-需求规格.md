# M9 管理后台 — 需求规格

> 模块定位:运营管理与监控的聚合层。两级管理(平台/学校),聚合各模块数据做看板与运营,只读不跨写。
> 核心理念:M9 不拥有业务数据,只拥有运维元数据;基础设施监控外接专业运维体系。
> 最后更新:2026-05-29

---

## 1. 模块目标

- **两级管理**:平台管理员(SaaS)管全平台;学校管理员管本校运营视图。
- **聚合看板**:跨模块只读聚合数据做运营看板与统计。
- **统一审计查询中心**:跨模块审计日志聚合查询(各模块自写,M9 只查)。
- **系统配置**:全局/租户级运维参数。
- **业务级告警 + 基础监控入口**:基础设施监控外接 Prometheus/Grafana,M9 嵌入展示。

## 2. 职责边界

| M9 负责 | M9 不负责(归属) |
| --- | --- |
| 两级运营看板(聚合只读) | 业务数据 CRUD(各业务模块) |
| 统一审计查询中心 | 业务模块审计日志写入(各模块自写);M9 自身高权限操作经统一审计能力写 M1 audit_log |
| 系统配置、备份记录 | 学校审核/师生导入的业务逻辑(→ M1,M9 仅入口) |
| 业务级告警规则 | 基础设施监控引擎(→ 外接 Prometheus/Grafana) |
| 平台租户管理入口 | 课程/竞赛/实验等业务(各模块) |

**核心原则**:M9 **聚合读取** 各模块数据,**绝不跨模块写**;M9 自有数据仅限运维元数据(配置/告警/统计/备份)。

## 3. 功能需求

### A. 平台管理(平台管理员,仅 SaaS)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 学校入驻审核视图 | 呈现 M1 申请列表与状态,提供跳转到 M1 对应业务入口 |
| A2 | 租户管理视图 | 全平台学校列表/状态/到期信息聚合展示,租户变更操作仍归 M1 |
| A3 | 全局看板 | 学校数/用户数/活跃度/资源用量聚合 |
| A4 | 平台统计 | 跨校趋势、使用分布 |

### B. 学校管理(学校管理员)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 本校看板 | 师生数/课程数/竞赛数/实验活跃/资源用量 |
| B2 | 本校运营统计 | 教学/竞赛/实验数据趋势 |
| B3 | 师生管理视图入口 | 呈现 M1 导入/管理能力的状态与跳转入口,不在 M9 执行业务命令 |

### C. 统一审计查询中心
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 跨模块审计查询 | 聚合查询各模块 audit_log(按人/动作/时间/对象) |
| C2 | 审计导出 | 导出审计记录 |
| C3 | 权限分级 | 平台管理员看平台级,学校管理员看本校 |

### D. 系统配置
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 全局配置 | 平台级参数(平台管理员) |
| D2 | 配置变更留痕 | 乐观锁 + 变更记录 + 生效刷新 |

### E. 告警与监控
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 业务级告警规则 | 如待审申请积压、资源配额超限、竞赛异常 |
| E2 | 告警处理 | 处理/忽略,经 M10 通知 |
| E3 | 基础监控入口 | 嵌入外接 Prometheus/Grafana 的服务健康/资源面板 |

### F. 数据备份
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 备份记录 | 备份任务记录与状态(实际备份由运维/CronJob 执行) |

## 4. 非功能需求

- **只读不跨写**:看板/审计/统计全部聚合只读;M9 写入仅限自有运维元数据,不承接下层业务命令。
- **多租户**:学校管理员看板严格限本校;平台管理员跨校(独立权限体系)。
- **私有化**:无平台管理员层;学校管理员看板即顶层。
- **监控外接**:基础设施监控不自建,外接专业体系,降低重复建设。

## 5. 明确不做(边界)

- 不做业务数据增删改(各业务模块)。
- 不自建审计表、不写审计副本;M9 自身高权限操作经统一审计能力追加到 M1 audit_log。
- 不自建基础设施监控引擎(外接 Prometheus/Grafana)。
- 不做学校审核/师生导入/租户状态变更的业务逻辑(M1 自有接口负责,M9 仅展示与跳转入口)。

## 6. 技术取向

Go 后端聚合查询;前端管理后台(React);监控外接 Prometheus/Grafana 嵌入;审计跨模块查询用统一查询服务。最终技术细节由《技术选型》统一收口。
