# M9 管理后台 — 数据模型

> PostgreSQL。M9 只拥有运维元数据;业务数据与审计日志归各模块,M9 聚合只读。
> 建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 | 范围 |
| --- | --- | --- |
| `system_config` | 系统配置(全局/租户) | 混合 |
| `config_change_log` | 配置变更留痕 | 混合 |
| `alert_rule` | 业务级告警规则 | 混合 |
| `alert_event` | 告警事件 | 混合 |
| `platform_statistics` | 统计快照(看板加速) | 混合 |
| `backup_record` | 备份任务记录 | 全局 |

> 消灭冗余/合并说明:
> - **不建审计表,统一共用 M1 的 `audit_log`**:全平台只有一张审计表(M1 定义),各模块产生关键操作时写入这张共享表(带 `tenant_id` + `action` 动作码 + `target_type` 区分来源模块)。M9 的"统一审计查询中心"是对**这张共享表的多维查询**(按 actor/action/target_type/time 过滤),不是跨多张表聚合。各模块只需在产生审计事件时写 M1 audit_log,无需各建表。
> - **看板数据不落库**:实时看板由跨模块聚合查询即时计算;仅"周期统计快照"落 `platform_statistics` 加速大盘趋势(避免每次全表扫)。
> - **告警规则与事件分两表**:规则是配置(少、稳定),事件是流水(多、增长),分开。
> - **配置全局/租户共表**:`system_config` 用 `scope` + `tenant_id(可空)` 区分全局与租户级,不拆两表。

## 2. 配置

### 2.1 system_config — 系统配置

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| scope | SMALLINT | 1全局 / 2租户 |
| tenant_id | BIGINT NULL | scope=租户时填 |
| key | VARCHAR(128) | 配置键 |
| value | JSONB | 配置值 |
| version | INT | 乐观锁版本 |
| updated_by | BIGINT | |
| updated_at | TIMESTAMPTZ | |

唯一约束:`(scope, tenant_id, key)`。

### 2.2 config_change_log — 变更留痕

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| config_id | BIGINT | |
| tenant_id | BIGINT NULL | |
| old_value / new_value | JSONB | 变更前后 |
| operator_id | BIGINT | |
| created_at | TIMESTAMPTZ | |

> **与 audit_log 的关系(口径澄清)**:配置变更**同时**写一条 audit_log(`action=config.update`,用于统一审计查询);本表是面向运维的**结构化配置回滚历史**(存完整 old/new JSONB,支持"回退到某版本配置"操作),与审计的"谁做了什么"用途不同。这是"全平台仅 M1 audit_log 一张审计表"铁律的显式例外——本表不是审计表,是配置版本表。若不需要配置回滚功能,可去掉本表仅留 audit_log。

## 4. 统计与备份

### 4.1 platform_statistics — 统计快照

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| scope | SMALLINT | 1平台 / 2租户 |
| tenant_id | BIGINT NULL | |
| stat_date | DATE | 统计日期 |
| metrics | JSONB | 各项指标快照(用户数/课程数/活跃/资源等) |
| created_at | TIMESTAMPTZ | |

唯一约束:`(scope, tenant_id, stat_date)`。

> 周期任务生成快照,加速大盘趋势查询;实时数据仍走聚合查询。

### 4.2 backup_record — 备份记录

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| type | SMALLINT | 1全量 |
| storage_ref | VARCHAR(255) | 备份清单对象引用;后台不向前端暴露 |
| size_bytes | BIGINT | |
| status | SMALLINT | 1进行中/2成功/3失败 |
| started_at / finished_at | TIMESTAMPTZ | |

> 当前备份能力只实现可恢复的每日全量备份。`storage_ref` 指向备份桶中的 `manifest.json`,清单记录 PostgreSQL dump 和 MinIO 业务对象桶复制前缀;不保留其他未实现备份类型枚举。

## 5. 聚合查询(无表,说明)

- **统一审计查询中心**:查询全平台共享的 `audit_log`(M1 定义的单表,各模块写入),按 actor/action/target_type/time 多维过滤,M9 不存审计副本。
- **运营看板**:跨模块只读聚合(M1 用户、M6 课程、M7 实验、M8 竞赛、M2 资源用量),即时计算 + 统计快照加速。

## 6. 多租户与索引

- 配置/告警/统计的租户级数据带 `tenant_id` + RLS;全局级(scope=1)仅平台管理员可访问。
- 私有化:无平台级数据,均为本租户。
- 关键索引:
  - `system_config(scope, tenant_id, key)` UNIQUE
  - `alert_event(tenant_id, status, triggered_at)`
  - `platform_statistics(scope, tenant_id, stat_date)` UNIQUE
