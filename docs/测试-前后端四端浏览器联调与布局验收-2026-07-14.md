# 前后端四端浏览器联调与布局验收（2026-07-14）

> 测试时间：2026-07-14 至 2026-07-15（Asia/Shanghai）<br>
> 测试方式：Playwright 浏览器 MCP 模拟真实用户点击、输入、选择、上传、下载、键盘与窗口缩放<br>
> 结论：**不通过，不满足当前项目文档的完整验收要求**<br>
> 修复范围：本轮只测试和诊断，**未修改任何前后端业务代码**

## 1. 结论摘要

本轮覆盖公共身份流程、学生端、教师端、学校管理员端、平台管理员端的全部已注册前端路由，并在存在可操作数据时执行新增、编辑、发布、审批、驳回、导入、导出、自检、配置、状态流转等真实写操作。共记录 **38 个独立问题：P0 16 个、P1 17 个、P2 5 个**。

当前版本不能进入修复后验收，主要阻断如下：

1. JavaScript `number` 破坏雪花 ID，找回密码、课程深页、平台配额、运行时详情/更新和判题器嵌套 ID 等场景发生 ID 改写。
2. 学生实验和竞赛入口被后端权限拒绝，PBFT 内置流程在第 7 步中断。
3. 教师实验编排、教师监控详情、平台运行时详情因 Checkbox 错误整页崩溃。
4. 成绩规则、权重、审批和默认学期/等级切换存在数据正确性或状态流转错误。
5. 入驻通过未初始化租户沙箱配额，新增租户会使平台宏观大盘整体失败。
6. 平台沙箱工具、漏洞题源存在前后端角色权限不一致。
7. 访问令牌到期后没有静默刷新，默认约 15 分钟后四端持续操作被中断。

因此，本报告中的“通过”只表示该具体页面动作在本地测试数据下成功，不代表整端达到发布或文档验收标准。

## 3. 环境与边界

| 项目 | 本轮配置 |
| --- | --- |
| 前端 | 本地源码，`http://127.0.0.1:4173`（Vite 实际服务 5173，经本地代理访问） |
| 后端 | 本地 Go 服务，`http://127.0.0.1:8080` |
| 依赖 | 独立 Docker PostgreSQL、Redis、NATS、MinIO、ClamAV；SMS mock `18888` |
| 数据 | 清库迁移后使用项目验收种子，并通过页面创建额外账号、组织、配置、申请、租户和资源记录 |
| K8s/Harbor | 本地依赖集群多数运行时镜像 `ErrImagePull`，涉及真实 Pod/镜像拉取的自检失败单独归类为环境限制 |
| 会话 | 为避免 15 分钟令牌缺陷反复打断全量测试，后端仅本地启动时将 `JWT_ACCESS_TTL_MIN` 临时调为 480；产品缺陷仍以默认 TTL 单独复现并记录 |

业务操作全部通过页面元素完成。终端只用于启动/停止服务、接收外部 SMS mock、读取日志/数据库诊断证据和整理测试文件；没有用 `curl`、脚本或浏览器 JavaScript 绕过页面直接调用业务 API。

## 9. 前端布局与规范符合度

| 规范项 | 实测结论 | 说明 |
| --- | --- | --- |
| 四端登录直达首功能页（FE-5） | 符合 | 学生/教师直达课程，校管直达用户，平台直达学校 |
| 日常深色框架 + 白底内容 | 基本符合 | 四端共享壳一致，青电主交互和琥珀状态强调可见 |

| 移动抽屉：遮罩 + Esc + 焦点 | 不符合 | Escape 不关闭；抽屉打开后切到桌面仍残留遮罩 |
| 桌面侧栏 248↔64 | 不符合 | 视觉收起但布局仍占 248 px |
| 表格/长文本重排 | 基本符合 | 375 下多数列表隐藏次要列或保持可滚动；未发现正文覆盖，但部分密集表单高度过长 |
| 错误/空/载入状态 | 部分符合 | 大部分 API 错误有用户文案和 trace_id；React 渲染异常没有错误边界，直接空白 |
| 用户文案不暴露内部契约 | 不符合 | 枚举数字、事件键、模块名、原始 JSON 多处直接展示，ISSUE-010/014/020/030/037/038 |
| 键盘可达与焦点 | 不符合 | Escape 问题明确；本轮未使用自动对比度工具做全页面 WCAG 色值审计，不对所有对比度作通过结论 |
| 无独立工作台落地页 | 符合 | 四端均使用侧栏首项作为落点 |
