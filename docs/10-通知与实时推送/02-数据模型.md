# M10 通知与实时推送 — 数据模型

> PostgreSQL + Redis。建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 | 范围 |
| --- | --- | --- |
| `notification` | 站内信(每接收者一条) | 租户 |
| `notification_template` | 通知模板 | 全局 |
| `notification_preference` | 用户接收偏好 | 租户 |
| `system_announcement` | 系统公告(不复制到每人) | 租户 |
| `announcement_read` | 公告已读状态 | 租户 |

> 消灭冗余/合并说明:
> - **系统公告不写进 notification**(旧版智慧):公告单独表 + 已读状态表跟踪,避免全校广播为每人复制一条(写放大)。
> - **未读计数不落库**:`notification.is_read` 是权威已读状态;Redis 只缓存实时未读数,缓存 miss 时按 `notification` 重建,不在表里冗余存计数。
> - **实时推送无持久表**:WebSocket 推送是瞬时的,在线状态/订阅在 Redis,不建表。
> - **模板全局共享**:`notification_template` 平台级,各租户复用。

## 2. 站内信

### 2.1 notification — 站内信

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| receiver_id | BIGINT | 接收者 |
| type | VARCHAR(64) | 事件类型(account/assignment/contest/grade/…) |
| title | VARCHAR(255) | 渲染后标题 |
| content | TEXT | 渲染后内容 |
| link | VARCHAR(255) NULL | 跳转链接 |
| is_read | BOOLEAN | 已读 |
| read_at | TIMESTAMPTZ NULL | 阅读时间 |
| created_at | TIMESTAMPTZ | |
| deleted_at | TIMESTAMPTZ NULL | 用户删除时间(软删) |

> 定期清理(如 180 天)。用户删除为软删,不物理删除业务数据。强制类(系统/成绩)不受偏好屏蔽。

### 2.2 notification_template — 模板

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| type | VARCHAR(64) UNIQUE | 事件类型 |
| title_tpl | VARCHAR(255) | 标题模板(模板变量) |
| content_tpl | TEXT | 内容模板 |
| channels | TEXT[] | 启用通道(站内信/邮件/短信) |
| force | BOOLEAN | 是否强制(不可屏蔽) |

### 2.3 notification_preference — 接收偏好

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| account_id | BIGINT | |
| type | VARCHAR(64) | 事件类型 |
| enabled | BOOLEAN | 是否接收(强制类忽略此设置) |

唯一约束:`(account_id, type)`。

## 3. 系统公告

### 3.1 system_announcement — 公告

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT NULL | 平台级公告可空(全平台) |
| title | VARCHAR(255) | |
| content | TEXT | |
| scope | SMALLINT | 1全平台/2本校/3指定角色 |
| target_roles | SMALLINT[] NULL | 指定角色 |
| publisher_id | BIGINT | |
| published_at | TIMESTAMPTZ | |
| expire_at | TIMESTAMPTZ NULL | |

### 3.2 announcement_read — 已读状态

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| announcement_id | BIGINT | |
| account_id | BIGINT | |
| read_at | TIMESTAMPTZ | |

唯一约束:`(tenant_id, announcement_id, account_id)`。

> 公告不复制到 notification;前端按"公告列表 - 已读状态"计算未读公告。

## 4. Redis 结构(非持久表)

| 结构 | 用途 |
| --- | --- |
| 未读计数 | `unread:{account_id}` 未读缓存;miss 时按 `notification(receiver_id,is_read,deleted_at)` 重建 |
| 在线状态 | WebSocket 在线连接映射 |
| 主题订阅 | 按竞赛/实验/课程主题的订阅关系 |
| 限频 | 通知发送限频 |

---

## 5. 多租户与索引

- 租户级表 `tenant_id` + RLS;平台级公告 `tenant_id` 空 + scope 控制。
- 关键索引:
  - `notification(tenant_id, receiver_id, is_read, created_at)`
  - `notification_template(type)` UNIQUE
  - `announcement_read(tenant_id, announcement_id, account_id)` UNIQUE
  - `system_announcement(tenant_id, scope, published_at)`
