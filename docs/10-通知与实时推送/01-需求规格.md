# M10 通知与实时推送 — 需求规格

> 模块定位:全平台通知与实时通道底座,被所有模块复用。
> 核心理念:统一接口、各模块不碰通知表;事件总线解耦;实时与异步分流。
> 最后更新:2026-05-29

---

## 1. 模块目标

- **异步站内信**:统一通知服务,各模块只调 `Send(事件)`,M10 负责模板/多通道/未读计数。
- **实时推送**:WebSocket Hub,服务排行榜、对局状态、沙箱进度、仿真状态、告警。
- **系统公告**:全校广播高效下发,不写放大。
- **事件总线解耦**:模块产生事件 → 总线 → M10 消费投递,发送方不阻塞。

## 2. 职责边界

| M10 负责 | M10 不负责(归属) |
| --- | --- |
| 统一通知服务(站内信) | 业务数据产生(各模块) |
| WebSocket 实时通道底座 | 实时业务数据内容(各模块产生) |
| 通知模板、未读计数、偏好 | 业务逻辑(各模块) |
| 系统公告下发 | — |
| 事件总线消费投递 | — |

## 3. 两类能力

### 3.1 异步站内信(非实时)
跨模块事件示例(约 18 类):账号开通、作业发布/截止、实验超时、竞赛报名/开赛、成绩审核、学业预警、系统维护/告警等。
- 各模块经统一接口 `Send(事件)` 触发,不直接写通知表。

### 3.2 实时推送(WebSocket)
| 来源 | 推送内容 |
| --- | --- |
| M8 | 竞赛排行榜、对局状态、天梯更新 |
| M2 | 沙箱启动分阶段进度 |
| M4 | 后端计算仿真状态 |
| M9 | 业务告警 |
| M6 | 课程实时消息(可选) |

> M10 提供通道,业务数据由各模块产生并经 Hub 推送。

## 4. 功能需求

### A. 统一通知服务
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | Send 接口 | 各模块统一调用,传事件类型 + 接收者 + 参数 |
| A2 | 模板渲染 | 模板变量渲染,按事件类型生成标题与正文 |
| A3 | 多通道 | 站内信(必备),邮件/短信留扩展 |
| A4 | 未读计数 | Redis 缓存未读数,缓存 miss 时按 `notification.is_read` 重建并实时推送红点 |
| A5 | 接收偏好 | 用户可配置接收偏好(系统/成绩类强制) |

### B. 站内信
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 收件箱 | 列表/已读/未读/删除 |
| B2 | 标记已读 | 单条/批量 |
| B3 | 清理 | 过期(如 180 天)自动清理 |

### C. 系统公告
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 发布公告 | 平台/学校管理员发布,不为每人写记录 |
| C2 | 已读跟踪 | 公告表 + 已读状态表,避免全员广播写放大 |

### D. 实时通道
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | WebSocket Hub | 统一连接管理、鉴权、订阅主题 |
| D2 | 主题订阅 | 按竞赛/实验/课程等主题订阅推送 |
| D3 | 连接管理 | 心跳、断线重连、单端登录联动(M1) |

### E. 事件总线
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 事件消费 | 订阅 NATS 事件,转通知/推送 |
| E2 | 解耦 | 发送方发事件即返回,不阻塞 |

## 5. 非功能需求

- **不写放大**:系统公告单独表,不为每接收者复制。
- **高并发**:竞赛排行榜实时推送支撑大量并发连接。
- **多租户**:通知/公告带 tenant_id 隔离;实时主题按租户隔离。
- **解耦**:各模块只发事件/调 Send,不依赖 M10 内部实现。

## 6. 明确不做(边界)

- 不产生业务数据(各模块产生,M10 只投递)。
- 不实现各模块业务逻辑。
- 邮件/短信通道首版可只留扩展点(站内信必备)。

## 7. 技术取向

Go + WebSocket(Hub)+ NATS(事件总线)+ Redis(未读计数/在线状态)。最终技术细节由《技术选型》统一收口。
