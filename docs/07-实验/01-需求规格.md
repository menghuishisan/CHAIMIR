# M7 实验 — 需求规格

> 模块定位:实验业务编排层。把 M2 沙箱、M3 评测、M4 仿真、M5 模板组装成完整实验业务,是引擎能力的综合出口。
> 核心理念:业务薄、引擎厚。M7 只编排,不含任何引擎能力。
> 最后更新:2026-05-29

---

## 1. 模块目标

- 教师**配置实验**:向导式组装(M5 模板 + M2 运行时工具 + M4 仿真 + M3 检查点 + 教学说明)并发布。
- 学生**做实验**:发起实例 → 引擎拉起环境/仿真 → 操作 → 检查点判分 → 报告 → 成绩。
- **组件自由组合**:一个实验 = 任意数量的沙箱环境 + 仿真 + 检查点,不预设固定类型。
- **多人协作**:小组共享一组环境,各扮角色,链上交互。
- 实验成绩由 M7 持久化并发布 `experiment.scored` 事件;M7 不直接调用同层 M6。

## 2. 职责边界

| M7 负责 | M7 不负责(归属) |
| --- | --- |
| 实验配置(组装引擎能力) | 沙箱起停/隔离(→ M2) |
| 实验实例生命周期编排 | 判题执行(→ M3) |
| 学生做实验流程 | 仿真运行/渲染(→ M4) |
| 多人协作分组/角色 | 实验模板内容存储(→ M5) |
| 检查点结果汇总、实验报告 | 课程总评聚合(→ M6/M11 后续消费事件或上层编排) |
| 实验进度与得分 | 课程/课时(→ M6,只存引用) |

**核心交互**:M7 实例化实验时,调 M2 起沙箱、调 M4 起仿真、传 M5 模板配置;学生操作后调 M3 判检查点;汇总得分后保存并发布事件,不直接依赖同层 M6。

## 3. 组件自由组合(替代旧版固定类型)

旧版硬分纯仿真/真实/混合(type=1/2/3)。M7 改为按组件组合:

```
一个实验 = [0..N 个沙箱环境(M2 运行时+工具集)]
         + [0..N 个仿真(M4 仿真包)]
         + [0..N 个检查点(M3 判题)]
         + 教学说明 + 实验报告要求
```

- 纯仿真实验 = 只含仿真组件;真实环境实验 = 只含沙箱组件;混合 = 都有。无需预设类型,教师配什么即什么。

## 4. 功能需求

### A. 实验配置(教师,向导式)
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 基于模板创建 | 从 M5 选实验模板作为起点,或从空白配 |
| A2 | 环境组件 | 选 M2 运行时 + 工具集(IDE/Remix/终端/浏览器),设初始代码/部署脚本。单个环境本身由 M2 编排为**多容器 Pod 组**(如联盟链 = peer+orderer+CA+CouchDB+工具),教师只选运行时,多容器编排由运行时 manifest 自动展开 |
| A3 | 仿真组件 | 选 M4 仿真包 + 参数 |
| A4 | 检查点组件 | 配 M3 判题(测试用例/链上断言/flag/仿真检查点),设分值 |
| A5 | 说明与报告 | 实验说明、是否要求提交报告 |
| A6 | 协作设置 | 单人 / 小组(组大小、角色定义) |
| A7 | 发布前校验 | 依赖完整性、组件可用性、分值合理性 |
| A8 | 发布 | 生成可被课程引用的实验 |
| A9 | 向导草稿持久化 | 编排向导每步配置自动存服务端草稿,刷新/重开/换设备回到原步骤不丢(见总纲约定·多步骤持久化) |

### B. 学生做实验
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 发起实例 | 创建实验实例;M7 调引擎拉起环境/仿真 |
| B2 | 实验工作台 | 展示沙箱工具(M2)、仿真(M4)、说明、检查点状态 |
| B3 | 操作交互 | 在沙箱写代码/部署、在仿真注入操作 |
| B4 | 检查点判分 | 触发/自动判检查点(调 M3),展示可解释结果 |
| B5 | 提交报告 | 上传实验报告(可选) |
| B6 | 完成 | 提交实验,汇总得分 |

### C. 多人协作
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 分组 | 教师分组或学生组队 |
| C2 | 角色 | 组内各成员扮演角色/节点(如联盟链各机构、攻防双方) |
| C3 | 共享环境 | 小组共享一组沙箱(M2),通过链上状态交互 |
| C4 | 不做实时协同编辑 | 各自操作入口,不做光标/同步打字 |

### D. 实例生命周期
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 状态流转 | 创建/进行/暂停/完成/回收(对接 M2 沙箱生命周期) |
| D2 | 断点续做 | 复用 M2 代码持久化 + 重建 |
| D3 | 回收 | 完成/超时回收;课程结束级联(通知 M2) |

### E. 结果与成绩
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 检查点汇总 | 各检查点得分汇总 |
| E2 | 报告批改 | 教师批改报告(主观分) |
| E3 | 实验得分 | 检查点 + 报告 → 实验总分 |
| E4 | 发布得分事件 | 发布 `experiment.scored`,供后续课程成绩或聚合流程按事件/上层编排消费 |

## 5. 非功能需求

- **业务薄**:不实现任何引擎能力,全部调用 M2/M3/M4/M5。
- **多租户**:实验定义与实例带 tenant_id 隔离。
- **可复现**:实验引用 M5 模板锁版本、M4 仿真锁版本。
- **秒进**:依赖 M2 分层启动,学生发起实验快速可用。

## 6. 明确不做(边界)

- 不做沙箱/判题/仿真/模板内容(全归引擎)。
- 不做实时协同编辑。
- 不算课程总评/GPA(M6/M11)。

## 7. 技术取向

Go 后端编排引擎调用;前端实验工作台(React)集成 M2 工具接入与 M4 仿真渲染。最终技术细节由《技术选型》统一收口。
