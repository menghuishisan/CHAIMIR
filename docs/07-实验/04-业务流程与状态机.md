## 2. 学生做实验编排流程

```mermaid
sequenceDiagram
    participant S as 学生
    participant M7 as 实验
    participant M2 as 沙箱
    participant M4 as 仿真
    participant M3 as 评测
    S->>M7: 发起实例
    M7->>M7: 读 components
    par 并行拉起组件
        M7->>M2: 起沙箱(运行时+工具+初始代码)
        M7->>M4: 起仿真(仿真包+参数)
    end
    M2-->>M7: 沙箱就绪(分阶段进度)
    M4-->>M7: 仿真就绪
    M7-->>S: 工作台(沙箱工具+仿真+检查点)
    S->>S: 写合约/部署 / 仿真注入操作
    S->>M7: 触发检查点
    M7->>M3: 判题(M5配置+沙箱/仿真状态)
    M3-->>M7: 结果(可解释)
    S->>M7: 提交报告 + 完成
    M7->>M7: 汇总得分并发布 experiment.scored
    M7->>M2: 回收沙箱
    M7->>M4: 回收仿真
```

> 发布前校验先做结构、分值与 M5 锁定版本校验;M2 沙箱、M4 仿真、M3 判题器的运行可用性由创建实例和触发检查点时的引擎契约实时确认。实例创建时生成并持久化 `experiment_instance.source_ref`,M2/M4 创建、恢复重建、手动回收、错误补偿与 M3 检查点判题均使用同一 `source_ref`。若任一引擎创建失败,M7 将实例置为错误态并按该 `source_ref` 补偿回收已成功创建的组件。

## 3. 实验实例状态机

```mermaid
stateDiagram-v2
    [*] --> 创建中: 发起实例
    创建中 --> 进行中: 引擎组件就绪
    创建中 --> 错误: 组件拉起失败
    进行中 --> 已暂停: 暂停
    已暂停 --> 进行中: 恢复(探活,沙箱已回收则重建)
    已暂停 --> 已回收: 暂停超时
    进行中 --> 已完成: 学生完成/提交
    进行中 --> 已回收: 空闲超时/课程结束级联
    已暂停 --> 环境已释放: M2 沙箱被独立回收(收到 sandbox.recycled 事件)
    环境已释放 --> 进行中: 学生恢复 → 重建环境
    已完成 --> 已回收: 环境回收(结果保留)
    错误 --> 已回收: 清理(补偿回收已起组件)
    已回收 --> [*]
```

- 回收只释放引擎资源(M2/M4),实验结果(检查点/报告/得分)保留。
- **断点续做与生命周期同步**(A4 修复):M2 沙箱被独立回收时发布 `sandbox.recycled` 事件,experiment 订阅后将实例转"环境已释放"(逻辑态,持有代码持久化引用);学生恢复时 experiment 使用实例持久化 `source_ref` 重建环境(M2 拉回代码 + 重放部署脚本)再进入"进行中"。避免持失效 sandbox_ref 或跨年重算来源引用。用事件解耦,sandbox 不反向依赖 experiment(见工程目录设计 §3.1.1)。
- **暂停超时回收**:已暂停长时间不恢复则回收,防僵尸。
- **错误态补偿回收**(B7 修复):组件部分拉起失败(如 M2 成功但 M4 失败)时,错误态清理需**补偿回收已成功起的另一组件**(调 M2/M4 回收),不残留半截资源。

## 4. 多人协作流程

```mermaid
flowchart TD
    A[教师设小组实验 + 角色定义] --> B[分组: 教师分/学生组队]
    B --> C[组员各分配角色]
    C --> D[组内发起共享实例]
    D --> E[M7 调 M2 起一组多节点环境]
    E --> F[各成员按角色获得操作入口]
    F --> G[各自操作本节点,链上状态共享]
    G --> H[通过链上交互完成协作任务]
    H --> I[检查点判分: 可针对组或个人]
```

> 不做实时协同编辑;协作通过共享链状态实现。
