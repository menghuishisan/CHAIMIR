# M7 实验 — 数据模型

> PostgreSQL,全部租户级(tenant_id + RLS)。建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 |
| --- | --- |
| `experiment` | 实验定义(组件编排 + 说明 + 协作设置) |
| `experiment_instance` | 实验实例(一次做实验) |
| `experiment_group` | 多人协作小组 |
| `group_member` | 小组成员 + 角色 |
| `checkpoint_result` | 检查点判分结果 |
| `experiment_report` | 实验报告 |

> 消灭冗余/合并说明:
> - **组件编排(沙箱/仿真/检查点列表)以 JSONB 存进 `experiment.components`**,不为每类组件单建表——组件是配置而非独立实体,且引用的内容正本在 M5/M2/M4。
> - **实验模板内容不拷贝**:`experiment` 只存 M5 模板引用 + 课程内编排差异,题目/判题配置正本在 M5。
> - **单人实验无小组**:`experiment_group`/`group_member` 仅多人实验用;单人实例 `group_id` 为空,不强制建组。
> - **检查点定义在 components 里**,`checkpoint_result` 只存某实例某检查点的判分结果(引用 M3 判题任务),不重复存检查点定义。

## 2. 实验定义

### 2.1 experiment — 实验定义

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| course_id | BIGINT NULL | 所属课程(独立实验任务可空) |
| author_id | BIGINT | 教师 |
| template_ref | VARCHAR(96) NULL | M5 实验模板引用(锁版本) |
| template_version | VARCHAR(32) NULL | |
| name | VARCHAR(255) | |
| description | TEXT | 实验说明 |
| components | JSONB | 组件编排:环境(M2运行时+工具+初始代码)/仿真(M4包+参数)/检查点(M3判题+分值)列表 |
| collab_mode | SMALLINT | 1单人 / 2小组 |
| group_config | JSONB NULL | 小组配置:组大小、角色定义 |
| require_report | BOOLEAN | 是否要求实验报告 |
| wizard_step | SMALLINT | 向导当前步骤(草稿自动保存/恢复) |
| status | SMALLINT | 1草稿 / 2已发布 / 3已下架 |
| created_at / updated_at | TIMESTAMPTZ | |

> `components` 示例结构:
> ```json
> { "envs": [{ "runtime_code": "evm-hardhat", "tools": ["code-server","terminal"], "init_code_ref": "..." }],
>   "sims": [{ "package_code": "builtin__pow-mining", "version": "1.0.0", "params": {...} }],
>   "checkpoints": [{ "id":"cp1", "judger":"testcase", "item_code":"prob-2031", "item_version":"1.0.0", "score":40 }] }
> ```

## 5. 多租户与索引

- 全部表 `tenant_id` + RLS(同 M1)。
- 关键索引:
  - `experiment(tenant_id, course_id, status)`、`experiment(author_id)`
  - `experiment_instance(tenant_id, experiment_id, owner_account_id)`、`experiment_instance(status, last_active_at)`(回收扫描)
  - `experiment_instance(tenant_id, source_ref)` UNIQUE
  - `experiment_instance(group_id)`
  - `group_member(group_id, student_id)` UNIQUE
  - `checkpoint_result(instance_id)`、`experiment_report(instance_id, student_id)`
