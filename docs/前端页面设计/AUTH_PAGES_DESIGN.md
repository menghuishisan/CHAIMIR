# 认证与公共页 (Auth & Public Pages) 全局页面设计规范

严格遵循无 Emoji、全令牌化、“青电×琥珀”双色规范。
**说明**：这是《前端后端功能对齐清单》中指定的“公共登录前页面”，所有认证相关页面均不包含普通的顶栏和侧边栏，而是采用全屏沉浸式的**左右分栏布局**（左侧品牌宣传/右侧操作表单）或**全屏居中卡片布局**。

所有页面同时遵循本目录 `README.md` 的公共契约。普通学校用户认证表单保持浅色内容表面；仅平台管理员专属登录使用受控深色表面，这不代表全站暗色主题。

**实现基准**：
1. 本文是 UI/UX 视觉与信息架构草图，不是后端契约来源。实现时以 `docs/前端后端功能对齐清单.md`、后端公开 API/DTO/枚举和 `frontend/packages/api-client` 为准。
2. 草图中的示例字段、状态、文案和布局可在实现时优化；凡与后端已实现能力冲突的，以后端实现为准。
3. 不得用模拟数据替代已有后端功能；没有后端契约的能力不得实现为可操作功能。
4. 最终界面不得使用 emoji，草图中的状态必须落地为 Lucide 图标 + `@chaimir/ui` 状态组件 + 文本文案。

## 全局布局模板 (Split 布局)
```text
======================================================================================================
[左侧: 品牌视觉展示区 (占宽 50% ~ 60%, 仅桌面端显示)]     | [右侧: 表单操作区 (占宽 40% ~ 50%, 浅色内容表面)]
[背景: 深色系青电色网格渐变与 3D 几何图形]              |
                                                        | [Logo] Chaimir 区块链教学平台
[Slogan H1] 虚实相生，铸造安全之盾                      |
[Slogan P] 下一代区块链与网络安全实训靶场               | (各页面的表单区见下文)
                                                        |
======================================================================================================
```

### 6. 统一认证中间页 (`/auth/sso`)
```text
[全屏居中浅色卡片]
------------------------------------------------------------------------------------------------------
[ 居中卡片 ]
[ Tabs: 网页统一认证 (CAS) | 学校账号认证 (LDAP) ]
[ CAS: 学校代号输入框 -> 获取后端跳转地址 -> 浏览器跳转 -> 回调换取登录会话。]

[ 加载状态使用设计系统 Spinner；失败展示用户向 message，并在可用时附 trace_id。]
```

### 8. 首次登录强制改密 (`/auth/change-pwd`)
```text
[右侧表单操作区]
------------------------------------------------------------------------------------------------------
[H1] 出于安全要求，请修改密码
[说明: 后端登录响应 `must_change_pwd` 时展示本拦截态。改密调用公开的个人密码接口 `POST /me/password`，当前会话只允许改密或退出，不能进入其它业务页。]

[ 当前密码输入框 ]
[ 新密码输入框；至少 8 位且同时包含字母和数字 ]

[ 完成修改并进入系统 (青电按钮) ] [ 退出并重新登录 ]
[ 若会话失效: 展示用户向错误文案 + trace_id，并引导返回登录。]
```
