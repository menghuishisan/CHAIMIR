# 前端页面设计公共契约

本文定义 `docs/前端页面设计/` 下五份角色页面草图共同遵守的实现边界。页面草图用于表达信息架构和交互意图，不覆盖后端契约，也不单独创造产品能力。

2026-07-22 的现状证据、问题优先级、104 个页面位置重构矩阵与验收顺序见 `docs/前端UIUX全面重构审计-2026-07-22.md`;共享组件、PageScaffold、WorkbenchShell 与 SimulationWorkbench 的目标设计见 `docs/前端共享UI与仿真工作台目标设计-2026-07-22.md`。两份文档共同组成当前重构执行基线,不改变下述权威顺序,也不新增后端能力。

## 1. 权威顺序

实现与文档冲突时，按以下顺序判断：

1. `docs/前端后端功能对齐清单.md`：页面范围、入口类型和后端能力边界。
2. 后端公开 API、DTO、枚举与 `frontend/packages/api-client`：字段、状态、权限和错误契约。
3. `docs/总-前端设计规范.md`：令牌、组件、布局、动效、响应式和无障碍的单一真相源。
4. 本目录角色页面草图：信息层级、页面结构与用户任务说明。

草图中的示例数据只用于说明布局。没有公开后端契约的能力不得渲染为可点击按钮、可提交表单或虚假进度；可预留的内容必须明确写入“明确不做”，不能用模拟成功替代。

## 2. 全局应用壳

- 日常页面采用深色顶栏和侧栏框架、浅色内容区；实验、仿真、答题和回放使用深色全屏工作台。
- 当前阶段不提供独立全站暗色主题。深色表面只能使用设计系统定义的受控深色表面令牌，不得通过通用主题属性把普通内容区整体换肤。
- 顶栏不提供跨业务资源的全局搜索。课程、用户、竞赛等资源搜索留在对应页面，由真实列表接口的查询能力驱动。
- 顶栏固定提供任务中心、通知中心和个人菜单；帮助入口只有在存在真实帮助内容或支持渠道时才展示。
- 通知、任务和个人中心属于顶栏深层页，不占侧栏主导航位置。
- SaaS 部署提供平台管理员入口与 `/platform-admin` 路由；学校私有化部署不展示也不注册平台管理员入口。

## 3. 页面状态与错误

每个数据页面必须覆盖加载、空、错误、无权限和正常状态。错误优先展示后端用户向 `message`；存在 `trace_id` 时附“如需帮助，请提供编号”，不得显示堆栈、字段名、内部错误码或第三方原始错误。

列表筛选、分页和排序必须与后端查询参数一致。会触发导出、生成、扫描或长任务的动作使用服务端任务状态，不在浏览器中伪造进度。

## 4. 表单与多步骤流程

- 每个输入控件必须有可见标签，placeholder 只提供示例，不能代替字段名称。
- 提交前做必要的格式和完整性校验，服务端仍是最终校验权威。
- 入驻申请、实验编排、作业草稿、竞赛报名/出题和导入预览等多步骤流程，中间状态必须持久化到服务端；当前接口没有草稿契约时，不得擅自改成纯前端多步骤向导。
- 密钥和密码字段按写入型敏感值处理：默认不回显明文，读取接口只显示是否已配置或掩码摘要。

## 5. 响应式与无障碍验收

- 必测宽度为 375、768、1024、1440 像素，并补充任意拖拽缩放检查。
- 页面不得出现非预期横向滚动；表格可以在独立容器中横滚，但窄屏优先隐藏次要列或转键值卡片。
- 触屏交互目标不小于 44×44 像素；键盘可到达所有操作，焦点可见，弹层支持 Esc 和焦点管理。
- 正文和控件文字达到 WCAG AA；颜色不是状态的唯一表达方式。
- 所有非必要动画遵循 `prefers-reduced-motion`，持续时间和缓动使用设计令牌；全屏遮罩不使用高成本毛玻璃模糊。

## 6. 文件与组件边界

- 页面业务代码放在 `frontend/apps/web/src/features/<module>/pages/`；应用装配放 `app/`、`routes/`、`layouts/`。
- HTTP/WS 契约统一经 `@chaimir/api-client`，页面不得散落硬编码接口路径。
- 通用视觉和交互原语统一来自 `@chaimir/ui`，图标统一使用 Lucide，组件样式只引用设计令牌。

## 7. 已实现共享工作台边界

- 实验与竞赛答题页只组合 `SandboxIdeWorkspace`，由它连接 M2 沙箱并复用 `@chaimir/ide`。角色草图不得再设计另一套文件树、编辑器、终端或三栏工作台壳。
- 仿真页只组合 `@chaimir/sim-sdk` 的 `SimulationWorkbench`，页面草图可以定义分享、回放和退出入口，但不得重新实现仿真画布、时间线、播放控制或检查器。分享结果使用无角色守卫的公共沉浸深链；前端计算、后端计算和只读回放仍由同一工作台按服务端 `compute` 与会话状态切换，不派生第二套布局。
- `@chaimir/ui/WorkbenchShell` 是沉浸式结构原语，不是完整业务页面。业务步骤、检查点、提交、排行和实例生命周期由各 feature 页面通过插槽注入。
- 日常页面与沉浸式页面不得保留同一能力的两套旧布局 CSS；迁移到共享层后同步删除不再被组件引用的页面级工作台样式。
