# Apps 路由与状态契约

本文件补充四端应用层的路由归属、共享入口和服务端数据边界。页面设计文档只描述视觉与交互方向；本文件明确实现时必须遵守的结构。

## 路由归属

| 类型 | 入口 | 外壳 | 数据来源 |
| --- | --- | --- | --- |
| 日常功能 | `/student/*`、`/teacher/*`、`/school-admin/*`、`/platform-admin/*` | `MainLayout` | `@chaimir/api-client` 公开接口 |
| 实验、仿真、竞赛操作 | `*/experiments/*/workspace`、`*/simulations/*/workspace`、`*/contests/*/{workspace,replay}` | `ImmersiveLayout` + 对应工作台 | API、授权 WebSocket、工作台包 |
| 跨角色共享 | `*/notifications`、`*/tasks`、`*/profile` | `MainLayout` | notify、transfer、identity 公开接口 |
| 分享仿真 | `/sim/shared/:shareCode` | 独立沉浸入口 | 分享码公开读取接口 |

通知、任务、个人中心属于顶栏共享入口，不重复加入角色侧栏。实验、仿真和竞赛工作台不在日常页面中复制编辑器或渲染器。

## 页面状态

所有 API 页面必须分别处理 `loading`、`error`、`empty`、`forbidden` 和 `ready`。`empty` 只由服务端返回的空集合触发；禁止使用固定百分比、固定数量或本地计时器伪造进度。工作台的阶段、资源、得分和运行状态只展示服务端快照或授权流事件。

## 数据边界

- 页面只读取 `@chaimir/api-client` 导出的公开 DTO；不得拼接 `/internal` 路径。
- 对象引用、学生编号、实例编号只作为请求参数或必要的辅助信息，不能替代用户可读标题、姓名或状态。
- 实验报告正文只通过 M7 的教师预览和一次性下载授权读取,页面不得读取对象存储引用或拼接直链。
- 学生实验和竞赛详情使用各自学生单条公开接口,不得调用教师接口或从分页列表反查。课程实验必须先读取 M6 课时取得短时启动授权,再读取 M7 详情或创建实例;独立实验授权由 M7 学生详情签发。
- 后端未提供的数据不得在前端计算成业务结论；需要新增能力时先补充后端 API 与本文件，再实现页面。
- M7 与 M6 同属业务层,不得为课程名称或成员关系建立同步依赖。独立实验由 M7 学生列表展示;课程实验从 M6 课时进入,两者通过统一短时启动授权衔接。

## 响应式与动效

日常页面使用内容优先布局：主内容不套固定 KPI 卡片墙，表格在窄屏转为可横向滚动或优先级列。沉浸工作台保持 100dvh，移动端检查器改为抽屉。交互动效只用于状态变化和空间关系，遵守 `prefers-reduced-motion`，不为频繁键盘操作添加动画。
