# 验收标准总览

> 按 11 模块分章,每模块列可验证的验收标准(做完什么算对)。配合各模块"安全设计"的安全测试要点共同验收。
> 验收维度:功能正确 / 边界与异常 / 安全 / 多租户隔离 / 性能(关键项)。
> 最后更新:2026-06-04

---

## 通用验收(所有模块)

- [ ] 多租户隔离:跨租户访问任何数据被 RLS 拦截(渗透用例验证)。
- [ ] 私有化同构:单租户部署逻辑一致,无 `/platform/*`,首个学校管理员由初始化脚本创建。
- [ ] 统一响应体/错误码段符合《API 接口总览》规范。
- [ ] 关键操作写入 M1 共享 audit_log。
- [ ] 软删而非物理删;归档数据可恢复。

---

## M1 身份与租户

- [ ] 学校入驻:申请→平台管理员审核→自动建租户+学校管理员(激活码),驳回有原因通知。
- [ ] 禁自助注册;师生仅由学校管理员导入(Excel 预览+部分失败逐行报错+提交)。
- [ ] 手机号必填登录 + 学号备用登录 + 短信验证码;一号多校返回选择租户。
- [ ] 双 Token 无感刷新;Refresh 轮转;**单端登录**(新登录踢旧会话,旧 Token 立即失效)。
- [ ] SSO(CAS/LDAP)登录:名单先导入,SSO 仅核验,匹配才放行。
- [ ] 四级 RBAC × 数据范围;教师可兼学校管理员;学生/教师身份互斥。
- [ ] 密码 argon2id;登录失败锁定;手机号加密存储 + 日志脱敏。
- [ ] 账号状态机流转正确(待激活/正常/停用/归档/注销);按学年归档与班级升级。

## M2 沙箱引擎

- [ ] 按链类型+工具集创建隔离沙箱(独立 Namespace + deny-all NetworkPolicy)。
- [ ] **启动性能**:简单实验环境就绪 ≤5s,复杂多节点 ≤15s;分阶段进度反馈。
- [ ] 两阶段启动:环境就绪即可进入,个性化初始化后台异步(按钮置灰→解灰)。
- [ ] 运行时三层适配器;新链加 manifest 即接入;**接入即测**(起链→部署→查询→重置)全绿才可用。
- [ ] 镜像预拉取真实生效:DaemonSet 覆盖全部目标沙箱节点,失败不写成功标记,新沙箱不因拉镜像阻塞启动。
- [ ] 工具可插拔:终端(K8s exec PTY)、Web 工具代理(code-server/Remix/浏览器)动态渲染工作台。
- [ ] 链上能力接口(deploy/tx/query/reset)跨运行时统一。
- [ ] 持久化:代码实时存 MinIO;默认重建(重放部署脚本);教师可选保活/快照(有上限)。
- [ ] 快照基于 CSI VolumeSnapshot,保活/快照受租户配额和过期清理约束;无快照能力环境明确拒绝开启。
- [ ] 三重回收(空闲/最长生命周期/来源级联)+ ready 超时回收 + 回收事件通知来源;无僵尸沙箱。
- [ ] 回收闭环:保存代码、按需快照、删除 K8s 资源、写 destroyed、写 audit_log、发布 `sandbox.recycled` 均可验证,失败可重试且不误报完成。
- [ ] 容器逃逸/跨沙箱网络/资源炸弹被隔离限制。

## M3 评测引擎

- [ ] 6 类判题器可插拔;判题器自检通过才可用。
- [ ] judge 专用沙箱与学生环境隔离;套件/答案对学生黑盒;用后即毁。
- [ ] `sandbox_mode`:fresh 新起干净沙箱 / reuse 复用现场沙箱(链上断言读现场)。
- [ ] 确定性:同提交多次判定结果一致;输入快照可复现;可重判。
- [ ] 可解释结果:逐用例/断言期望 vs 实际 + 提示。
- [ ] 异步优先级队列;超时/失败重试;**重试耗尽进 failed 终态并发布 `judge.failed` 事件**(不死锁)。
- [ ] 提交级防作弊:限频、指纹/相似度查重能力(判定归 M8)。

## M4 仿真可视化引擎

- [ ] 自描述仿真插件加载运行;前端确定性状态机演化。
- [ ] 通用交互:任意 kind(按钮/滑块/按住/选中/拖拽/表单)按声明自动渲染控件;攻击=普通交互。
- [ ] 可视化 SDK:复合组件 + 低层能力;响应式布局 + 基础无障碍(键盘可达)。
- [ ] 种子化确定性:单步回退/回放/分享操作序列一致;检查点可供 M3。
- [ ] 教学叙事:分步引导/释义/设问可选层。
- [ ] 仿真包审核(沙箱化预览 + 静态扫描 + 确定性校验);Web Worker 隔离执行,无法访问 DOM/token/网络。
- [ ] 会话状态机含 creating/failed/idle 归档;来源级联回收不悬挂。

## M5 题库与模板中心

- [ ] 三类内容统一外壳 + 类型化内容体;创建/草稿/发布/弃用。
- [ ] **答案存 M5**;取题面接口过滤敏感字段,学生拿不到答案;full 接口仅教师/内部。
- [ ] 版本化:发布定版,引用锁版本,改题发新版不影响旧引用。
- [ ] 克隆(深拷贝独立)+ 跨校共享库;**取消共享不影响已克隆副本**。
- [ ] 分类树/标签/难度/知识点检索;按条件组卷。
- [ ] system-import:M8 漏洞题固化入库(自包含)。

## M6 教学

- [ ] 课程全生命周期状态机;邀请码加入/批量添加成员。
- [ ] 章节课时多形态内容;课时关联 M7 实验(评分)/ M4 仿真(展示)路径清晰不重复。
- [ ] 作业引用 M5 题目(锁版本);客观/编程题调 M3、主观题教师批;草稿双存。
- [ ] 迟交策略;提交次数限制;编程题查重(教师据 M3 结果裁定)。
- [ ] 学习进度(视频续播/时长/完成);讨论/公告/评价。
- [ ] 单课程成绩:权重和=100% 校验、加权计算、手动调整(留痕)、导出;锁定后的申诉改分由 M11 编排调用 M6 受控内部入口。

## M7 实验

- [ ] 组件自由组合(沙箱+仿真+检查点任意组合),无固定类型。
- [ ] 向导式配置 + 发布前校验(依赖完整/组件可用/分值合理)。
- [ ] 实例编排:并行起 M2 沙箱 + M4 仿真;工作台集成工具/仿真/检查点。
- [ ] 检查点判分调 M3;报告提交批改;得分=检查点+报告,M7 保存并发布 `experiment.scored` 事件。
- [ ] 多人协作:小组共享环境 + 角色入口 + 链上交互(不做实时协同编辑)。
- [ ] 实例状态机:组件失败补偿回收已起组件;M2 回收后"环境已释放"→恢复重建;不悬挂。

## M8 竞赛

- [ ] 解题赛(含理论题型):自动判定 + 实时排行(经 M10);真实漏洞复现题型。
- [ ] 对抗赛:提交参战物→异步撮合→对局沙箱(M2)→判定→ELO 天梯;可改进重提。
- [ ] **战报回放**:对局链上交易序列完整记录可重放。
- [ ] 报名组队(个人/团队/跨校组队受控);封榜。
- [ ] 行为级防作弊:限频/冷却/查重(用 M3 能力)/行为监控/违规处理留痕。
- [ ] 真实漏洞源:多源接入 + A/B/C 分级 + isolated/forked + **导入即固化(答题零外部依赖)** + 预验证(正向+反向)。
- [ ] 竞赛结束级联回收环境 + 生成成绩快照;个人战绩档案可查;竞赛成绩不计 GPA。

## M9 管理后台

- [ ] 平台/学校两级看板(聚合只读,不跨写);严格按角色范围。
- [ ] 统一审计查询中心(查 M1 audit_log,多维过滤,按权限范围)。
- [ ] 系统配置(乐观锁 + 变更留痕);业务级告警规则/事件处理(经 M10 通知)。
- [ ] 监控外接 Prometheus/Grafana 嵌入展示(不自建监控引擎)。
- [ ] 备份记录;无任何跨模块写路径(审核/导入转发对应模块执行)。

## M10 通知与实时推送

- [ ] 统一 Send 接口:模板渲染 + 多通道 + 未读计数;各模块不直接写通知表。
- [ ] 系统公告单表 + 已读状态(不写放大)。
- [ ] WebSocket Hub:鉴权建连、主题订阅、心跳重连、单端被踢关闭。
- [ ] 业务实时数据(排行榜/进度/红点)经 push+ws;引擎内部流走各模块自有 WS。
- [ ] 事件总线消费失败重试 + 死信队列;强制类通知至少一次投递 + 站内信兜底。

## M11 成绩中心

- [ ] 跨课程聚合(只读 M6,不写);等级映射可配;GPA(学分加权)正确。
- [ ] 成绩审核工作流:提交→审核→锁定(is_locked);解锁回退。
- [ ] **申诉完整回路**:受理→解锁 M11 审核状态→M6 教师成绩入口改分→M6 发布成绩变更事件→M11 重算 GPA→重审重锁;M11 只保存自有审核与聚合状态。
- [ ] 学业预警(挂科/低 GPA,经 M10 通知);成绩单 PDF 生成/批量导出。
- [ ] 竞赛成绩不计 GPA;学生只读自己成绩。

## 工程与部署验收

- [ ] 模块化单体:模块间仅经 contracts 接口调用,无直接 import 其他模块内部包(架构守护检查)。
- [ ] 模块只读写自己的表;聚合层(M9/M11)无写路径。
- [ ] SaaS(K8s)与私有化(k3s)均可部署;私有化关闭平台层。
- [ ] 镜像 Trivy 扫描 + Cosign 签名门禁;预拉取生效。
- [ ] CI/CD:lint+测试+构建+扫描+签名+部署链路通;CronJob 兜底回收/备份。
- [ ] 私有化离线镜像包可导入并完整运行。

---

## 验收方式

- 每条标准对应可执行的测试用例(单元/集成/端到端/渗透)。
- 安全类标准配合各模块"安全设计"的"安全测试要点"执行。
- 跨模块链路(见《API 接口总览》§三)做端到端验收,确认闭环无断点。
