# M2 沙箱引擎 — 数据模型

> 数据库:PostgreSQL。沙箱实例等租户相关表带 `tenant_id` + RLS;运行时/工具等平台级配置表全局共享。
> 建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-06-04

---

## 1. 表清单总览

| 表名 | 用途 | 范围 |
| --- | --- | --- |
| `runtime` | 区块链运行时(链类型)+ 适配器清单 | 全局 |
| `runtime_image` | 运行时镜像与版本 | 全局 |
| `tool` | 可接入工具定义 | 全局 |
| `sandbox` | 沙箱实例 | 租户 |
| `sandbox_tool` | 沙箱已挂载的工具实例 | 租户 |
| `sandbox_event` | 沙箱生命周期事件/操作日志 | 租户 |
| `tenant_quota` | 租户资源配额 | 租户 |

> 说明(消灭冗余/合并有度):
> - **适配器清单(L1 YAML)直接以 JSONB 存进 `runtime.adapter_spec`**,不单设适配器表——它与运行时一对一。
> - **工具与链的兼容关系**用 `tool.eco_tags` 与 `runtime.eco` 标签匹配推导,不建中间关联表(避免 N×M 冗余表)。
> - **Pod 组与网络规则属于 `runtime.adapter_spec` 的执行清单**,不单设拓扑表。`pods` 描述本运行时需要的工作负载 Pod 与容器组;`network_rules` 描述同一沙箱内不同 Pod 之间的最小互通端口,由 M2 转换为 NetworkPolicy。未声明网络规则时,即使在同一 Namespace 内也默认不可互访。
> - 预热池首版不做,故无预热池表;扩展时再加 `warm_pool` 表,现有结构无需改动。

## 2. 平台级配置表

### 2.1 runtime — 运行时(链类型)

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| code | VARCHAR(64) UNIQUE | 运行时标识,如 `evm-hardhat`、`fabric` |
| name | VARCHAR(128) | 展示名 |
| eco | VARCHAR(32) | 生态标签,如 `evm`/`fabric`/`solana`,用于工具兼容匹配 |
| adapter_level | SMALLINT | 适配器最高层级:1=仅L1 / 2=L1+L2 / 3=含L3插件 |
| adapter_spec | JSONB | L1 声明式清单:启动进程/端口/资源/健康检查/默认工具/Pod 组拓扑/显式 Pod 间网络规则;可含 `capability_commands` 作为声明式 L2 deploy/tx/query/reset 执行入口 |
| capability_impl | VARCHAR(128) NULL | L2 标准能力实现标识(部署/交易/查询/重置的实现器) |
| plugin_ref | VARCHAR(128) NULL | L3 深度插件引用(可空) |
| selftest_status | SMALLINT | 接入即测结果:1待测 / 2通过 / 3失败 |
| selftest_detail | JSONB | 自检明细 |
| status | SMALLINT | 1可用 / 2接入中 / 3停用 |
| created_at / updated_at | TIMESTAMPTZ | |

### 2.2 runtime_image — 运行时镜像

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| runtime_id | BIGINT | 所属运行时 |
| image_url | VARCHAR(255) | 仓库镜像地址(Harbor) |
| version | VARCHAR(64) | 版本 |
| status | SMALLINT | 1可用 / 2停用 |
| prepulled | BOOLEAN | 是否已 DaemonSet 全节点预拉取 |
| prepull_status | SMALLINT | 预拉取状态:1未预拉取 / 2已完成 / 3失败 / 4进行中 |
| prepull_detail | JSONB | DaemonSet 名称、目标节点数、已完成节点、失败原因等明细 |
| prepulled_at | TIMESTAMPTZ NULL | 全目标节点预拉取完成时间 |
| genesis_baked | BOOLEAN | 是否已固化创世就绪状态 |
| is_default | BOOLEAN | 是否该运行时默认版本 |
| created_at | TIMESTAMPTZ | |

### 2.3 tool — 工具定义

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| code | VARCHAR(64) UNIQUE | 工具标识,如 `code-server`/`remix`/`blockscout`/`terminal` |
| name | VARCHAR(128) | 展示名 |
| kind | SMALLINT | 1=platform-builtin / 2=terminal / 3=web-embed |
| image_url | VARCHAR(255) NULL | web-embed 类工具镜像(内置类为空) |
| port | INT NULL | 接入端口(web-embed/terminal) |
| eco_tags | VARCHAR(255) | 适用生态标签(逗号分隔),与 runtime.eco 匹配过滤 |
| resource_spec | JSONB | 资源需求 |
| status | SMALLINT | 1可用 / 2停用 |
| created_at / updated_at | TIMESTAMPTZ | |

## 4. 多租户隔离

- 租户级表(`sandbox`/`sandbox_tool`/`sandbox_event`/`tenant_quota`)带 `tenant_id` 并启用 RLS,与 M1 同方案。
- 平台级配置表(`runtime`/`runtime_image`/`tool`)全局共享,仅平台管理员可写,所有租户可读用。
- 私有化:`tenant_id` 恒定,逻辑一致。

## 5. 关键索引

- `runtime(code)` UNIQUE、`tool(code)` UNIQUE
- `sandbox(tenant_id, status)`、`sandbox(tenant_id, owner_account_id)`、`sandbox(last_active_at)`(回收扫描)
- `sandbox(source_ref)`(来源级联回收)
- `sandbox(snapshot_expire_at)`(快照过期清理)
- `sandbox_event(tenant_id, sandbox_id, created_at)`

## 6. 审计与事件边界

- `sandbox_event` 是 M2 私有技术事件表,记录启动阶段、K8s 资源、文件保存、终端打开、回收步骤、错误明细等,用于沙箱排障与进度推送。
- 全平台关键操作审计必须写 M1 `audit_log`,包括运行时创建/更新/自检、镜像登记/预拉取、工具注册/更新、沙箱创建/暂停/恢复/销毁/回收、配额调整。
- M2 不新增审计表,也不把 `sandbox_event` 当作用户可查询审计来源。
