# M2 沙箱引擎 — 接口设计

> 风格:RESTful + WebSocket(终端/进度)。鉴权沿用 M1 JWT。
> 接口分两类:**内部调用接口**(供 M3/M7/M8 调用,服务间鉴权)与 **用户交互接口**(供前端工作台)。
> 最后更新:2026-06-04

---

## 1. 通用约定

- Base:`/api/v1/sandbox`
- 成功响应体:`{ "code": 0, "message": "ok", "data": {} }`;错误响应体:`{ "code": 22001, "message": "实验环境准备失败,请稍后重试", "trace_id": "..." }`,错误时不返回 `data`。
- `[内部]` 接口在模块化单体内优先通过 `internal/contracts` 调用;需要 HTTP 暴露时仅走内网路由,必须服务间鉴权,并强制传 `tenant_id`、调用方模块、`source_ref` 与 trace 上下文。
- 错误码段:`2xxxx` 沙箱引擎(21xxx 运行时/22xxx 沙箱/23xxx 工具/24xxx 配额)。

## 2. 运行时与镜像管理 `[平台管理员]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/runtimes` | 运行时列表(含适配器层级、自检状态) |
| POST | `/runtimes` | 注册运行时(提交 L1 清单 + 能力实现/插件引用) |
| PATCH | `/runtimes/{id}` | 更新运行时配置 |
| POST | `/runtimes/{id}/selftest` | 触发接入即测(起链→部署→查询→重置) |
| GET | `/runtimes/{id}/selftest` | 查询自检结果 |
| POST | `/runtimes/{id}/images` | 登记镜像版本 |
| POST | `/runtimes/{id}/images/{img}/prepull` | 触发 DaemonSet 全节点预拉取 |
| GET | `/runtimes/{id}/images/{img}/prepull` | 查询预拉取状态与节点进度 |
| GET | `/tools` | 工具列表 |
| POST | `/tools` | 注册工具定义 |

镜像登记请求必须携带不可变 digest;Cosign 签名与 Trivy/Harbor 扫描结论由后端启动时加载的 `SANDBOX_IMAGE_ATTESTATIONS_JSON` 受控证明清单校验,请求体不得直接提交签名/扫描结论:
```json
{
  "image_url": "harbor.chaimir.local/runtime/evm-hardhat@sha256:...",
  "version": "2026.06.01",
  "digest": "sha256:...",
  "genesis_baked": true,
  "is_default": true
}
```

web-embed 工具注册同样必须对 `image_url` 携带 `digest`,并命中受控证明清单;terminal/platform-builtin 工具不需要镜像字段。

预拉取响应:
```json
{
  "image_id": 88,
  "prepull_status": 4,
  "desired_nodes": 6,
  "ready_nodes": 3,
  "daemonset": "chaimir-prepull-88"
}
```

`prepulled=true` 与 `prepull_status=2` 只表示目标节点真实完成拉取。失败时返回用户向提示,节点、事件与镜像拉取错误只进入日志和 `prepull_detail`。

## 3. 沙箱生命周期 `[内部]`

### 3.1 创建沙箱
`POST /sandboxes` `[内部]`
请求:
```json
{
  "runtime_code": "evm-hardhat",
  "runtime_image_version": "2026.06.01",
  "tools": ["code-server", "terminal", "blockscout"],
  "init_code_ref": "minio://.../student-code",
  "init_script_ref": "minio://.../deploy.sh",
  "owner_account_id": 1001,
  "source_ref": "exp:2026:instance:55",
  "keep_alive": false,
  "snapshot_enabled": false,
  "keep_alive_minutes": 0,
  "snapshot_retention_minutes": 0
}
```
响应:`{ "sandbox_id": 9001, "namespace": "sbx-9001", "phase": 1 }`

> M2 立即返回,异步推进启动阶段;前端通过 §3.2 订阅进度。
> `runtime_image_version` 为内部引擎可选固定版本字段。M3 fresh judge 沙箱必须传入该字段,M2 按 `runtime_code + runtime_image_version` 选择已预拉取且固化创世态的镜像;普通交互沙箱未传时使用当前默认镜像。
> `keep_alive_minutes` 与 `snapshot_retention_minutes` 只能在对应开关开启时生效,且不得超过 `tenant_quota` 上限。

### 3.2 启动进度(WebSocket)
`WS /sandboxes/{id}/progress`
推送:`{ "phase": 2, "stage": "环境就绪", "message": "节点已就绪,可进入" }`
阶段:1分配环境 → 2环境就绪(可进入)→ 3个性化初始化中 → 4完全就绪。
失败推送示例:`{ "phase": 1, "stage": "准备失败", "message": "实验环境准备失败,请稍后重试", "trace_id": "..." }`。用户消息不得包含 Pod、Namespace、SQL、镜像拉取等技术细节。

### 3.3 查询 / 控制 / 销毁
- `GET /sandboxes/{id}`:状态、阶段、资源用量、工具接入地址。
- `POST /sandboxes/{id}/pause` / `POST /sandboxes/{id}/resume` `[内部]`:暂停/恢复(可选;默认空闲超时回收)。
- `DELETE /sandboxes/{id}` `[内部]`:主动销毁。
- `POST /sandboxes/recycle` `[内部]`:按 `source_ref` 批量级联回收(来源结束时调用)。
- **回收事件**:沙箱因任何原因进入回收(空闲/最长生命周期/就绪超时/主动)时,M2 发布 `sandbox.recycled` 事件(eventbus,携带 sandbox_id、source_ref、原因),来源方(experiment/contest)订阅后同步实例状态。**用事件而非反向接口回调,避免引擎层反向依赖业务层(见工程目录设计 §3.1.1)**。

回收请求:
```json
{
  "source_ref": "exp:2026:instance:55",
  "reason": "source_finished"
}
```

回收必须在代码保存、快照处理、K8s 资源删除、DB 标记 `destroyed` 后才发布 `sandbox.recycled`。删除失败时不得发布完成事件。

## 4. 沙箱交互 `[用户]`

### 4.1 终端(WebSocket)
`WS /sandboxes/{id}/terminal?container=<name>`
- 后端代理 K8s exec PTY;前端 xterm.js 双向流。
- 支持多容器切换(query 指定 container)。

### 4.2 文件操作
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/sandboxes/{id}/files?path=` | 列目录/读文件 |
| PUT | `/sandboxes/{id}/files?path=` | 写文件(触发防抖持久化 MinIO) |
| POST | `/sandboxes/{id}/files/save` | 立即持久化当前代码 |

文件路径必须相对 `workspace_dir`,服务端拒绝绝对路径、`..` 逃逸和符号链接逃逸。`files/save` 成功响应返回 `code_storage_key` 与 `code_hash`。

### 4.3 Web 工具代理
`ANY /sandboxes/{id}/tools/{tool_code}/*`
- HTTP 反向代理到沙箱内工具端口(code-server/Remix/blockscout 等),iframe 嵌入。

## 5. 链上能力 `[内部]`(L2 标准能力的 HTTP 暴露)

供 M3 判分、工具浏览器使用:

| 方法 | 路径 | 对应 L2 动作 |
| --- | --- | --- |
| POST | `/sandboxes/{id}/chain/deploy` | Deploy(code) 部署合约/链码 |
| POST | `/sandboxes/{id}/chain/tx` | SendTx(tx) 发起交易 |
| GET | `/sandboxes/{id}/chain/query?target=` | Query 查账户/区块/交易/合约状态 |
| POST | `/sandboxes/{id}/chain/reset` | Reset() 链恢复创世就绪态 |

> 这四个接口是跨运行时统一的;底层由各运行时的 L2 能力实现器适配(EVM JSON-RPC / Fabric SDK 等)。

## 6. 配额 `[学校管理员/平台管理员]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/quota` | 查看本租户配额与当前用量 |
| PATCH | `/quota` | 调整配额(平台管理员;私有化为校管) |

配额字段必须包含 `max_concurrent_sandbox`、`max_cpu`、`max_memory_mb`、`idle_timeout_min`、`max_lifetime_min`、`max_keepalive_min`、`max_snapshot_retention_min`。

## 7. 调用时序(典型:M7 实验启动)

```
M7 → POST /sandboxes(runtime+tools+code+source_ref)
M2 → 返回 sandbox_id, phase=1
前端 → WS /progress 订阅
M2 → phase=2 环境就绪(前端放行进入,展示工具工作台)
M2 → phase=3 个性化初始化(按钮置灰)→ phase=4 完全就绪(解灰)
学生 → WS /terminal 交互、/tools/* 用工具
M3 → /chain/query 判分
实验结束 → M7 调 /sandboxes/recycle(source_ref)
```
