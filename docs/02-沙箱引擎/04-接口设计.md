# M2 沙箱引擎 — 接口设计

> 风格:RESTful + WebSocket(终端/进度)。鉴权沿用 M1 JWT。
> 接口分两类:**内部调用接口**(供 M3/M7/M8 调用,服务间鉴权)与 **用户交互接口**(供前端工作台)。
> 最后更新:2026-06-04

---

## 1. 通用约定

- Base:`/api/v1/sandbox`
- 成功响应体:`{ "code": 0, "message": "ok", "data": {} }`;错误响应体:`{ "code": 22001, "message": "实验环境准备失败,请稍后重试", "trace_id": "..." }`,错误时不返回 `data`。
- `[内部]` 接口在模块化单体内优先通过 `internal/contracts` 调用;需要 HTTP 暴露时仅走内网路由,必须服务间鉴权,并强制传 `tenant_id`、调用方模块、`source_ref` 与 trace 上下文。
- 错误码段:`2xxxx` 沙箱引擎(21xxx 运行时/22xxx 沙箱/23xxx 工具/24xxx 配额)。

## 2. 运行时与镜像管理 `[平台管理员]`

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/runtimes` | 运行时列表(含适配器层级、自检状态) |
| POST | `/runtimes` | 注册运行时(提交 L1 清单 + 能力实现/插件引用) |
| PATCH | `/runtimes/{id}` | 更新运行时配置 |
| POST | `/runtimes/{id}/selftest` | 触发接入即测(起链→部署→查询→重置) |
| GET | `/runtimes/{id}/selftest` | 查询自检结果 |
| POST | `/runtimes/{id}/images` | 登记镜像版本 |
| DELETE | `/runtimes/{id}/images/{img}` | 停用镜像版本并删除对应预拉取 DaemonSet |
| POST | `/runtimes/{id}/images/{img}/prepull` | 触发 DaemonSet 全节点预拉取 |
| GET | `/runtimes/{id}/images/{img}/prepull` | 查询预拉取状态与节点进度 |
| GET | `/tools` | 工具列表 |
| POST | `/tools` | 注册工具定义 |

镜像登记请求必须携带不可变 digest;Cosign 签名与 Trivy/Harbor 扫描结论由后端启动时加载的 `SANDBOX_IMAGE_ATTESTATIONS_JSON` 受控证明清单校验,请求体不得直接提交签名/扫描结论:
```json
{
  "image_url": "harbor.chaimir/runtime/evm-hardhat@sha256:...",
  "version": "2026.06.01",
  "digest": "sha256:...",
  "genesis_baked": true,
  "is_default": true
}
```

web-embed 与 command-tool 工具注册必须在 `resource_spec.components[].image_url` 使用带 digest 的不可变镜像引用,并逐个组件命中受控证明清单;terminal/platform-builtin 工具不得声明组件镜像。
command-tool 工具必须只声明唯一 `command` 组件,不得声明 Service、route 或网络规则;`command_policy.allowed_commands` 是唯一可执行命令白名单,请求只能提交 argv 数组,不得提交 shell 字符串。
platform-builtin 工具必须在 `resource_spec.builtin_endpoint` 中声明平台端点模板,模板必须以 `/api/v1/sandbox/sandboxes/{sandbox_id}` 开头,由 M2 挂载时渲染实际沙箱 ID;此类工具不创建 sidecar,也不得落入 `/tools/{code}` HTTP 反向代理。
L2 运行时若不使用服务端注册的 `capability_impl` 或 `plugin_ref`,必须在 `adapter_spec.capability_commands` 中声明 `deploy`、`tx`、`query`、`reset` 四个受控命令,命令通过 stdin/stdout 交换 JSON。

预拉取响应:
```json
{
  "image_id": 88,
  "prepull_status": 4,
  "desired_nodes": 6,
  "ready_nodes": 3,
  "image_count": 12,
  "images": [
    "harbor.chaimir/runtime/evm-foundry@sha256:...",
    "harbor.chaimir/tool/code-server@sha256:..."
  ],
  "daemonset": "chaimir-prepull-88"
}
```

`prepulled=true` 与 `prepull_status=2` 只表示目标节点真实完成拉取。失败时返回用户向提示,节点、事件与镜像拉取错误只进入日志和 `prepull_detail`。

## 3. 沙箱生命周期 `[内部]`

### 3.1 创建沙箱
`POST /sandboxes` `[内部]`
请求:
```json
{
  "runtime_code": "evm-hardhat",
  "runtime_image_version": "2026.06.01",
  "tools": ["code-server", "blockscout"],
  "init_code_ref": "minio://.../student-code",
  "init_script_ref": "minio://.../deploy.sh",
  "owner_account_id": 1001,
  "source_ref": "experiment:2026:instance:55",
  "keep_alive": false,
  "snapshot_enabled": false,
  "keep_alive_minutes": 0,
  "snapshot_retention_minutes": 0
}
```
响应:`{ "sandbox_id": 9001, "phase": 1 }`

> M2 立即返回,异步推进启动阶段;前端通过 §3.2 订阅进度。
> `runtime_image_version` 为内部引擎可选固定版本字段。M3 fresh judge 沙箱必须传入该字段,M2 按 `runtime_code + runtime_image_version` 选择已预拉取且固化创世态的镜像;普通交互沙箱未传时使用当前默认镜像。
> 创建沙箱的后端调用方只能通过 `internal/contracts.SandboxService` 使用 M2,不得 import `internal/modules/sandbox` 内部包或复制 M2 校验逻辑。
> `keep_alive` 与 `snapshot_enabled` 默认必须为 `false`;只有业务需求明确要求保活或快照时才能开启。开启 `keep_alive` 必须同时提交正数 `keep_alive_minutes`;开启 `snapshot_enabled` 必须同时提交正数 `snapshot_retention_minutes`;未开启对应能力时分钟数字段必须为 `0`。调用方不得依赖隐式默认时长或上游模块私有默认值绕过 M2 配额校验。

### 3.2 启动进度(WebSocket)
`WS /sandboxes/{id}/progress`
服务端校验沙箱归属后订阅 `tenant:{tenant_id}:sandbox:{sandbox_id}:progress`,禁止使用旧式 `sandbox:{tenant_id}:{sandbox_id}:progress` 主题。
浏览器原生 WebSocket 不能设置 `Authorization` 头时,前端必须追加一次性 `token=<access_token>` 查询参数;后端只在平台代理入口校验该 token,不得向沙箱 Pod 或事件 topic 透传。
推送:`{ "phase": 2, "stage": "环境就绪", "message": "节点已就绪,可进入" }`
阶段:1分配环境 → 2环境就绪(可进入)→ 3个性化初始化中 → 4完全就绪。
失败推送示例:`{ "phase": 1, "stage": "准备失败", "message": "实验环境准备失败,请稍后重试", "trace_id": "..." }`。用户消息不得包含 Pod、Namespace、SQL、镜像拉取等技术细节。

### 3.3 查询 / 控制 / 销毁
- `GET /sandboxes/{id}`:状态、阶段、资源用量、工具接入地址;用户响应不得包含 Namespace、Pod、Service 等集群内部资源名。资源用量必须包含
  `cpu_usage_milli`、`memory_usage_mib`、`cpu_request_milli`、`cpu_limit_milli`、
  `memory_request_mib`、`memory_limit_mib`、`storage_bytes`;实时 CPU/内存用量来自
  Kubernetes `metrics.k8s.io` API,若集群未安装或暂不可读取 metrics-server,返回 `22031`。
- `POST /sandboxes/{id}/pause` / `POST /sandboxes/{id}/resume` `[内部]`:暂停/恢复(可选;默认空闲超时回收)。
- `DELETE /sandboxes/{id}` `[内部]`:主动销毁。
- `POST /sandboxes/recycle` `[内部]`:按 `source_ref` 批量级联回收(来源结束时调用)。
- `RestoreSandboxArchive` `[内部契约]`:把调用方已保存的公开代码/参战物归档安全恢复到 `workspace` 子目录。M2 必须校验对象引用属于当前租户的代码或附件桶;请求携带 `expected_hash` 时,M2 必须按对象实际内容计算 SHA-256 并逐字匹配;随后复用初始化归档安全原语重打 TAR,并只解包到调用方传入的相对目录。该能力不得用于隐藏测试、答案、flag 或判题私有资产。
- `PutSandboxPrivateArchive` `[内部契约]`:M3 注入隐藏测试、flag、答案摘要或评分脚本时只能调用该契约,目标域必须是 `judge-private`;M2 在后端用统一归档安全原语校验并重打 ZIP/TAR,拒绝目录穿越、绝对路径、Windows 盘符、重复条目、软/硬链接、特殊文件和炸弹包,随后只解包到 `judge-private` 卷域。声明 `judge-private` 的运行时不得把 `runtime_container` 标记为学生可进入容器;学生终端必须走单独 sidecar,避免隐藏资产与学生 shell 处于同一容器。该能力不提供用户 HTTP 接口,也不会触发工作区 MinIO 持久化。
- **回收事件**:沙箱因任何原因进入回收(空闲/最长生命周期/就绪超时/主动)时,M2 发布 `sandbox.recycled` 事件(eventbus,携带 sandbox_id、source_ref、原因),来源方(experiment/contest)订阅后同步实例状态。**用事件而非反向接口回调,避免引擎层反向依赖业务层(见工程目录设计 §3.1.1)**。

回收请求:
```json
{
  "source_ref": "experiment:2026:instance:55",
  "reason": "source_finished"
}
```

回收必须在代码保存、快照处理、K8s 资源删除、DB 标记 `destroyed` 后才发布 `sandbox.recycled`。删除失败时不得发布完成事件。

## 4. 沙箱交互 `[用户]`

### 4.1 终端(WebSocket)
`WS /sandboxes/{id}/terminal?container=<name>`
- 后端代理 K8s exec PTY;前端 xterm.js 双向流。
- 浏览器客户端使用一次性 `token=<access_token>` 建立 WebSocket;服务端完成会话校验后只把租户/账号身份写入请求上下文,不得把 token 传入容器环境或命令行。
- 支持多容器切换(query 指定 container),但目标容器必须在运行时声明中显式标注 `chaimir.io/student-access=true`。
- 未传 `container` 时,M2 只选择第一个显式学生入口容器;没有安全入口容器则拒绝连接。禁止默认进入运行时主容器,避免通过 shell 读取 `runtime-state`、`judge-private`、Secret 或控制面凭据。

### 4.2 文件操作
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/sandboxes/{id}/files?path=` | 列目录/读文件 |
| PUT | `/sandboxes/{id}/files?path=` | 写文件(触发防抖持久化 MinIO) |
| POST | `/sandboxes/{id}/files/save` | 立即持久化当前代码 |

文件路径必须相对 `workspace_dir`,服务端拒绝绝对路径、`..` 逃逸和符号链接逃逸。`files/save` 成功响应返回 `code_storage_key` 与 `code_hash`。文件操作接口只能访问 `workspace`,不得读取或写入 `runtime-state`、`judge-private`、Secret、隐藏测试、答案或 flag。

### 4.3 Web 工具代理
`ANY /sandboxes/{id}/tools/{tool_code}/*`
- HTTP 反向代理到沙箱内工具端口(code-server/Remix/blockscout 等),iframe 嵌入。
- 浏览器首次打开工具 URL 时可带一次性 `token=<access_token>`;M2 校验用户、租户、owner 后写入当前工具路径受限的 HttpOnly Cookie,再重定向到无 token 的同一路径。后续工具 HTML、静态资源、WebSocket 或 API 请求只能通过该路径受限 Cookie 经过平台代理鉴权,不得裸连 Service 或把 token、平台 Cookie、内部服务签名转发给工具容器。
- 代理必须重写同源 `Location` 到 `/api/v1/sandbox/sandboxes/{id}/tools/{tool_code}` 前缀,并把上游 `Set-Cookie` 限定到当前工具路径,避免不同工具或平台同域 Cookie 混用。

### 4.4 受控命令工具
`POST /sandboxes/{id}/command-tools/{tool_code}/run`
- 只允许沙箱 owner 调用已挂载且 Ready 的 command-tool。
- 请求:`{ "command": ["slither", "."], "stdin_base64": "", "timeout_sec": 30 }`。
- 响应:`{ "stdout_base64": "...", "stderr_base64": "..." }`。
- 后端通过 `internal/platform/k8s.Exec` 执行,不自建 K8s 客户端;只记录工具 code 和入口命令,不记录 stdin/stdout 内容。

## 5. 链上能力 `[内部]`(L2 标准能力的 HTTP 暴露)

供 M3 判分、工具浏览器使用:

| 方法 | 路径 | 对应 L2 动作 |
| --- | --- | --- |
| POST | `/sandboxes/{id}/chain/deploy` | Deploy(code) 部署合约/链码 |
| POST | `/sandboxes/{id}/chain/tx` | SendTx(tx) 发起交易 |
| GET | `/sandboxes/{id}/chain/query?target=` | Query 查账户/区块/交易/合约状态 |
| POST | `/sandboxes/{id}/chain/reset` | Reset() 链恢复创世就绪态 |

> 这四个接口是跨运行时统一的;底层由各运行时的 L2 能力实现器适配(EVM JSON-RPC / Fabric SDK 等)。

| 方法 | 路径 | 说明 |
