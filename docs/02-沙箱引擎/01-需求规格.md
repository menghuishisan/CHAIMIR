# M2 沙箱引擎 — 需求规格

> 模块定位:平台技术底座。提供"能运行区块链的隔离环境"的纯技术能力,被实验(M7)、竞赛(M8)、评测(M3)复用。
> 核心理念:业务薄、引擎厚。M2 不含任何教学/竞赛业务概念。
> 最后更新:2026-06-04

---

## 1. 模块目标

- 按"链类型 + 工具集"按需创建**隔离沙箱**,生命周期可控。
- **可扩展接入任意区块链**:标准链加一份声明式清单即可,不改平台代码。
- **启动快**:简单实验 5 秒内可用,复杂多节点 15 秒内,分阶段进度反馈。
- **可插拔工具**:IDE/Remix/区块链浏览器/终端等工具按需挂载,不绑定单一 IDE。
- 多租户隔离、资源配额、安全沙箱。

## 2. 职责边界

| M2 负责 | M2 不负责(归属) |
| --- | --- |
| 运行时适配器(什么链)、镜像管理 | 判分/评测(→ M3) |
| 容器编排(K8s)、沙箱生命周期 | 仿真可视化动画(→ M4) |
| 工具接入(IDE/Remix/终端/浏览器) | "第几个实验、谁的、配什么题"(→ M7/M8) |
| 沙箱内链节点与工具链管理 | 多人协作组网(→ M7/M8 业务层) |
| 链上状态读取接口(供 M3/工具用) | 题库与模板(→ M5) |
| 资源配额、回收、安全隔离 | 课程/班级/成绩(→ M6) |

**核心交互**:调用方传入 `链类型 + 工具集 + 初始代码/脚本 + 来源标识`,M2 返回一个可交互沙箱。M2 不解析"来源标识"的业务含义,仅用于归属与回收。

## 3. 功能需求

### A. 运行时与镜像管理

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 运行时注册 | 注册一种区块链运行时(链类型),含声明式适配器清单 |
| A2 | 三层适配器 | L1 声明式清单(YAML,零代码)/ L2 标准能力接口(部署合约/发交易/查询/重置)/ L3 深度定制插件(可选) |
| A3 | 镜像管理 | 镜像登记、版本、DaemonSet 全节点预拉取 |
| A4 | 接入即测 | 每个适配器附自检套件(起链→部署样例→查询→重置),通过才算接入成功 |
| A5 | 链状态固化 | 镜像内预置创世状态(创世块/账户/已编译依赖),启动即加载就绪 |

### B. 沙箱生命周期

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 创建沙箱 | 按链类型 + 工具集创建隔离环境(K8s Namespace/Pod) |
| B2 | 分层启动 | 环境就绪(5秒内可进)与个性化初始化(异步)分离,分阶段进度反馈 |
| B3 | 交互 | 终端(K8s exec)、文件读写、命令执行 |
| B4 | 暂停/恢复 | 可选;默认空闲超时回收 |
| B5 | 销毁/回收 | 主动销毁 + 空闲超时 + 最长生命周期强制 + 来源结束级联 |
| B6 | 状态查询 | 沙箱状态、资源用量、就绪进度 |

### C. 工具接入

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 工具注册 | 声明工具:类型(iframe/ws终端/http代理)、连接端口、资源需求 |
| C2 | 终端 | 基于 K8s exec PTY + WebSocket 代理,前端 xterm.js 渲染,不自研协议 |
| C3 | Web 工具代理 | code-server/Remix/区块链浏览器等通过 HTTP 反向代理接入 |
| C4 | 动态工作台 | 前端按实验配置的工具清单动态渲染(本模块提供工具元数据与接入端点) |
| C5 | 工具-链兼容过滤 | 按链生态过滤适用工具 |

### D. 链上状态访问

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 统一状态接口 | 通过 L2 标准能力查询账户/区块/交易/合约状态 |
| D2 | 部署与交易 | 供调用方在沙箱内部署合约、发交易 |
| D3 | 链重置 | 将链恢复到创世就绪态 |

### E. 持久化

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 代码实时持久化 | 学生代码挂载卷 + 防抖保存到对象存储(MinIO),绝不丢失 |
| E2 | 状态可重建 | 链状态默认不快照,记录代码与部署脚本,恢复时重放重建 |
| E3 | 按需保活/快照 | 教师对重建代价高的实验可显式开启保活/磁盘快照,默认关,有最长时长上限 |

### F. 资源与配额

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F1 | 租户配额 | 按租户限制并发沙箱数/CPU/内存,防止单校占满集群 |
| F2 | 单沙箱限额 | ResourceQuota/LimitRange 限制单沙箱资源 |
| F3 | 回收策略 | 空闲超时(默认 30min)、最长生命周期、来源级联回收 |

## 4. 非功能需求

- **启动性能**:简单实验(单节点)环境就绪 ≤5s;复杂多节点 ≤15s;镜像预拉取消除下载等待。
- **隔离**:每沙箱独占 K8s Namespace,NetworkPolicy 默认 deny-all,沙箱间/租户间网络隔离。
- **可扩展**:标准链零代码接入(L1 清单);新链接入有自检保证真实可用。
- **多租户/私有化同构**:SaaS 按租户配额与隔离;私有化单租户,逻辑一致。
- **预拉取闭环**:`runtime_image.prepulled=true` 只能在目标节点镜像真实可用后写入;禁止仅修改数据库标记。
- **保活/快照闭环**:保活与快照必须受租户配额、最长保活时长和最长快照保留时长约束;超限必须拒绝或回收,不能静默延长。
- **错误与审计**:HTTP 错误响应统一为 `{ code, message, trace_id }`;技术原因只进结构化日志。关键操作同时写 M1 `audit_log`,M2 `sandbox_event` 只记录沙箱技术事件。

## 5. 明确不做(边界)

- 不做判分评测(M3)、不做仿真可视化(M4)。
- 不做多人协作组网(留 M7/M8)。
- 首版不做预热池(数据模型/接口预留扩展点,上线后按需开启)。
- 不感知教学/竞赛业务语义(仅持有来源标识用于归属与回收)。
- 不绕过 `internal/contracts` 调用其他模块;M2 不直接 import M3/M7/M8 等模块 package。

## 6. 技术取向

K8s(SaaS)/ k3s(私有化轻量)为编排底座;Go(client-go/operator)实现控制面;终端复用 K8s exec;对象存储 MinIO 持久化代码;镜像仓库 Harbor + 安全扫描。最终技术细节由《技术选型》统一收口。
