# M11 成绩中心 — 需求规格

> 模块定位:跨课程成绩聚合层。从 M6 读单课程成绩,聚合 GPA、学业预警、成绩单、审核工作流。
> 核心理念:只读聚合不跨写(同 M9 哲学)。
> 最后更新:2026-05-29

---

## 1. 模块目标

- **跨课程聚合**:从 M6 读各课程成绩,聚合学生整体学业数据。
- **GPA 计算**:等级映射(分数段→等级→绩点)+ 学分加权。
- **成绩审核工作流**:教师提交 → 学校管理员审核 → 锁定;支持申诉。
- **学业预警**:挂科/低 GPA 预警。
- **成绩单**:生成正式成绩单(PDF)。

## 2. 职责边界

| M11 负责 | M11 不负责(归属) |
| --- | --- |
| 跨课程成绩聚合(只读 M6) | 单课程成绩计算(→ M6) |
| GPA、等级映射、学分加权 | 课程/作业/实验(→ M6/M7) |
| 成绩审核工作流、申诉 | 竞赛成绩(竞赛不计 GPA) |
| 学业预警、成绩单 | 用户/学籍(→ M1) |

**核心原则**:单课程成绩归属 M6,M11 **只读聚合并仅保存自有审核/GPA/预警/成绩单元数据**。竞赛成绩不计 GPA(沿用旧版)。

## 3. 功能需求

### A. 成绩聚合与 GPA
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| A1 | 跨课程聚合 | 读 M6 各课程成绩 + 学分 |
| A2 | 等级映射 | 分数段→等级→绩点,按学校配置 |
| A3 | GPA 计算 | Σ(绩点×学分)/Σ学分 |
| A4 | 学期/总 GPA | 按学期与累计 |

### B. 成绩审核工作流
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| B1 | 提交审核 | 教师提交本课程成绩待审 |
| B2 | 审核 | 学校管理员通过/驳回 |
| B3 | 锁定 | 通过后锁定,不可随意改 |
| B4 | 解锁回退 | 管理员可解锁回退 |

### C. 申诉
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| C1 | 学生申诉 | 时效内(如 30 天)对成绩申诉 |
| C2 | 改分处理 | M11 记录受理与解锁审核状态;单课程改分由 M6 执行,M6 成绩变更事件触发 M11 GPA 重算 + 复核 |

### D. 学业预警
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| D1 | 预警规则 | 挂科数/GPA 阈值 |
| D2 | 预警生成 | 触发预警 + 经 M10 通知 |

### E. 成绩单
| 编号 | 功能 | 说明 |
| --- | --- | --- |
| E1 | 成绩单生成 | 学生学业成绩单 PDF |
| E2 | 导出 | 学校批量导出 |

## 4. 非功能需求

- **只读不跨写**:不写 M6 单课程成绩,也不调用 M6 写接口;M11 仅存聚合/审核/预警/成绩单元数据。
- **可重算**:改分/申诉后 GPA 一致重算。
- **多租户**:成绩数据 tenant_id 隔离。
- **缓存**:跨课程聚合读耦合较重,需缓存学业概览(明确缓存失效策略)。

## 5. 明确不做(边界)

- 不算单课程成绩(M6)。
- 不计竞赛成绩进 GPA。
- 不管课程/作业/实验业务。

## 6. 技术取向

Go + PostgreSQL;成绩单 PDF 生成;聚合读 M6 + 缓存。最终技术细节由《技术选型》统一收口。
