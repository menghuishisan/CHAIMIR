# M11 成绩中心 — 数据模型

> PostgreSQL,全部租户级(tenant_id + RLS)。M11 只存聚合/审核/预警元数据,单课程成绩在 M6。
> 建模遵循:消灭数据冗余、表合并有度。
> 最后更新:2026-05-29

---

## 1. 表清单总览

| 表名 | 用途 |
| --- | --- |
| `grade_level_config` | 等级映射配置(分数段→等级→绩点) |
| `semester` | 学期 |
| `grade_review` | 成绩审核记录 |
| `grade_lock_outbox` | 成绩锁定状态事件 outbox |
| `student_semester_grade` | 学生学期成绩聚合(GPA) |
| `grade_appeal` | 成绩申诉 |
| `academic_warning` | 学业预警 |
| `transcript_record` | 成绩单记录 |

> 消灭冗余/合并说明:
> - **单课程成绩不复制**:`student_semester_grade` 存聚合结果(学期 GPA/学分),明细课程成绩从 M6 实时读,不复制 M6 成绩到 M11。
> - **等级映射按学校配置**:`grade_level_config` 一校一套(或按学期),不写死。
> - **审核记录与聚合分表**:审核是流程流水,聚合是结果,分开。

## 3. 审核与聚合

### 3.1 grade_review — 成绩审核

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| course_id | BIGINT | 关联 M6 课程 |
| semester_id | BIGINT NULL | 审核通过时归属学期,供后续 M6 成绩变更事件重算 GPA |
| submitter_id | BIGINT | 提交教师 |
| reviewer_id | BIGINT NULL | 审核学校管理员 |
| status | SMALLINT | 1待审/2通过/3驳回 |
| is_locked | BOOLEAN | 审核状态是否已进入锁定态,供申诉/重审流程使用 |
| comment | VARCHAR(500) NULL | |
| submitted_at / reviewed_at | TIMESTAMPTZ | |

### 3.2 grade_lock_outbox — 成绩锁定事件 outbox

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| review_id | BIGINT | 关联 `grade_review(tenant_id,id)` |
| course_id | BIGINT | M6 课程 ID |
| locked | BOOLEAN | true 锁定 / false 解锁 |
| reason | VARCHAR(64) | 事件原因,如 review_approved/appeal_accepted |
| trace_id | VARCHAR(128) | 原请求或事件链路编号 |
| status | SMALLINT | 1待发布/2已发布/3发布失败待重试/4发布中 |
| retry_count | INT | 发布领取次数 |
| last_error | VARCHAR(255) NULL | 脱敏失败原因 |
| created_at / updated_at | TIMESTAMPTZ | |

> 成绩锁定状态变更必须与 `grade_review` 状态变更在同一事务内写入本表,再由后台任务发布 `grade.review.lock_changed` 驱动 M6 同步写保护投影。发布失败只更新 outbox 状态并重试,不得在事务提交后裸调用事件总线作为唯一投递路径。

### 3.3 student_semester_grade — 学期成绩聚合

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |

## 4. 申诉与预警

### 4.1 grade_appeal — 申诉

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK | |
| tenant_id | BIGINT | |
| student_id | BIGINT | |
| course_id | BIGINT | |
| reason | TEXT | 申诉理由 |
| status | SMALLINT | 1待处理/2受理改分中/3已完成/4驳回 |
| handler_id | BIGINT NULL | |
| result_comment | TEXT NULL | |
| created_at / handled_at | TIMESTAMPTZ | |

唯一约束:`(tenant_id, student_id, course_id) WHERE status IN (1,2)`,同一学生同一课程只能存在一个待处理或受理改分中的申诉,防止反复无效申诉。

> 受理改分:发布事件驱动 M6 解除本模块 `course_grade.is_locked` 写保护投影 → 教师在 M6 改单课程成绩 → 触发 M11 GPA 重算。

## 6. 多租户与索引

- 全部表 `tenant_id` + RLS(同 M1)。
- 所有租户级 RLS policy 必须同时包含 `USING` 与 `WITH CHECK`,禁止只限制读而放开错误租户写入。
- 关键索引:
  - `grade_review(tenant_id, course_id, status)`
  - `grade_lock_outbox(status, created_at)`
  - `student_semester_grade(student_id, semester_id)` UNIQUE
  - `grade_appeal(tenant_id, student_id, status)`
  - `grade_appeal(tenant_id, student_id, course_id) WHERE status IN (1,2)` UNIQUE
  - `academic_warning(tenant_id, student_id, semester_id)`
