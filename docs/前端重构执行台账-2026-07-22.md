## 0. 边界与规则（强制）

- **严格门禁**：Phase 1 未通过前，`frontend/apps/web/src/features/**` 零 UI 改动；`packages/` 四包按本台账 §1 收敛，业务页面不动。
- **分层推进**：P0 → P1 → P2 跨端一次扫完，不做纵向按端推进。
- **不兼容旧逻辑 / 旧数据**：该重构就完整重构，该清理就清理干净。禁止为了保留旧字段名、旧枚举、旧对象形态、旧 CSS 类名、旧样式约定、旧组件 API 或旧数据结构而写兼容分支、降级 shim、旧路径别名或"两套并存"。存量数据通过版本化迁移统一，种子、导入器、后端校验、API 类型和全部页面同步更新；不符合当前契约的数据由边界明确拒绝并给出用户向提示。
- **后端契约缺失只落文档**：不臆造接口 / 字段 / 进度 / 图表 / 推荐 / 状态机；把缺口写入《前端后端功能对齐清单.md》"待补齐"段落，前端**删除**该 UI 承诺，不做占位 / 假态 / 假接口。
- **只认新证据**：验收截图统一存放 `.tmp/e2e-<日期>/`，Playwright 视口固定 `375 / 768 / 1024 / 1440`；`.tmp/e2e-2026-07-14`、`.tmp/e2e-regression-2026-07-15` 仅作历史存档，不作为视觉正确性依据。
- **Skill 参考来源**：`D:\skill\emilkowalski-skills` 的 `emil-design-eng`、`review-animations/STANDARDS.md`、`animation-vocabulary`；当 Skill 与本项目规范冲突时，以本项目规范为准（见规范 §4.2「明确不采纳」）。
- **禁止占位**：与 CLAUDE.md §4 一致，禁止 TODO 占位、禁止"先做个能跑的版本"；功能要么完整实现，要么明确写入台账 §6「后端契约缺失」并从前端删除。

## 6. 后端契约缺失（前端删，不臆造）

> 与《前端后端功能对齐清单.md》"待补齐"段落同步。Phase 1 起每项以证据登记，前端删除对应 UI 承诺；不做占位、假态、假接口；不为已下线接口保留 UI 入口。

- 结构化编辑 schema 缺口：影响系统配置、租户功能开关、链操作 payload、仿真参数、判分附加输入、竞赛规则、漏洞源与漏洞链步骤；公开 DTO 仅提供自由对象而无字段描述，详见《前端后端功能对齐清单.md》§6.5。前端删除自由键值编辑器，保留已建模字段、已存值与可读展示；不做假 schema 或键名允许列表。
- 实验报告展示缺口：`ReportDTO` 没有学生展示身份或授权内容读取接口，前端不展示 `student_id`、`content_ref` 或原始外链；后端已有 `POST /experiment/reports/{id}/grade`，因此保留基于真实 DTO 的分数与评语批改入口，正文查看待后端补齐授权接口后再增加。
- 学生单条详情缺口：实验和竞赛仅提供学生列表接口；教师单条竞赛与赛果快照接口不授权学生。学生详情不得调用教师接口，当前只组合学生列表中的真实定义与共享题目/榜单，待后端补齐学生视角单条详情后直接切换。
- 课程作业列表缺口：后端没有学生视角的课程作业列表接口，前端不调用教师管理接口、不伪造列表；学生只能从后端已返回的真实作业入口进入详情。
- 教学身份展示缺口：课程成员、提交和课程成绩 DTO 只返回学生标识，没有学生姓名等展示字段；前端不额外拼接身份接口或缓存第二份身份数据，待后端在公开 DTO 补齐展示信息。
- 竞赛队伍展示缺口：榜单 DTO 只返回 `team_id`、名次和成绩；学生榜单只展示真实名次与成绩，不把内部标识伪装成队伍名称。
- 平台告警学校展示缺口：告警 DTO 只返回 `tenant_id`，没有学校名称；平台告警只展示平台范围或学校范围，不把租户标识当名称。
